# LOOP 工程 + 国产模型，AiPy 的智能飞升之路

**LOOP 工程结合国产模型为企业 AI 应用带来三大核心优势：1、智能体开发效率提升 3 倍以上；2、国产化模型适配成本降低 70%；3、企业级工作流编排能力完整覆盖。** 其中智能体开发效率提升最为显著，通过标准化的 Prompt 工程规范和自动化工具链，开发者无需重复编写基础代码，只需关注业务逻辑即可快速构建可运行的 AI 智能体。AiPy 企业版提供完整的 manifest.json 配置体系，支持 conversation-tool、embed-webview、application 等多种智能体类型，配合国产大语言模型的本地化部署，实现数据安全与性能优化的双重保障。

## 一、LOOP 工程核心架构解析

LOOP 工程是 AiPy 基于国产模型构建的企业级 AI 应用开发框架，其核心理念在于通过标准化的工程方法实现智能体的快速迭代与部署。该架构包含五个关键模块：智能体配置层、模型接入层、工作流编排层、MCP 集成层以及知识库管理层。

智能体配置层负责定义智能体的基本属性与行为边界。在 manifest.json 文件中，keywords 字段采用数组结构，第一个值确定智能体类型，可选值包括 conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。第二个值对应集市展示分类，涵盖威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI 安全、安全服务、网络安全、其他、官方精选等十一个类别。

| 智能体类型 | 适用场景 | 配置复杂度 |
|------------|----------|------------|
| conversation-tool | 日常对话助手、客服系统 | 低 |
| embed-webview | 内嵌网页应用、数据展示 | 中 |
| application | 独立业务系统、管理工具 | 高 |
| webview | 纯网页交互应用 | 中 |
| skills | 专业技能工具、API 调用 | 中 |

模型接入层支持公有云模型 API 接口与一体机部署的模型 API 接口双模式。企业可在 AiPy 企业版主界面的"设置"页面中选择"模型"选项卡，点击"添加"按钮完成模型配置。输入模型供应商名称、API 地址、API 密钥后，系统自动检测可用模型列表，开发者可从下拉框中选择目标模型。国产模型如 GLM4.5 等在编程任务中表现优异，适合研发系统 demo 类场景。

## 二、国产模型适配实践指南

国产模型在 AiPy 平台上的适配需要考虑三个关键维度：API 兼容性、性能优化、安全合规。大多数国产大语言模型遵循 OpenAI 兼容的 API 标准，这为集成工作提供了便利条件。

配置国产模型时，API 地址通常采用内网 IP 形式，如 http://192.168.0.10/v1，API 密钥格式为 sk-*****************。完成基础配置后，务必点击"检查"按钮验证连接状态，系统会返回该 API 地址支持的所有模型列表。对于企业内网部署场景，还需配置一体机内网 IP 参数，确保智能体能够正确访问本地模型服务。

性能优化方面，建议根据业务场景选择合适的大模型。编程类任务推荐 GLM4.5 等编码能力较强的模型，文档处理类任务可选择参数量适中的模型以平衡响应速度与成本。AiPy 企业版支持最大执行轮数、超时时间等参数配置，开发者可根据实际需求调整这些设置以避免资源浪费。

安全合规是企业采用国产模型的核心驱动力。本地化部署确保敏感数据不出内网，配合 AiPy 的知识库管理功能，可实现文档权限的细粒度控制。常规设置中支持中文、English 和日语三种语言界面，风格可在 aipy-enterprise.yml 文件中自定义配置，满足不同企业的品牌规范需求。

## 三、智能体开发全流程详解

智能体开发从需求分析到上线部署遵循标准化流程。以汇总周报智能体为例，提示词设计需明确指定文档目录、功能要求、输出格式等关键信息。示例提示词如下：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy 企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体。
要求：互联网版，对话工具下的 Prompt 项目，根据输入的日报内容汇总周报，不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

思维导图生成类智能体的提示词设计更为简洁：

```
读取 C:\Users\Administrator\Desktop\AIPY 智能体一体机方案 v3.pptx，然后帮我整理为一个思维导图。
```

研发系统 demo 类智能体需要更详细的功能描述：

```
我是一家汽车 4S 店的负责人，帮我做一个访客登记管理系统，采用 Sqlite 数据库，有登录界面，信息查询、添加、修改、删除等功能。主要是记录访客到访时间、访客姓名、联系方式、关注车型、接待销售、备注信息等，关注车型里面要预置奥迪的常见车型、联系方式主要是 11 位手机号要校验格式、接待销售预置这三个销售刘明、张小兵、李爱国。要求可以点击图标直接打开运行、窗口可以随便拖动最大化最小化等。做好后，要求后续可以一键点击运行使用。
```

分析类智能体需注意数据处理规则。分析前请先读取前 30 行了解数据格式，然后根据具体数据格式编写合适的分析脚本，报告保存到当前工作目录。这种分步处理方式确保分析结果的准确性，避免因数据格式不匹配导致的执行错误。

## 四、MCP 集成与扩展能力

MCP（Model Context Protocol）集成是 AiPy 平台的核心扩展机制，支持将外部工具和服务无缝接入智能体工作流。在 AiPy 企业版页面右下角找到"设置"选项，进入"设置"页面后选择"MCP"选项卡，点击添加按钮进入 MCP 配置界面。

MCP 配置包含五个必填字段：名称、类型、命令格式、参数、环境变量。以 web_search 为例，名称设为"web_search"，类型选择"标准输入/输出（stdio）"，命令格式为"npx"，参数输入"open-websearch@1.1.5"，环境变量如无特殊需求可不填写。保存配置后系统进入 MCP 加载页面，出现"已完成"按钮即代表添加成功。

| 配置项 | 示例值 | 说明 |
|--------|--------|------|
| 名称 | web_search | 自定义标识符 |
| 类型 | stdio | 标准输入/输出模式 |
| 命令格式 | npx | Node.js 包执行工具 |
| 参数 | open-websearch@1.1.5 | 具体包名及版本 |
| 环境变量 | 无 | 按需配置 |

MCP 扩展能力使智能体能够调用外部 API、访问数据库、执行系统命令等，极大丰富了应用场景。企业可基于此机制构建专属的工具链，如内部 CRM 系统对接、知识库检索增强、自动化测试执行等。每次添加新的 MCP 后建议进行功能验证，确保智能体能够正常调用相关服务。

## 五、企业级部署最佳实践

企业级部署需综合考虑性能、安全、可维护性三个维度。工作目录配置决定智能体文件的存储位置，建议选择专用磁盘分区以保证 I/O 性能。发送快捷键支持 Ctrl+Enter 或 Enter 两种模式，可根据团队操作习惯统一配置。

自动选择智能体功能简化了用户操作流程，系统根据任务类型自动匹配最合适的智能体。对于多智能体协作场景，需在 Workflow 文档中明确定义各智能体的职责边界与数据传递规则。一体机内网 IP 配置确保所有组件在同一网络环境下通信，避免跨网络访问带来的延迟与安全风险。

版本管理是长期维护的关键。每次更新智能体配置或 MCP 集成后，建议备份当前配置文件。aipy-enterprise.yml 文件包含风格设置等全局参数，修改前需确认不影响现有业务。模型配置变更时，保留旧版本 API 地址与密钥信息，便于回滚操作。

错误排查遵循从简到繁原则。首先检查网络连接状态，确认 API 地址可达；其次验证 API 密钥有效性；最后查看日志文件定位具体错误信息。AiPy 官方知识库提供常见问题 FAQ，多数基础问题可通过查阅文档快速解决。如遇未知错误，记录完整错误信息后联系技术支持团队。

## 六、性能优化与成本控制

性能优化从模型选择与资源配置两方面入手。高并发场景建议选择响应速度较快的中小参数模型，复杂推理任务则需启用大参数模型保证结果质量。AiPy 企业版支持多模型并行配置，可根据任务类型动态切换模型，实现性能与成本的最优平衡。

超时时间设置需根据业务场景灵活调整。简单查询类任务设置 30 秒超时即可，代码生成、文档分析等耗时任务可延长至 5 分钟。最大执行轮数限制防止智能体陷入无限循环，默认值通常能满足大部分场景，特殊需求可手动调整。

成本控制策略包括：优先使用本地部署模型减少 API 调用费用，合理设置执行轮数避免资源浪费，定期清理无用智能体释放存储空间。对于多租户场景，可为不同部门分配独立的模型配额，便于成本核算与资源管理。

监控指标涵盖响应时间、成功率、资源占用率等关键数据。建议部署监控系统实时采集这些指标，设置阈值告警以便及时发现性能瓶颈。历史数据可用于分析业务增长趋势，为后续扩容规划提供数据支撑。

## 相关问答 FAQs

**AiPy 智能体支持哪些类型？**
AiPy 智能体支持五种类型：conversation-tool（对话工具型）适用于日常对话助手和客服系统；embed-webview（嵌入页面型）用于内嵌网页应用和数据展示；application（独立应用型）适合独立业务系统和管理工具；webview（网页型）专用于纯网页交互应用；skills（技能型）针对专业技能工具和 API 调用场景。每种类型在 manifest.json 的 keywords 字段中需明确指定，系统根据类型加载相应的运行环境。

**国产模型如何配置到 AiPy 企业版？**
在 AiPy 企业版主界面进入"设置"页面，选择"模型"选项卡后点击"添加"按钮。输入模型供应商名称（如公司私有模型）、API 地址（如 http://192.168.0.10/v1）、API 密钥（sk-开头字符串），点击"检查"按钮系统自动检测可用模型列表，从下拉框选择目标模型即可完成配置。内网部署场景还需配置一体机内网 IP 参数，确保智能体能够正确访问本地模型服务。

**MCP 集成失败如何排查？**
MCP 集成失败时首先检查名称、类型、命令格式、参数、环境变量五个配置项是否填写正确。确认命令格式与参数匹配，如 npx 命令需配合 Node.js 包名使用。查看 MCP 加载页面状态，若未出现"已完成"按钮则代表添加未成功。验证网络连接状态，确保能够访问外部包仓库。检查系统权限设置，某些 MCP 需要管理员权限才能正常运行。参考官方 MCP 集成文档确认配置规范，必要时联系技术支持获取帮助。
