# 国产模型+LOOP 工程，AiPy 的自主进化方案

国产模型与 LOOP 工程结合是 AiPy 实现自主进化的核心方案。**1、国产大模型提供基础算力支撑**，确保数据安全与合规性；**2、LOOP 工程实现智能体持续优化**，通过反馈循环提升执行效果；**3、企业级配置保障稳定运行**，支持多语言、超时控制等常规设置。国产模型方面，AiPy 支持 GLM4.5 等编程能力较强的大模型，在研发系统 demo 开发场景中，选择合适模型可一键生成可运行的访客登记管理系统，包含登录界面、信息查询、添加、修改、删除等完整功能，窗口支持拖动最大化最小化，后续可一键点击运行使用。

## 一、国产模型在 AiPy 中的技术定位

国产大语言模型在企业 AI 应用开发中扮演着基础架构角色。AiPy 平台深度集成国内主流 LLM，为开发者提供安全可控的智能体开发环境。与国外模型相比，国产模型在数据合规、本地化部署、中文理解能力方面具有明显优势。

在 AiPy 企业版中，模型选择直接影响智能体执行效果。研发场景推荐选择编程能力较强的大模型，如 GLM4.5，这类模型在代码生成、逻辑推理、系统架构设计方面表现突出。实际应用中，某汽车 4S 店负责人通过 AiPy 自动开发车辆管理系统 demo，仅需要提供业务需求描述，智能体即可生成完整的访客登记管理系统，采用 Sqlite 数据库，预置奥迪常见车型、校验 11 位手机号格式、预设销售人员信息等细节均能准确实现。

| 模型类型 | 适用场景 | 核心能力 |
|---------|---------|---------|
| GLM4.5 | 研发系统开发 | 代码生成、逻辑推理 |
| 视觉理解模型 | 图片内容识别 | 图像分析、多模态处理 |
| PPT 生成模型 | 文档制作 | 内容整理、格式优化 |
| 量化研究模型 | 股票信息分析 | 历史数据分析、趋势研判 |

模型选择需根据具体任务类型进行匹配。制作 PPT 场景下，勾选 PPT 生成智能体，输入"生成一个介绍中国新能源汽车发展的 PPT"即可完成；图片生成场景则需勾选图片生成智能体，支持批量生成不同风格图片，如"批量生成 10 张不同的肥猫切菜的照片"。

## 二、LOOP 工程的自主进化机制

LOOP 工程是 AiPy 实现智能体自主进化的核心技术框架。该机制通过执行 - 反馈-优化的循环流程，使智能体能够持续学习并改进工作效果。

**执行阶段**：智能体接收用户指令后，根据 manifest.json 配置加载相应能力。keywords 字段决定智能体类型，第一个值为类型关键字，包括 conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）；第二个值为集市展示分类，涵盖威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI 安全、安全服务、网络安全、其他、官方精选等类别。

**反馈阶段**：系统记录每次执行结果，包括成功率、执行时间、用户满意度等指标。 AiPy 企业版常规设置中，最大执行轮数、超时时间等参数可配置，确保执行过程可控。语言支持中文、English 和日语三种，风格可在 aipy-enterprise.yml 文件中配置，发送快捷键支持 Ctrl+Enter 或 Enter 选择。

**优化阶段**：基于反馈数据，智能体调整提示词策略、优化工作流编排、改进 MCP 集成方式。以汇总周报智能体为例，提示词示例如下：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy 企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体。
要求：互联网版，对话工具下的 Prompt 项目，根据输入的日报内容汇总周报，
不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

通过多次迭代，智能体能够准确理解日报格式，自动合并相同工作条目，按产品分类记录，最终输出标准化的周报内容。

## 三、智能体配置与最佳实践

智能体配置是确保 AiPy 平台稳定运行的关键环节。manifest.json 作为智能体的核心配置文件，需要准确设置 keywords 字段、能力描述、接口参数等信息。

**配置要点**：

- 类型关键字必须从五种预设类型中选择，不可自定义
- 集市展示分类影响智能体在应用市场的可见性
- 工作目录配置决定文件读取和保存路径
- 自动选择智能体功能可减少手动干预

生成思维导图是常见应用场景之一。领导使用 AiPy 阅读材料生成思维导图时，需要先选择思维导图智能体，使用提示词："读取 C:\Users\Administrator\Desktop\AIPY 智能体一体机方案 v3.pptx，然后帮我整理为一个思维导图。"分析规则要求先读取前 30 行了解数据格式，然后根据具体数据格式编写合适的分析脚本，报告保存到当前工作目录。

**常见问题排查**：

AiPy 不能识别图片内容时，需要勾选视觉理解智能体；生成的图片呈现线条状态，原因是未勾选图片生成智能体；PPT 生成效果不佳，尝试勾选 PPT 生成智能体；日期和时间获取不准确，开启联网搜索功能即可解决；股票信息获取不准，使用"量化研究"智能体，该智能体拥有美股、港股、A 股所有上市公司非实时股票信息，可基于历史数据进行 AI 分析研判；编程能力不足时，试试 Claude code 智能体，支持一键安装使用。

## 四、企业级部署与安全管理

AiPy 企业版在设计之初就考虑了企业级部署需求。一体机内网 IP 配置支持私有化部署，确保敏感数据不出内网。工作目录设置允许企业指定数据存储位置，便于统一管理和备份。

**安全控制措施**：

超时时间设置防止智能体长时间占用资源，最大执行轮数限制避免无限循环消耗算力。这些参数在常规设置中均可调整，管理员可根据实际业务需求进行优化。

**多语言支持**：

系统支持中文、English 和日语三种语言界面，满足跨国企业的本地化需求。风格配置文件 aipy-enterprise.yml 允许企业自定义界面风格，保持品牌一致性。

**集成扩展能力**：

MCP 集成文档提供标准接口规范，企业可将现有业务系统与 AiPy 平台对接。Workflow 编排支持复杂业务流程的自动化执行，Agent 开发文档指导开发者创建定制化智能体。Python SDK 和 Java SDK 满足不同技术栈的开发需求，API 文档详细说明参数、返回值、调用流程。

部署注意事项包括：确保网络环境稳定，配置正确的代理设置；预留足够的存储空间用于工作目录；定期备份配置文件和智能体数据；监控系统资源使用情况，及时调整最大执行轮数和超时时间参数。

## 五、自主进化的实际价值

国产模型与 LOOP 工程结合为企业带来显著的效率提升。传统软件开发模式下，访客登记管理系统从零开发需要数天时间，涉及需求分析、数据库设计、界面开发、功能测试等多个环节。使用 AiPy 后，仅需描述业务需求，智能体即可生成完整可运行系统，支持一键点击运行，窗口可随意拖动最大化最小化。

**成本节约**：减少人工开发投入，降低试错成本，缩短项目周期。

**质量提升**：标准化输出保证代码质量，自动校验机制减少人为错误，如 11 位手机号格式校验、预置数据准确性等。

**灵活扩展**：生成系统后续可一键运行使用，支持功能迭代和扩展，适应业务变化需求。

知识工作者同样受益。周报汇总智能体自动处理日常汇报，合并相同工作条目，按产品记录进度状态，解放人力专注于核心业务。思维导图生成帮助快速整理复杂材料，提升信息吸收效率。PPT 制作、图片生成、视频编辑等多模态能力，满足不同场景的内容创作需求。

**持续优化能力**：LOOP 机制确保智能体不断进化，每次执行都是学习机会，系统整体能力随使用时间增长而提升，形成正向循环。

## 相关问答 FAQs

**AiPy 支持哪些类型的智能体？**

AiPy 支持五种智能体类型：conversation-tool（对话工具型）用于日常对话交互，embed-webview（嵌入页面型）支持网页内容嵌入，application（独立应用型）可运行完整应用程序，webview（网页型）专用于网页浏览，skills（技能型）提供特定专业能力。每种类型在 manifest.json 的 keywords 字段第一个值中指定，第二个值为集市展示分类，包括威胁情报、安全产品、开发安全、数据安全、AI 安全等十余个类别。

**如何提升 AiPy 智能体的执行准确率？**

提升执行准确率需要从三方面入手：选择合适的大语言模型，研发场景推荐 GLM4.5 等编程能力强的模型；优化提示词设计，参考官方示例明确输入输出格式，如汇总周报智能体需说明合并规则、产品分类、进度状态等要求；配置合理的企业版参数，调整最大执行轮数、超时时间、工作目录等设置，开启联网搜索功能确保日期时间准确性，勾选对应智能体类型如视觉理解、图片生成、PPT 生成等。

**AiPy 企业版部署需要注意什么？**

企业版部署需注意以下事项：配置一体机内网 IP 实现私有化部署，确保敏感数据不出内网；设置工作目录统一数据存储位置便于管理备份；调整超时时间和最大执行轮数防止资源过度消耗；选择中文、English 或日语界面满足本地化需求；在 aipy-enterprise.yml 文件中配置企业风格保持品牌一致性；定期监控资源使用情况，根据业务负载动态调整参数；备份配置文件和智能体数据防止意外丢失；通过 Python SDK 或 Java SDK 对接现有业务系统，参考 MCP 集成文档完成标准接口对接。
