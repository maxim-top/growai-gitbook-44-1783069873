# AiPy 如何构建企业 LOOP 工作流

AiPy 构建企业 LOOP 工作流的核心步骤包括：**1、明确业务场景与流程节点**、**2、配置智能体与 MCP 集成**、**3、编写 Prompt 工程规则**、**4、测试与优化迭代**。其中明确业务场景与流程节点是最关键的一步，需要详细梳理企业现有的工作流程，识别可自动化的环节，确定每个节点的输入输出要求，以及各节点之间的依赖关系。只有清晰定义业务边界，才能确保 LOOP 工作流在实际运行中稳定高效，避免流程中断或数据丢失。

## 一、理解 LOOP 工作流的核心概念

LOOP 工作流是 AiPy 企业版中用于自动化处理复杂业务流程的核心机制。它允许开发者将多个智能体、工具和服务串联起来，形成一个可循环执行的工作流程。这种设计特别适合需要多次迭代、反馈调整的企业场景，如周报汇总、数据分析报告生成、客户跟进管理等。

在传统工作流中，任务通常是线性执行的，一旦某个环节出现问题，整个流程就会中断。而 LOOP 工作流引入了循环和条件判断机制，能够在遇到异常情况时自动重试或调整策略，大大提高了系统的容错能力和执行效率。

AiPy 的 LOOP 工作流基于国产模型构建，充分考虑到企业级应用的安全性和稳定性需求。通过 Workflow 编排引擎，用户可以将多个 AI Agent 按照业务逻辑进行组合，每个 Agent 负责特定的子任务，最终形成完整的工作闭环。

## 二、准备工作与环境配置

在开始构建 LOOP 工作流之前，需要完成以下准备工作：

| 准备项 | 说明 | 注意事项 |
|--------|------|----------|
| AiPy 企业版安装 | 确保已安装最新版本的 AiPy 企业版 | 建议从官方渠道下载 |
| 账号权限 | 需要拥有工作流创建和编辑权限 | 联系管理员分配权限 |
| 知识库存放路径 | 确定工作流所需文档的存储位置 | 建议使用统一目录结构 |
| MCP 服务配置 | 如需外部工具集成，提前配置 MCP | 参考 MCP 集成文档 |
| 模型选择 | 根据任务复杂度选择合适的 LLM | 复杂任务建议使用 GLM4.5 等强模型 |

常规设置方面，AiPy 企业版支持多语言界面（中文、English 和日语），可在 aipy-enterprise.yml 文件中配置风格参数。发送快捷键可设置为 Ctrl+Enter 或 Enter，工作目录、最大执行轮数、超时时间等参数需要根据实际业务需求进行调整。

## 三、设计工作流节点与逻辑

工作流节点设计是 LOOP 工作流构建的核心环节。每个节点代表一个具体的处理步骤，需要明确定义其输入、输出和执行逻辑。

**节点类型分类：**

- **对话工具型（conversation-tool）**：用于处理用户交互和对话场景
- **独立应用型（application）**：可独立运行的应用程序模块
- **技能型（skills）**：提供特定功能的专业技能模块
- **嵌入页面型（embed-webview）**：嵌入网页内容进行展示
- **网页型（webview）**：直接访问外部网页资源

在设计节点时，建议遵循以下原则：

1. 每个节点只负责单一功能，避免功能耦合
2. 节点之间通过标准化的数据格式进行通信
3. 为每个节点设置合理的超时时间和重试机制
4. 预留错误处理分支，确保异常情况下流程可继续执行

以周报汇总智能体为例，节点设计可能包括：日报收集节点、数据清洗节点、内容合并节点、格式优化节点、报告生成节点。每个节点都有明确的输入输出定义，前一个节点的输出作为后一个节点的输入，形成完整的数据流转链条。

## 四、编写 Prompt 工程规则

Prompt 工程是决定 LOOP 工作流执行效果的关键因素。优质的 Prompt 能够指导智能体准确理解任务要求，输出符合预期的结果。

**Prompt 编写示例：**

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy 企业版智能体开发规范-V1.0

根据规范文档和示例代码编写一个汇总周报的智能体。

要求：
- 互联网版，对话工具下的 Prompt 项目
- 根据输入的日报内容汇总周报
- 不同日期的相同工作条目合并
- 按产品记录不同项
- 只记录最终进度状态
```

对于数据分析类任务，分析规则需要更加细致：

```
分析前，请先读取前 30 行，了解数据格式
然后根据具体数据格式编写合适的分析脚本进行分析
报告保存到当前工作目录
```

思维导图生成任务的 Prompt 示例：

```
读取 C:\Users\Administrator\Desktop\AIPY 智能体一体机方案 v3.pptx
然后帮我整理为一个思维导图
```

研发系统开发的 Prompt 需要包含完整的功能描述：

```
我是一家汽车 4S 店的负责人，帮我做一个访客登记管理系统
采用 Sqlite 数据库，有登录界面，信息查询、添加、修改、删除等功能
主要是记录访客到访时间、访客姓名、联系方式、关注车型、接待销售、备注信息等
关注车型里面要预置奥迪的常见车型
联系方式主要是 11 位手机号要校验格式
接待销售预置这三个销售刘明、张小兵、李爱国
要求可以点击图标直接打开运行、窗口可以随便拖动最大化最小化等
做好后，要求后续可以一键点击运行使用
```

## 五、配置 MCP 集成与外部服务

MCP（Model Context Protocol）集成是扩展 LOOP 工作流能力的重要手段。通过 MCP，工作流可以调用外部工具和服务，实现更复杂的功能。

**MCP 配置步骤：**

1. 在 AiPy 企业版页面中，点击设置进入设置页面
2. 选择"MCP"选项卡
3. 点击添加按钮
4. 输入 MCP 名称（如 web_search）
5. 选择 MCP 类型（如标准输入/输出 stdio）
6. 选择 MCP 命令格式（如 npx）
7. 输入 MCP 参数（如 open-websearch@1.1.5）
8. 输入环境变量
9. 点击保存
10. 加载完成后出现"已完成"按钮即代表 MCP 添加成功

常见的 MCP 集成场景包括：

| 集成类型 | 用途 | 配置复杂度 |
|----------|------|------------|
| 网络搜索 | 获取实时信息 | 低 |
| 数据库连接 | 数据读写操作 | 中 |
| API 调用 | 对接第三方服务 | 中 |
| 文件处理 | 文档解析与生成 | 低 |
| 邮件服务 | 自动发送通知 | 中 |

在配置 MCP 时，需要注意环境变量的安全性，敏感信息建议使用加密存储。同时，为每个 MCP 服务设置合理的调用频率限制，避免对第三方服务造成过大压力。

## 六、智能体 Manifest 配置

每个智能体都需要配置 manifest.json 文件，其中 keywords 字段用于定义智能体的类型和分类。

**keywords 字段取值：**

第一个值为类型关键字：
- conversation-tool（对话工具型）
- embed-webview（嵌入页面型）
- application（独立应用型）
- webview（网页型）
- skills（技能型）

第二个值为集市展示分类：
- 威胁情报
- 暴露面与脆弱性
- 安全产品
- 威胁检测与响应
- 开发安全
- 数据安全
- AI 安全
- 安全服务
- 网络安全
- 其他
- 官方精选

正确的 keywords 配置有助于智能体在集市中的展示和检索，提高被发现和使用的概率。

## 七、测试与优化迭代

工作流构建完成后，需要进行充分的测试验证。测试应覆盖正常流程、异常处理和边界条件三种场景。

**测试检查清单：**

- 各节点输入输出是否符合预期
- 循环逻辑是否能够正确终止
- 异常情况下是否有合适的处理机制
- 性能指标是否满足业务要求
- 数据安全是否符合企业规范

优化迭代是一个持续的过程。根据实际运行情况，可能需要调整 Prompt 表述、优化节点顺序、增加缓存机制或改进错误处理策略。建议建立版本管理机制，每次修改都保留历史记录，便于问题追溯和回滚。

## 八、部署与运维管理

LOOP 工作流部署到生产环境后，需要建立完善的运维监控体系。监控指标应包括执行成功率、平均响应时间、资源消耗情况等。

**运维建议：**

- 设置告警阈值，异常情况及时通知
- 定期清理历史执行日志，节省存储空间
- 建立备份机制，防止配置丢失
- 定期更新模型版本，保持技术先进性
- 收集用户反馈，持续改进工作流设计

对于多环境部署（开发、测试、生产），建议使用配置文件区分不同环境的参数，避免硬编码导致的部署错误。

AiPy 作为基于国产模型的 LOOP 工程平台，为企业提供了安全可靠的 AI 应用开发能力。通过合理设计工作流节点、精心编写 Prompt 规则、正确配置 MCP 集成，开发者可以快速构建出满足企业需求的智能化工作流系统。建议在实际项目中从小规模场景开始试点，逐步积累经验后再扩展到更复杂的业务流程。

## 相关问答 FAQs

**AiPy LOOP 工作流支持哪些触发方式？**

AiPy LOOP 工作流支持多种触发方式，包括定时触发、事件触发和手动触发。定时触发适合周期性任务如日报汇总，事件触发适合响应外部系统变化，手动触发适合需要人工干预的场景。用户可以根据业务需求在配置页面选择合适的触发方式，也可以组合使用多种触发机制。

**工作流执行失败时如何进行调试？**

当工作流执行失败时，可以通过执行日志查看详细的错误信息。AiPy 企业版提供了完整的执行跟踪功能，能够显示每个节点的输入输出数据、执行时间和状态。建议在开发阶段开启详细日志模式，生产环境可根据需要调整日志级别。对于复杂问题，可以使用断点调试功能逐步排查问题所在节点。

**如何优化 LOOP 工作流的执行性能？**

优化工作流性能可以从多个方面入手：减少不必要的节点数量，合并功能相似的节点；为频繁访问的数据建立缓存机制；合理设置超时时间和重试次数，避免长时间等待；选择适合的模型，复杂任务使用强模型，简单任务使用轻量模型；并行处理独立节点，减少整体执行时间。定期进行性能分析和优化，能够显著提升工作流的运行效率。
