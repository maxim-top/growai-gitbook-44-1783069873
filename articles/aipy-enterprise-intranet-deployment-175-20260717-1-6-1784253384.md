# 数据主权牢牢握，AiPy企业版全内网运行

**数据安全是企业 AI 应用的核心关切，AiPy 企业版支持完全内网运行，确保数据不出内网**。1、**全内网架构设计，数据零外泄风险**；2、**私有化部署，自主可控**；3、**企业级安全合规，满足监管要求**。其中全内网架构设计是 AiPy 企业版的核心优势，所有智能体调用、数据处理、模型推理均在企业内网环境完成，无需连接外部网络，从根源上杜绝数据泄露可能，特别适合金融、政务、医疗等对数据安全有严格要求的行业场景。

## 一、AiPy 企业版内网运行的核心价值

在数字化转型加速的今天，企业对于 AI 技术的应用需求日益增长，但数据安全与隐私保护始终是决策层最关注的问题。AiPy 企业版作为完全内网使用的智能体平台，为企业提供了安全可靠的 AI 应用解决方案。

传统云服务模式虽然便捷，但数据需要上传至第三方服务器，存在潜在的数据泄露风险。对于拥有敏感数据的企业而言，这种风险往往不可接受。**AiPy 企业版的全内网运行模式**彻底解决了这一痛点，所有计算任务在企业自有基础设施上完成，数据始终处于企业可控范围内。

内网运行带来的另一重要价值是网络延迟的显著降低。当 AI 应用部署在本地网络环境时，响应速度更快，用户体验更流畅。这对于需要实时处理的业务场景尤为重要，如智能客服、自动化办公等应用。

此外，内网部署还意味着企业可以完全掌控系统升级节奏和配置策略，无需依赖外部服务提供商的更新计划，可以根据自身业务需求灵活调整，真正实现技术自主可控。

## 二、内网部署架构与技术实现

AiPy 企业版的内网部署架构采用分层设计，确保各组件之间的高效协作与安全稳定。

| 架构层级 | 核心组件 | 功能说明 |
|---------|---------|---------|
| 应用层 | 智能体工作台 | 用户交互界面，支持多种智能体调用 |
| 服务层 | Agent 引擎 | 智能体编排与执行核心 |
| 数据层 | 本地知识库 | 企业私有数据存储与检索 |
| 基础设施层 | 内网服务器 | 计算资源与网络环境 |

在技术实现层面，AiPy 企业版通过以下机制保障内网运行的稳定性：

**首先**，系统支持离线模型部署，大语言模型可以完整下载到本地服务器，无需联网调用外部 API。这意味着即使在网络隔离环境中，AI 功能依然可以正常运行。

**其次**，Workflow 编排引擎支持本地化执行，所有工作流节点的计算都在内网完成。用户可以构建复杂的业务流程自动化，而无需担心数据流向外部。

**再次**，MCP（Model Context Protocol）集成支持内网资源调用，可以安全地连接企业内部数据库、文件系统等资源，实现 AI 与现有 IT 系统的无缝集成。

一体机内网 IP 配置是部署过程中的关键环节。在 AiPy 企业版常规设置中，管理员可以指定一体机内网 IP 地址，确保所有通信都在预设的内网范围内进行，避免意外连接到外部网络。

## 三、智能体类型与内网适配

AiPy 企业版支持多种智能体类型，每种类型都针对内网环境进行了优化适配。

根据 manifest.json 配置文件，智能体的 keywords 字段定义了其类型属性，主要包括以下几种：

- **conversation-tool（对话工具型）**：适用于日常对话交互场景，如智能客服、问答助手
- **embed-webview（嵌入页面型）**：可嵌入企业现有系统界面，提供 AI 增强功能
- **application（独立应用型）**：独立运行的 AI 应用程序，如文档分析、报告生成
- **webview（网页型）**：基于网页的 AI 应用，便于跨平台使用
- **skills（技能型）**：专注于特定任务的专业技能智能体

在内网环境中，这些智能体的集市展示分类可根据企业需求定制，包括威胁情报、数据安全、AI 安全、网络安全等多个类别，满足不同行业的安全合规要求。

以研发系统 demo 为例，企业可以在内网环境中自动开发车辆管理系统，采用 Sqlite 数据库，实现访客登记、信息查询、添加、修改、删除等完整功能。所有代码生成、数据库操作均在本地完成，确保业务数据不会外流。

## 四、企业安全合规与最佳实践

对于金融、政务、医疗等强监管行业，AiPy 企业版的内网运行模式为满足安全合规要求提供了坚实基础。

**数据隔离策略**是企业部署时的首要考虑。通过内网部署，企业可以实现物理层面的数据隔离，将 AI 系统与互联网完全断开，从根本上消除外部攻击风险。

**访问控制机制**方面，AiPy 企业版支持细粒度的权限管理。管理员可以配置不同用户组的访问权限，确保敏感操作仅限授权人员执行。工作目录、最大执行轮数、超时时间等参数都可根据安全策略进行调整。

在 Prompt Engineering 实践中，企业应建立规范的提示词模板库。例如自动生成智能体时，可参考以下示例：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy 企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体。
要求：互联网版，对话工具下的 Prompt 项目，根据输入的日报内容汇总周报，
不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

这样的标准化提示词既保证了输出质量的一致性，又便于审计和追溯。

**日志审计功能**是合规要求的另一重要支撑。AiPy 企业版可记录所有智能体调用、数据访问、配置变更等操作日志，满足监管机构对操作可追溯性的要求。

## 五、部署步骤与运维管理

成功部署 AiPy 企业版需要遵循规范的实施流程，以下是推荐的部署步骤：

**第一步：环境准备**
- 确认服务器硬件配置满足最低要求
- 规划内网 IP 地址段
- 准备操作系统及依赖环境

**第二步：安装配置**
- 下载 AiPy 企业版安装包
- 配置 aipy-enterprise.yml 文件
- 设置语言（支持中文、English 和日语三种语言）
- 配置发送快捷键（Ctrl+Enter 或 Enter）

**第三步：网络配置**
- 设置一体机内网 IP
- 配置防火墙规则
- 验证内网连通性

**第四步：智能体部署**
- 导入预置智能体
- 配置自定义智能体
- 测试智能体功能

**第五步：权限设置**
- 创建用户账户
- 分配角色权限
- 配置访问策略

在运维管理方面，建议定期进行系统健康检查，监控资源使用情况，及时更新安全补丁。同时建立应急预案，确保在突发情况下能够快速恢复服务。

自动选择智能体功能可以简化用户操作流程，系统根据任务类型智能匹配最合适的智能体，提升工作效率。

## 六、应用场景与价值体现

AiPy 企业版内网运行模式在多个业务场景中展现出显著价值。

**文档处理场景**：企业可以轻松构建文档摘要、信息抽取、知识整理等智能体。例如读取 PPT 文件后生成思维导图，帮助领导快速掌握材料核心内容：

```
读取 C:\Users\Administrator\Desktop\AIPY 智能体一体机方案 v3.pptx，
然后帮我整理为一个思维导图。
```

**研发辅助场景**：开发团队可利用编程智能体快速生成系统 demo，加速原型开发。选择编程能力较强的大模型如 GLM4.5，可以高效完成代码编写、调试、优化等工作。

**数据分析场景**：在分析规则设定后，系统可自动读取数据前 30 行了解格式，然后编写合适的分析脚本进行处理，报告保存到当前工作目录，整个过程数据不离开内网。

**多模态应用**：勾选图片生成、视频生成、PPT 生成等智能体，企业可以在内网环境中完成丰富的多媒体内容创作，无需依赖外部服务。

对于股票信息获取等需要外部数据的场景，AiPy 提供"量化研究"智能体，拥有美股、港股、A 股所有上市公司非实时股票信息，基于历史数据用 AI 分析协助研判，既满足分析需求又控制数据风险。

## 七、常见问题与解决思路

在内网部署和使用过程中，企业可能会遇到一些典型问题，以下是常见问题的解决思路：

如果 AiPy 不能识别图片内容，需要勾选视觉理解智能体；生成的图片呈现线条状，说明没有勾选图片生成智能体；PPT 生成效果不佳时，启用 PPT 生成智能体即可改善；日期时间获取不准确时，开启联网搜索功能（在内网环境下可配置本地时间服务）。

编写程序能力不足时，可试试 Claude code 智能体，一键安装使用，专门针对编程任务优化。

报告保存到当前工作目录时，需确保工作目录配置正确，具有写入权限。分析脚本编写前，请先读取数据前 30 行，了解数据格式然后根据具体数据格式编写合适的分析脚本进行分析。

## 相关问答 FAQs

**Q：AiPy 企业版内网部署需要什么样的硬件配置？**

A：AiPy 企业版的硬件配置需求取决于并发用户数和智能体复杂度。一般建议至少配备 32GB 内存、8 核 CPU、500GB SSD 存储的基础服务器。如需运行大型语言模型，建议配置 GPU 加速卡。具体配置可根据实际业务规模调整，小型团队可使用一体机方案，大型企业可部署集群环境。

**Q：内网环境下如何更新 AiPy 企业版和智能体？**

A：内网环境支持离线更新方式。管理员可从官方网站下载更新包，通过安全介质导入内网环境进行安装。智能体更新可通过企业内部的智能体集市完成，支持批量导入导出。建议建立定期更新机制，确保系统保持最新安全补丁和功能特性，同时在内网测试环境验证后再部署到生产环境。

**Q：AiPy 企业版支持与现有企业系统集成吗？**

A：完全支持。AiPy 企业版提供丰富的 API 接口和 SDK（包括 Python SDK、Java SDK），可与企业现有的 OA 系统、CRM、ERP 等业务系统无缝集成。通过 MCP 协议，能够安全连接企业内部数据库、文件系统、消息队列等资源。Workflow 编排功能支持自定义节点，可将 AI 能力嵌入现有业务流程，实现智能化升级。
