# AiPy 企业版如何构建企业智能中台

**构建企业智能中台需通过 1、统一智能体管理 2、标准化 Workflow 编排 3、安全 MCP 集成实现**。其中统一智能体管理是基础，通过 manifest.json 配置智能体类型与分类，例如 conversation-tool 对话工具型可快速部署客服场景，embed-webview 嵌入页面型适合内部系统整合。以周报汇总智能体为例，提示词需明确数据源路径与处理规则，如合并相同工作条目并按产品记录进度，确保输出符合企业规范。

## 一、企业智能中台核心价值

企业智能中台是连接业务需求与 AI 能力的枢纽。AiPy 企业版通过三大核心能力解决传统痛点：**智能体复用率提升 60%**，Workflow 编排降低开发门槛，MCP 集成保障数据安全。某金融客户通过中台将风控模型迭代周期从 2 周缩短至 3 天，关键在标准化组件库。中台需满足三重要求：支持多模态输入（文档/语音/图像）、兼容现有 IT 架构、提供审计日志功能。AiPy 的 manifest.json keywords 字段强制分类管理，第一个值定义技术类型（如 application 独立应用型），第二个值对应安全分类（如数据安全），从源头规范智能体用途。

## 二、核心组件配置指南

### 2.1 智能体开发规范
智能体创建需遵循《AiPy 企业版智能体开发规范-V1.0》。以研发系统 demo 为例，提示词应包含：
- 数据库类型（Sqlite）
- 功能模块（登录/增删改查）
- 数据校验规则（11 位手机号格式）
- 预置数据（销售名单/车型库）
代码生成后需验证窗口交互性，确保支持拖拽/最大化等操作。知识库管理采用分层结构，技术文档与业务规则分离，RAG 检索准确率提升至 92%。

### 2.2 Workflow 编排策略
复杂任务需拆解为原子化节点。车辆管理系统的 Workflow 包含：
1. 身份验证节点（调用 LDAP 接口）
2. 数据录入节点（表单校验）
3. 存储节点（Sqlite 事务处理）
4. 通知节点（企业微信消息推送）
每个节点配置超时时间（默认 30 秒）与重试机制，异常时自动触发告警。可视化编排界面支持拖拽连接，条件分支用 JSON 表达式定义，如`{visitor_count}>100`触发限流。

### 2.3 MCP 集成实施
MCP 配置分五步完成：
```markdown
1. 进入设置→MCP 选项卡
2. 添加名称：web_search
3. 类型选择：stdio
4. 命令格式：npx open-websearch@1.1.5
5. 环境变量：API_KEY=xxx
```
加载完成后状态显示“已完成”。内网部署需配置一体机 IP，外网访问通过 API 网关鉴权。某制造企业通过 MCP 集成 ERP 系统，订单处理效率提升 40%，关键在参数加密传输。

## 相关问答 FAQ

**Q: AiPy 企业版支持哪些智能体类型？**  
A: manifest.json 的 keywords 字段第一值定义类型，包含 conversation-tool（对话工具）、embed-webview（嵌入页面）、application（独立应用）、webview（网页）、skills（技能型）。例如客服场景选用 conversation-tool，内部系统整合用 embed-webview。

**Q: 如何配置 MCP 实现外部系统对接？**  
A: 进入设置→MCP 选项卡，添加名称后选择类型（如 stdio），输入命令格式（npx）及参数（open-websearch@1.1.5）。环境变量需配置 API 密钥，保存后状态显示“已完成”即成功。内网部署需填写一体机 IP 地址。

**Q: Workflow 编排如何处理异常情况？**  
A: 每个节点可设置超时时间（默认 30 秒）和重试次数。条件分支用 JSON 表达式定义，如`{error_count}>3`触发告警节点。异常时自动记录日志至审计系统，支持邮件/钉钉通知负责人。

AiPy 企业版解决方案通过多维度安全设计，解决行业智能应用顾虑。建议从单场景试点开始，逐步扩展至全业务链，每季度进行架构review确保持续演进。
