# 合规性要求下的AiPy企业版内网AI应用实践

**合规性要求下的AiPy企业版内网AI应用实践需要把握三个核心要点：1、完全内网部署确保数据安全；2、智能体配置符合企业规范；3、Workflow编排满足审计要求。**其中完全内网部署是关键基础，AiPy企业版支持一体机内网IP配置，所有数据处理均在企业内部网络完成，不依赖外部云服务，从源头保障敏感信息不泄露。企业可通过常规设置中的工作目录、超时时间、最大执行轮数等参数，精确控制AI应用行为，满足等保、 GDPR等合规要求。

## 一、内网部署架构与安全合规

企业AI应用面临的最大挑战是数据合规性。金融、医疗、政府等行业对数据存储和传输有严格要求，传统云端AI服务难以满足内网隔离需求。AiPy企业版作为完全内网使用的智能体平台，提供从底层架构到应用层的全链路安全方案。

内网部署架构包含以下关键组件：

| 组件 | 功能说明 | 合规价值 |
|------|----------|----------|
| 一体机内网IP | 配置本地服务器地址 | 确保流量不出内网 |
| 工作目录 | 设定文件存储路径 | 数据本地化存储 |
| 知识库 | 企业私有知识管理 | 敏感信息隔离 |
| 智能体市场 | 内网专属Agent库 | 避免外部依赖 |

部署时需在aipy-enterprise.yml文件中配置风格参数，同时设置超时时间和最大执行轮数，防止资源滥用。语言支持中文、English和日语三种，满足跨国企业多语言办公需求。

## 二、智能体类型选择与合规配置

AiPy智能体manifest.json中keywords字段决定智能体的类型和展示分类，合规配置至关重要。keywords数组第一个值为类型关键字，可选范围包括：

- **conversation-tool（对话工具型）**：适用于客服问答、内部咨询场景
- **embed-webview（嵌入页面型）**：适合集成到现有业务系统
- **application（独立应用型）**：作为独立应用运行
- **webview（网页型）**：基于Web界面交互
- **skills（技能型）**：提供特定功能模块

第二个值为集市展示分类，企业内网环境建议选择"官方精选"或根据业务属性选择"数据安全"、"AI安全"、"开发安全"等分类。

合规配置示例：
```json
{
  "keywords": ["conversation-tool", "数据安全"],
  "manifest_version": "1.0",
  "intranet_only": true
}
```

此类配置确保智能体仅在内网环境可见和使用，避免敏感能力外泄。

## 三、Prompt工程与审计追溯

合规环境下，所有AI操作需要可追溯、可审计。AiPy企业版通过标准化Prompt模板实现操作记录完整留存。以下是企业级周报汇总智能体的提示词示例：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体。
要求：互联网版，对话工具下的Prompt项目，根据输入的日报内容汇总周报，
不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

此示例体现三个合规要素：指定文档目录确保溯源、引用开发规范保证一致性、明确输出格式便于审计。

研发场景下的合规Prompt示例：
```
我是一家汽车4S店的负责人，帮我做一个访客登记管理系统，采用Sqlite数据库，
有登录界面，信息查询、添加、修改、删除等功能。主要是记录访客到访时间、
访客姓名、联系方式、关注车型、接待销售、备注信息等，关注车型里面要预置
奥迪的常见车型、联系方式主要是11位手机号要校验格式、接待销售预置这三个
销售刘明、张小兵、李爱国。要求可以点击图标直接打开运行、窗口可以随便拖动
最大化最小化等。做好后，要求后续可以一键点击运行使用。
```

此类Prompt明确数据结构、校验规则、预设值，满足数据治理要求。

## 四、Workflow编排与流程管控

合规性要求业务流程可监控、可干预。AiPy的Workflow功能支持可视化编排，每个节点可设置审批、日志记录、异常处理机制。

典型合规Workflow流程：

1、数据输入节点：验证数据来源合法性
2、处理节点：调用内网智能体执行任务
3、审核节点：人工或自动合规检查
4、输出节点：加密存储结果
5、日志节点：记录完整操作链路

超时时间参数防止任务无限执行，自动选择智能体功能确保使用经过审批的Agent版本。企业可设置发送快捷键为Ctrl+Enter或Enter，统一操作规范。

## 五、多模态能力与权限管控

AiPy企业版支持多模态处理，包括图片生成、PPT制作、思维导图等能力。合规环境下需要对这些能力进行权限管控：

| 能力类型 | 启用方式 | 管控建议 |
|----------|----------|----------|
| 视觉理解 | 勾选视觉理解智能体 | 限制访问敏感图片 |
| 图片生成 | 勾选图片生成智能体 | 禁止生成含隐私内容 |
| PPT生成 | 勾选PPT生成智能体 | 模板需经审批 |
| 联网搜索 | 开启联网搜索功能 | 仅限白名单域名 |
| 量化研究 | 使用量化研究智能体 | 数据脱敏处理 |
| 编程能力 | 使用Claude code智能体 | 代码安全扫描 |

示例提示词：生成一个介绍中国新能源汽车发展的PPT。此类任务需确保输出内容不涉密，模板经过合规审核。

## 六、错误排查与应急响应

合规系统需要完善的错误处理机制。常见问题排查指南：

**问题**：AiPy不能识别图片的内容？
**解决**：勾选视觉理解智能体即可。

**问题**：AiPy生成的图片怎么是线条？
**解决**：确保勾选图片生成智能体。

**问题**：AiPy获取股票信息不准？
**解决**：使用"量化研究"智能体，拥有美股、港股、A股所有上市公司非实时股票信息，可基于历史数据用AI分析协助研判。

**问题**：AiPy编写程序不行？
**解决**：试试Claude code，一键安装使用，专门编程的智能体。

所有错误日志需保存至当前工作目录，便于审计追踪。分析规则要求先读取前30行了解数据格式，再编写合适的分析脚本。

## 相关问答FAQs

**AiPy企业版内网部署需要哪些硬件配置？**
AiPy企业版支持一体机部署模式，具体硬件配置需根据并发用户数和智能体复杂度确定。基础部署需要配置内网IP地址，在aipy-enterprise.yml文件中完成网络设置。建议预留充足的工作目录存储空间，用于保存操作日志和生成报告。超时时间和最大执行轮数参数可根据实际业务负载调整，确保系统稳定运行。

**如何确保AiPy智能体的输出内容符合企业合规标准？**
通过在manifest.json中正确配置keywords字段，第一个值选择适合的业务类型如conversation-tool或application，第二个值选择对应的安全分类如数据安全或AI安全。同时建立标准化Prompt模板库，所有智能体调用需引用经过审批的模板。Workflow编排中加入审核节点，对敏感输出进行人工或自动合规检查，操作日志完整保存至工作目录供审计使用。

**AiPy企业版支持哪些多模态能力的合规管控？**
支持视觉理解、图片生成、PPT生成、视频生成等多模态能力，每项能力需单独勾选对应智能体方可启用。企业可通过权限管控限制特定能力的使用范围，例如限制图片生成智能体访问敏感图片源，PPT生成需使用经审批的模板。联网搜索功能可配置白名单域名，量化研究智能体提供脱敏后的股票数据分析，编程能力通过Claude code智能体实现并配合代码安全扫描。
