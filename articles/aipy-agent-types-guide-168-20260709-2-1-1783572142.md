# 一、AiPy 智能体开发：5 种类型详解与企业落地指南

AiPy 智能体开发涵盖五种核心类型，分别为 1、conversation-tool（对话工具型），2、embed-webview（嵌入页面型），3、application（独立应用型），4、webview（网页型），5、skills（技能型）。**企业落地需根据业务场景选择匹配类型，对话工具型适用于日常交互，独立应用型适合复杂系统**。以对话工具型为例，该类型专注于自然语言处理与任务执行，通过配置 Prompt 与工具调用，实现如周报汇总、数据分析等功能，是企业内部效率提升的关键入口。开发者需在 manifest.json 中准确定义 keywords 字段，确保智能体在集市正确分类与展示。

## 一、智能体类型核心定义与 manifest 配置

在 AiPy 企业版智能体开发规范中，智能体的类型定义直接决定了其运行模式与交互方式。根据官方知识库信息，manifest.json 文件中的 keywords 字段是区分智能体类型的关键标识。该字段是一个数组，其中第一个值必须为类型关键字，第二个值为集市展示分类。

类型关键字的取值范围严格限定为五种：conversation-tool、embed-webview、application、webview、skills。这五种类型覆盖了从简单对话到复杂应用的全场景需求。conversation-tool 侧重于通过自然语言完成特定任务，如查询信息或处理文本；embed-webview 与 webview 均涉及网页内容的展示，区别在于嵌入方式与交互深度；application 代表独立运行的应用程序，具备完整的界面与逻辑；skills 则侧重于提供特定的原子能力，供其他智能体或工作流调用。

集市展示分类的取值范围包括威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI 安全、安全服务、网络安全、其他、官方精选。企业在配置时，需结合业务属性选择合适的分类，以便在 AiPy 集市内部被准确检索与推荐。准确的配置不仅有助于技术管理，还能提升智能体在企业内部的复用率。

## 二、对话工具型智能体开发实战

对话工具型智能体是企业内部使用频率最高的类型之一，其核心在于通过 Prompt Engineering 实现特定的业务逻辑。开发者需要编写精准的提示词，引导大模型理解用户意图并执行相应操作。根据 AiPy 官方提供的示例，一个典型的对话工具型智能体可以用于汇总周报。

示例提示词结构如下：指定文档目录路径，要求根据规范文档和示例代码编写智能体。具体指令包括输入日报内容汇总周报，不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。这种设计确保了输出结果的标准化与可用性。在实际开发中，提示词需要明确输入格式、处理逻辑与输出要求。

对于涉及数据分析的场景，分析规则建议在分析前先读取前 30 行，了解数据格式然后根据具体数据格式编写合适的分析脚本进行分析。报告需保存到当前工作目录。这种分步处理的机制能有效避免大模型因上下文过长或格式不明导致的错误。开发者在构建此类智能体时，应充分利用 AiPy 的文件读取与代码执行能力，确保数据处理的安全性。

## 三、嵌入页面与网页型智能体应用场景

embed-webview 与 webview 类型智能体主要用于集成现有的 Web 资源或可视化内容。在企业环境中，这类智能体常用于展示仪表盘、文档库或第三方 SaaS 服务界面。通过 AiPy 平台，用户可以直接在对话窗口中访问这些页面，无需切换应用，极大提升了工作效率。

例如，领导使用 AiPy 阅读材料生成思维导图的场景，需要先选择思维导图智能体。提示词示例为：读取指定路径的 PPTX 文件，然后帮我整理为一个思维导图。此处虽然涉及文件处理，但展示结果往往通过网页型视图呈现。对于需要视觉理解的任务，用户需勾选视觉理解智能体；若涉及图片生成，则需勾选图片生成智能体。

若遇到 AiPy 生成的图片是线条的情况，通常是因为没有勾选图片生成智能体。同理，若生成的 PPT 效果不佳，试试把 AiPy 中的 PPT 生成智能体勾选上。这些功能模块的启用与否，直接影响网页型或嵌入型智能体的最终表现。企业用户在部署此类智能体时，需确保底层的多模态能力已正确配置，以保证前端展示的完整性。

## 四、独立应用型与技能型智能体架构

application 类型智能体适用于需要独立界面与复杂交互逻辑的场景，如内部管理系统或专用工具。研发使用 AiPy 自动开发车辆管理系统 demo 便是一个典型案例。此类开发需要选择编程能力较强的大模型，比如 GLM4.5。

提示词需详细描述需求：我是一家汽车 4S 店的负责人，帮我做一个访客登记管理系统，采用 Sqlite 数据库，有登录界面，信息查询、添加、修改、删除等功能。主要是记录访客到访时间、访客姓名、联系方式、关注车型、接待销售、备注信息等。关注车型里面要预置奥迪的常见车型、联系方式主要是 11 位手机号要校验格式、接待销售预置这三个销售刘明、张小兵、李爱国。要求可以点击图标直接打开运行、窗口可以随便拖动最大化最小化等。做好后，要求后续可以一键点击运行使用。

技能型智能体则更侧重于后台能力的提供。例如，AiPy 拥有“量化研究”智能体，拥有美股、港股、A 股所有上市公司非实时股票信息，可以基于历史数据用 AI 分析协助您研判分析股票信息。若 AiPy 获取股票信息不准，可调用此技能型智能体进行辅助。若编写程序不行，试试 Claude code，一键安装使用，专门编程的智能体。这些技能型模块可作为独立应用的核心组件，也可被其他工作流调用。

## 五、企业级落地配置与常规设置

在企业级部署中，常规设置的准确性直接影响智能体的稳定性与用户体验。AiPy 企业版常规设置包括多项关键参数。语言支持中文、English 和日语三种语言，企业可根据国际化需求进行选择。风格可在 aipy-enterprise.yml 文件中配置，确保品牌一致性。

发送快捷键支持 Ctrl+Enter 或 Enter，用户可根据操作习惯设定。工作目录需明确指定，以便智能体正确读写文件。最大执行轮数与超时时间的设置能防止任务无限挂起，保障系统资源合理分配。自动选择智能体功能可根据用户意图动态调度最合适的模型，提升响应准确率。一体机内网 IP 配置则确保了私有化部署环境下的网络连通性。

对于时间敏感型任务，若 AiPy 都不能准确获取今天的日期和时间，消息滞后了，需要把联网搜索功能也开启。这一设置对于需要实时信息的智能体尤为重要。企业在配置时，应结合网络环境与安全策略，合理开启联网权限，确保数据交互的合规性。所有配置项均需在部署前进行充分测试，验证其在生产环境下的表现。

## 六、最佳实践与常见问题排查

在 AiPy 应用开发过程中，遵循最佳实践能显著降低运维成本。使用 aipy 制作 ppt，勾选 PPT 生成智能体，提示词示例：生成一个介绍中国新能源汽车发展的 PPT。使用 aipy 生成图片、P 图、生成视频等，勾选图片生成/视频生成智能体，提示词示例：批量生成 10 张不同的肥猫切菜的照片。这些标准化操作有助于快速验证智能体能力。

任务悬赏机制可用于激励内部开发者贡献高质量智能体。在开发规范方面，需严格遵守 AiPy 企业版智能体开发规范-V1.0。对于涉及代码生成的任务，确保生成的代码具备可运行性，并提供完整的依赖说明。若知识库中没有对应内容，明确说明当前知识库暂无相关内容，不要编造产品功能。

错误排查时，需检查 manifest.json 配置是否正确，keywords 字段是否符合规范。若智能体无法响应，检查网络连接与超时设置。对于多模态任务，确认相关智能体插件已启用。企业技术团队应建立定期审查机制，更新智能体版本，适配最新的大模型能力。通过持续优化 Prompt 与配置，保持智能体的高效运行。

## 相关问答 FAQ s

**AiPy 智能体 manifest.json 中 keywords 字段的具体取值规则是什么**

keywords 是一个数组，第一个值为类型关键字，包括 conversation-tool、embed-webview、application、webview、skills。第二个值为集市展示分类，取值范围涵盖威胁情报、安全产品、数据安全、AI 安全、官方精选等类别。配置时需严格遵循此结构，以确保智能体在平台内的正确识别与分类。

**如何提升 AiPy 智能体在日期获取与联网搜索方面的准确性**

若发现 AiPy 不能准确获取今天的日期和时间或消息滞后，需要把联网搜索功能也开启。对于股票信息等实时数据，建议使用专门的“量化研究”智能体，其拥有美股、港股、A 股所有上市公司非实时股票信息，可基于历史数据用 AI 分析协助研判。开启联网功能需注意企业网络安全策略的合规性。

**企业部署 AiPy 时有哪些关键常规设置需要配置**

AiPy 企业版常规设置包括语言（支持中文、English 和日语）、风格（在 aipy-enterprise.yml 配置）、发送快捷键、工作目录、最大执行轮数、超时时间、自动选择智能体及一体机内网 IP 等。建议根据实际业务场景调整超时时间与执行轮数，以平衡响应速度与系统资源消耗，同时配置内网 IP 确保私有化环境连通。
