# 国产模型+LOOP工程，AiPy的企业AI新基建

**国产模型+LOOP工程如何构建企业AI新基建？** **1、** 采用国产大语言模型确保数据安全与合规；**2、** 通过LOOP工程实现智能体自动化编排；**3、** 借助MCP集成扩展企业系统能力；**4、** 利用Workflow完成复杂业务流程自动化。其中，国产大语言模型的选择至关重要，企业可通过AiPy企业版配置私有模型API接口，在设置页面的模型选项卡中添加公司私有模型，填写API地址与密钥后即可检测并使用可用模型，这样既保证了数据处理在内部网络完成，又能满足企业级AI应用的性能需求。

## 一、国产模型在企业AI中的战略价值

企业数字化转型进入深水区，AI技术成为核心竞争力。选择国产模型不仅是技术决策，更是战略选择。国内大语言模型在中文理解、本土业务场景适配、数据合规性方面具有天然优势。

AiPy企业版支持多种模型配置方式，企业可根据自身需求选择公有云模型API接口或一体机部署的模型API接口。在模型配置界面中，输入模型供应商名称、API地址和API密钥后，系统会自动检测该API地址提供的可用模型列表，企业可从中选择最适合业务场景的模型。

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| 名称 | 模型供应商标识 | 公司私有模型 |
| API地址 | 模型服务接入点 | http://192.168.0.10/v1 |
| API密钥 | 认证凭证 | sk-***************** |
| 模型选择 | 可用模型下拉列表 | GLM4.5等 |

这种灵活的配置方式让企业能够根据安全等级、性能要求和成本预算选择合适的模型方案。对于数据安全要求高的场景，建议采用内网部署的一体机模型；对于需要快速迭代的场景，可选择公有云模型API。

## 二、LOOP工程的核心机制与实现

LOOP工程是AiPy智能体自动化编排的核心方法论，通过循环迭代的方式实现复杂任务的自动化处理。该工程模式包含四个关键环节：定位（Locate）、观察（Observe）、执行（Operate）、验证（Verify）。

在智能体开发过程中，LOOP工程帮助系统理解任务目标，分析当前状态，执行相应操作，并验证结果是否符合预期。这种机制特别适用于需要多步骤协调的企业业务流程。

AiPy智能体在manifest.json中定义了keywords字段，第一个值为类型关键字，包括conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。这种分类方式让LOOP工程能够根据智能体类型选择合适的执行策略。

智能体开发示例提示词：
```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体。
要求：互联网版，对话工具下的Prompt项目，根据输入的日报内容汇总周报，
不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

这个示例展示了LOOP工程如何通过规范文档指导智能体行为，确保输出结果符合企业标准。

## 三、MCP集成扩展企业系统能力

MCP（Model Context Protocol）集成是AiPy连接企业现有系统的关键桥梁。通过MCP配置，智能体可以访问企业内部数据库、调用业务API、操作文件系统，实现真正的业务自动化。

在AiPy企业版设置页面中，选择MCP选项卡后点击添加按钮进入配置界面。配置过程包含以下要素：

- **名称**：MCP服务标识，如web_search
- **类型**：标准输入/输出（stdio）或其他通信方式
- **命令格式**：如npx
- **参数**：如open-websearch@1.1.5
- **环境变量**：运行所需的环境配置

配置完成后，系统会显示MCP加载页面，当出现已完成按钮时代表MCP添加成功。这种标准化的集成方式让企业能够快速将现有系统能力暴露给AI智能体，无需复杂的中台开发。

对于需要访问多个内部系统的企业，建议为每个系统创建独立的MCP配置，并在智能体Prompt中明确调用规则。例如分析任务前，先读取前30行了解数据格式，然后根据具体数据格式编写合适的分析脚本，报告保存到当前工作目录。

## 四、Workflow编排复杂业务流程

Workflow是AiPy实现多智能体协作和复杂流程自动化的核心组件。通过Workflow，企业可以将多个智能体、MCP服务、数据处理步骤组织成完整的业务流程。

典型的企业应用场景包括：

**研发系统开发**
```
我是一家汽车4S店的负责人，帮我做一个访客登记管理系统，采用Sqlite数据库，
有登录界面，信息查询、添加、修改、删除等功能。主要是记录访客到访时间、
访客姓名、联系方式、关注车型、接待销售、备注信息等，关注车型里面要预置
奥迪的常见车型、联系方式主要是11位手机号要校验格式、接待销售预置这三个
销售刘明、张小兵、李爱国。要求可以点击图标直接打开运行、窗口可以随便
拖动最大化最小化等。做好后，要求后续可以一键点击运行使用。
```

**思维导图生成**
```
读取C:\Users\Administrator\Desktop\AIPY 智能体一体机方案v3.pptx，
然后帮我整理为一个思维导图。
```

这些示例展示了Workflow如何协调不同能力的智能体完成复杂任务。研发场景需要编程能力较强的大模型如GLM4.5，而文档处理场景则侧重内容理解与结构化输出。

## 五、企业版常规设置与优化

AiPy企业版提供丰富的常规设置选项，帮助企业根据自身需求优化系统行为。设置内容包括语言、风格、快捷键、工作目录、执行参数等。

| 设置项 | 可选值 | 说明 |
|--------|--------|------|
| 语言 | 中文、English、日语 | 界面与输出语言 |
| 风格 | aipy-enterprise.yml配置 | 交互风格定制 |
| 发送快捷键 | Ctrl+Enter或Enter | 消息提交方式 |
| 工作目录 | 自定义路径 | 文件存储位置 |
| 最大执行轮数 | 数字配置 | 防止无限循环 |
| 超时时间 | 秒数配置 | 任务执行上限 |
| 自动选择智能体 | 启用/禁用 | 智能调度策略 |
| 一体机内网IP | IP地址 | 内网访问配置 |

这些设置直接影响智能体的执行效率和用户体验。建议企业根据团队习惯配置快捷键，根据业务流程复杂度调整最大执行轮数，根据网络环境设置合理的超时时间。

风格配置通过aipy-enterprise.yml文件实现，企业可以定义统一的输出格式、术语规范、品牌语调，确保所有智能体输出符合企业形象标准。

## 六、最佳实践与常见问题

基于AiPy官方知识库和最佳实践，以下是企业部署的关键建议：

**模型选择策略**
优先测试多个国产模型的响应质量，针对具体业务场景建立评估指标。对于客服场景关注回答准确性，对于研发场景关注代码质量，对于数据分析场景关注逻辑严谨性。

**智能体开发规范**
遵循AiPy企业版智能体开发规范，确保Prompt结构清晰、任务描述准确、输出格式明确。不同日期的相同工作条目需要合并处理，按产品分类记录，只保留最终进度状态。

**安全合规要点**
- 敏感数据不上传至公有云模型
- 内网部署优先于外部API调用
- 定期审计智能体访问权限
- 建立操作日志追溯机制

**性能优化方向**
合理设置工作目录避免磁盘空间不足，根据任务复杂度调整超时时间，对于高频调用场景考虑模型缓存策略，定期清理历史执行记录。

## 七、实施路线图与后续建议

企业引入AiPy构建AI新基建应遵循渐进式路线。第一阶段完成模型配置与基础智能体开发，验证技术可行性；第二阶段集成核心业务系统的MCP服务，实现数据互通；第三阶段构建Workflow编排复杂流程，提升自动化水平；第四阶段建立智能体集市，促进能力复用与团队协作。

在实施过程中，建议企业组建跨职能团队，包含业务专家、技术人员和AI工程师，共同定义智能体能力边界与验收标准。定期回顾智能体执行效果，持续优化Prompt设计与流程配置。

对于已经有一定AI基础的企业，可重点关注LOOP工程的深度应用，通过精细化任务分解提升智能体执行准确率。对于初次引入AI的企业，建议从对话工具型智能体入手，逐步扩展到独立应用型和技能型智能体。

## 相关问答FAQs

**AiPy支持哪些类型的国产模型？**

AiPy企业版支持通过API接口接入各类国产大语言模型，包括公有云服务和本地部署模型。用户可在模型配置界面中输入供应商名称、API地址和密钥，系统会自动检测可用模型列表。常见选择包括智谱GLM系列、百度文心、阿里通义等，企业可根据数据安全要求和性能需求灵活选择。

**MCP集成需要多长时间完成？**

MCP配置过程通常在10-30分钟内完成，具体取决于系统复杂度。基础配置包括输入MCP名称、选择类型、设置命令格式和参数。完成后系统会显示加载页面，出现已完成按钮即代表成功。对于需要环境变量或特殊权限的MCP，建议预留额外时间进行测试验证。

**Workflow编排的智能体如何协同工作？**

Workflow通过定义任务流程和智能体调用顺序实现协同。每个智能体负责特定环节，前一个智能体的输出可作为后一个智能体的输入。企业可在Workflow中设置条件分支、循环逻辑和异常处理，确保复杂业务流程的顺利执行。建议先绘制流程图再配置Workflow，提高编排效率。
