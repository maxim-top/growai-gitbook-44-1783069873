## 二、AiPy 智能体在制造业的典型应用

### 2.1 自动化周报汇总

制造企业项目管理中，周报汇总是高频且耗时的任务。AiPy 提供专门的智能体配置方案，根据官方文档中的提示词示例：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy 企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体。
要求：互联网版，对话工具下的 Prompt 项目，根据输入的日报内容汇总周报，
不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

该智能体工作原理如下：

1. 读取指定目录下的规范文档
2. 解析输入的日报内容
3. 识别相同工作条目并合并
4. 按产品分类整理
5. 输出最终进度状态

企业实施时，只需在 AiPy 企业版常规设置中配置工作目录，选择对话工具型智能体，即可实现自动化处理。语言支持中文、English 和日语三种，可根据企业国际化需求灵活切换。

### 2.2 内部管理系统快速开发

制造业内部管理系统需求频繁，传统开发周期长、成本高。AiPy 集成编程能力较强的智能体（如 GLM4.5），可快速生成可运行的系统 demo。

某汽车 4S 店使用 AiPy 开发访客登记管理系统的提示词示例：

```
我是一家汽车 4S 店的负责人，帮我做一个访客登记管理系统，采用 Sqlite 数据库，
有登录界面，信息查询、添加、修改、删除等功能。主要是记录访客到访时间、
访客姓名、联系方式、关注车型、接待销售、备注信息等，关注车型里面要预置
奥迪的常见车型、联系方式主要是 11 位手机号要校验格式、接待销售预置这三个
销售刘明、张小兵、李爱国。要求可以点击图标直接打开运行、窗口可以随便拖动
最大化最小化等。做好后，要求后续可以一键点击运行使用。
```

系统功能包括：

- 登录界面验证
- 访客信息 CRUD 操作
- 手机号格式校验（11 位）
- 预置车型库（奥迪常见车型）
- 销售人员名单管理
- 一键运行部署

该系统开发周期从传统 3 周缩短至 2 天，且支持后续一键点击运行，大幅降低维护成本。

### 2.3 多模态内容生成

制造企业营销、培训、汇报场景需要大量 PPT、图片、视频素材。AiPy 支持多模态智能体配置：

**PPT 生成**：勾选 PPT 生成智能体，提示词示例"生成一个介绍中国新能源汽车发展的 PPT"，2 小时内即可完成专业级演示文稿。

**图片/视频生成**：勾选图片生成/视频生成智能体，提示词示例"批量生成 10 张不同的工业设备操作示意图"，满足培训材料制作需求。

**思维导图生成**：领导层阅读材料后生成思维导图，提示词示例"读取 C:\Users\Administrator\Desktop\AIPY 智能体一体机方案 v3.pptx，然后帮我整理为一个思维导图"，快速提炼核心要点。

## 三、AiPy 企业版部署配置指南

### 3.1 常规设置配置

AiPy 企业版常规设置包含以下关键参数：

| 配置项 | 可选值 | 推荐设置 |
|-------|-------|---------|
| 语言 | 中文、English、日语 | 中文 |
| 风格 | aipy-enterprise.yml 配置文件 | 根据企业规范 |
| 发送快捷键 | Ctrl+Enter 或 Enter | Ctrl+Enter |
| 工作目录 | 自定义路径 | C:\project\git\aigw |
| 最大执行轮数 | 数字 | 10 |
| 超时时间 | 秒数 | 300 |
| 自动选择智能体 | 开启/关闭 | 开启 |
| 一体机内网 IP | IP 地址 | 根据网络配置 |

### 3.2 智能体类型选择

AiPy 智能体 manifest.json 中 keywords 字段决定智能体类型和集市展示分类：

**类型关键字**（第一个值）：
- conversation-tool（对话工具型）
- embed-webview（嵌入页面型）
- application（独立应用型）
- webview（网页型）
- skills（技能型）

**集市展示分类**（第二个值）：
- 威胁情报
- 暴露面与脆弱性
- 安全产品
- 威胁检测与响应
- 开发安全
- 数据安全
- AI 安全
- 安全服务
- 网络安全
- 其他
- 官方精选

制造企业推荐选择 conversation-tool 类型，集市分类选择"其他"或"官方精选"。

### 3.3 特殊功能启用

部分功能需要手动勾选对应智能体：

- **图片识别**：勾选视觉理解智能体
- **图片生成**：勾选图片生成智能体
- **PPT 优化**：勾选 PPT 生成智能体
- **实时信息**：开启联网搜索功能
- **股票分析**：使用"量化研究"智能体（美股、港股、A 股历史数据）
- **编程辅助**：使用 Claude code 智能体

## 五、实施建议与注意事项

### 5.1 分阶段推进策略

第一阶段（1-2 周）：完成基础配置，包括工作目录设置、语言选择、智能体启用。优先部署周报汇总、文档处理等高频场景。

第二阶段（3-4 周）：扩展至内部系统开发、多模态内容生成。培训核心团队掌握提示词工程技巧。

第三阶段（5-8 周）：深度集成企业现有系统，配置 Workflow 编排，实现跨系统自动化。

### 5.2 关键成功因素

**明确业务场景**：避免盲目部署，优先选择痛点明显、ROI 清晰的场景。

**规范提示词编写**：参考 AiPy 官方提示词示例，确保指令清晰、参数完整。

**持续优化迭代**：定期复盘使用效果，根据反馈调整智能体配置和工作流程。

**团队能力建设**：组织 Prompt Engineering 培训，提升全员 AI 应用能力。

### 5.3 常见问题解决

**日期时间不准**：开启联网搜索功能获取实时信息。

**图片生成异常**：确认已勾选图片生成智能体。

**PPT 质量不佳**：启用 PPT 生成智能体并优化提示词。

**代码运行失败**：使用 Claude code 智能体，选择编程能力较强的大模型如 GLM4.5。

## 相关问答 FAQs

**AiPy 企业版支持哪些语言配置？**

AiPy 企业版常规设置中语言选项支持中文、English 和日语三种语言。用户可在 aipy-enterprise.yml 配置文件中进行修改，根据企业国际化需求灵活切换。建议在部署初期就确定语言设置，避免后续因语言切换导致的工作目录、配置文件路径等问题。

**如何选择合适的 AiPy 智能体类型？**

智能体类型选择需根据 manifest.json 中 keywords 字段配置。第一个值为类型关键字，包括 conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。制造企业日常办公场景推荐使用 conversation-tool 类型，内部系统开发选择 application 类型，网页集成场景选择 webview 类型。第二个值为集市展示分类，可根据业务属性选择相应分类或"官方精选"。

**AiPy 生成的内容质量如何保证？**

内容质量取决于多个因素。首先需勾选对应的智能体，如图片生成需勾选图片生成智能体，PPT 制作需勾选 PPT 生成智能体。其次提示词编写要规范，参考官方示例，明确输入输出要求。第三选择合适的大模型，编程任务选择 GLM4.5 等编程能力较强的模型。第四开启必要功能，如实时信息需开启联网搜索。最后持续优化提示词，根据输出效果迭代调整，形成企业专属的提示词库。
