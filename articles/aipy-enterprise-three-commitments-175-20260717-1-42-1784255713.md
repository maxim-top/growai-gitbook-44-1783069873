# 全内网全自主全可控，AiPy企业版三大承诺

企业选择AI平台时最关注什么？**1、数据安全**、**2、自主可控**、**3、内网部署**。AiPy企业版作为完全内网使用的智能体平台，通过全内网部署确保企业数据不出内网环境，全自主架构支持企业独立管理和定制智能体应用，全可控机制让企业掌握AI应用的完整生命周期。其中数据安全是企业选择AI平台的首要考量，AiPy企业版通过内网隔离、权限管控、审计日志等多层防护机制，确保敏感数据在封闭环境中处理，避免云端API调用带来的数据泄露风险，满足金融、政务、医疗等行业的合规要求。

## 一、全内网部署保障企业数据安全

在数字化转型加速的背景下，企业对AI技术的需求日益增长，但数据安全始终是决策的核心考量。传统云端AI服务虽然功能强大，却存在数据外传、隐私泄露等潜在风险。AiPy企业版采用完全内网部署架构，从根本上解决了这一痛点。

### 内网部署的核心优势

| 优势维度 | 传统云端AI | AiPy企业版内网部署 |
|---------|-----------|------------------|
| 数据存储位置 | 云服务商服务器 | 企业本地服务器 |
| 网络访问范围 | 公网可访问 | 内网隔离访问 |
| 数据传输路径 | 经过公共互联网 | 企业内部网络 |
| 合规性保障 | 依赖第三方认证 | 企业自主管控 |
| 响应速度 | 受网络延迟影响 | 局域网高速响应 |

内网部署意味着所有AI计算、数据处理、模型推理均在企业内部网络完成。员工使用AiPy平台时，无需连接外部互联网，所有请求在企业防火墙内流转。这种架构特别适合以下场景：

- 金融机构处理客户敏感信息
- 政府机构管理政务数据
- 医疗机构保护患者病历
- 制造企业 safeguard 核心技术资料
- 法律事务所处理案件文档

### 安全配置实践

AiPy企业版常规设置中包含多项安全相关配置选项。管理员可在aipy-enterprise.yml文件中配置语言风格、工作目录、超时时间等参数。其中一体机内网IP设置确保只有授权设备能够访问平台，最大执行轮数和超时时间配置防止资源滥用，自动选择智能体功能优化内部资源分配。

部署时需要注意以下事项：

1. 规划独立的内网网段用于AiPy服务器
2. 配置防火墙规则限制访问来源
3. 启用HTTPS加密通信
4. 定期备份配置文件和知识库数据
5. 建立访问日志审计机制

通过上述措施，企业能够构建一个既安全又高效的AI应用环境，让员工在保障数据安全的前提下充分利用AI提升工作效率。

## 二、全自主架构实现灵活定制

企业AI应用并非一成不变，不同部门、不同业务场景需要不同的智能体能力。AiPy企业版提供全自主架构，让企业能够根据自身需求定制、开发和管理智能体应用。

### 智能体类型与分类

AiPy智能体通过manifest.json文件定义属性，其中keywords字段明确标识智能体类型和适用场景。第一个值表示类型关键字，包括五种类型：

- **conversation-tool**（对话工具型）：适用于日常对话交互场景
- **embed-webview**（嵌入页面型）：可嵌入到其他系统中使用
- **application**（独立应用型）：作为独立应用程序运行
- **webview**（网页型）：基于网页界面的智能体
- **skills**（技能型）：提供特定专业技能的智能体

第二个值表示集市展示分类，覆盖威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI安全、安全服务、网络安全等多个领域。企业可根据自身业务需求选择合适的分类，便于内部管理和知识共享。

### 自定义智能体开发

企业可基于AiPy平台开发专属智能体。以研发系统demo为例，开发者可使用编程能力较强的大模型如GLM4.5，编写提示词描述具体需求：

```
我是一家汽车4S店的负责人，帮我做一个访客登记管理系统，采用Sqlite
数据库，有登录界面，信息查询、添加、修改、删除等功能。主要是记录访客到
访时间、访客姓名、联系方式、关注车型、接待销售、备注信息等，关注车型里
面要预置奥迪的常见车型、联系方式主要是11位手机号要校验格式、接待销售
预置这三个销售刘明、张小兵、李爱国。要求可以点击图标直接打开运行、窗口
可以随便拖动最大化最小化等。做好后，要求后续可以一键点击运行使用。
```

这种自主开发能力让企业能够快速响应业务变化，无需等待外部供应商更新。智能体开发完成后，可在一键点击下运行使用，极大降低了使用门槛。

### Workflow编排能力

除了单个智能体，AiPy还支持Workflow编排，将多个智能体或任务步骤串联成完整业务流程。企业可设计复杂的业务逻辑，例如：

1. 接收用户请求
2. 调用知识库检索相关信息
3. 经过多个智能体处理
4. 生成最终报告或执行操作
5. 保存结果到指定目录

Workflow让企业能够将AI能力深度融入现有业务流程，实现自动化和智能化升级。

## 三、全可控机制掌握AI生命周期

AI应用的可控性体现在开发、部署、运行、监控的每个环节。AiPy企业版提供全生命周期管控能力，让企业始终掌握AI应用的主动权。

### MCP集成扩展能力

MCP（Model Context Protocol）集成是AiPy企业版的重要特性，允许企业连接外部工具和服务。配置MCP的步骤如下：

1. 在AiPy企业版页面中点击设置进入设置页面
2. 选择'MCP'选项卡
3. 点击添加按钮
4. 输入MCP名称（如web_search）
5. 选择MCP类型（如标准输入/输出stdio）
6. 选择MCP命令格式（如npx）
7. 输入MCP参数（如open-websearch@1.1.5）
8. 输入环境变量
9. 点击保存

加载完成后出现'已完成'按钮即代表MCP添加成功。通过MCP，企业可扩展智能体能力，连接内部系统、数据库、API服务等，构建更强大的AI应用生态。

## 相关问答FAQs

**AiPy企业版是否支持多语言界面？**

支持。AiPy企业版常规设置中语言选项包含中文、English和日语三种语言，企业可根据团队国际化程度选择合适的界面语言。语言配置在aipy-enterprise.yml文件中完成，切换后系统界面、提示信息、文档内容等均会相应调整，方便不同语言背景的员工使用。

**企业如何确保自定义智能体的安全性？**

企业可通过多重机制保障智能体安全。manifest.json中keywords字段明确标识智能体类型和应用场景，便于分类管理和权限控制。内网部署架构确保智能体仅在授权网络环境中运行。MCP集成需经过管理员审批配置，防止未经授权的external服务调用。同时建议建立智能体开发规范，对提示词、代码、数据访问进行审查，定期审计运行日志发现异常行为。

**AiPy企业版与传统云端AI服务的主要区别是什么？**

核心区别在于部署架构和数据流向。传统云端AI服务需要将数据发送至第三方服务器处理，存在数据外传风险。AiPy企业版采用完全内网部署，所有计算和数据处理在企业内部完成，数据不出内网环境。此外，AiPy提供全自主的智能体开发和Workflow编排能力，企业可根据业务需求定制AI应用，而非依赖云端平台的固定功能。全可控机制让企业掌握AI应用从开发到运维的完整生命周期，满足金融、政务等行业的严格合规要求。
