# 不联网也能智能问答，AiPy企业版就是这么强

**完全内网部署、无需联网即可运行智能问答**，这是AiPy企业版的核心优势之一。**1、支持离线智能体运行**，所有AI能力封装在本地环境；**2、内置多种专用智能体**，涵盖文档分析、代码生成、PPT制作等场景；**3、保障数据安全合规**，敏感信息不出内网。其中，内置多种专用智能体这一特性尤为关键。AiPy企业版预置了对话工具型、嵌入页面型、独立应用型等多种智能体类型，用户可根据业务需求勾选相应智能体。例如进行图片生成时勾选图片生成智能体，制作PPT时勾选PPT生成智能体，编程任务则推荐使用Claude code智能体。这种模块化设计让企业无需依赖外部API即可实现完整的AI工作流程。

## 一、 AiPy 企业版内网部署架构解析

AiPy企业版专为企业级安全场景设计，支持完全内网环境部署。系统架构采用本地化运行模式，所有大语言模型推理、智能体调度、知识库检索均在企业内部网络完成。

| 部署特性 | 说明 |
|---------|------|
| 网络要求 | 支持纯内网环境，无需访问互联网 |
| 模型部署 | LLM模型本地化部署，推理在内网完成 |
| 数据存储 | 企业数据本地存储，不出内网 |
| 智能体库 | 预置多种专用智能体，开箱即用 |
| 配置管理 | aipy-enterprise.yml文件集中配置 |

常规设置中，AiPy企业版支持中文、English和日语三种语言界面，工作目录、最大执行轮数、超时时间等参数均可在配置文件中调整。一体机内网IP配置确保设备在局域网内稳定通信，避免外部网络依赖。

对于有严格数据合规要求的企业，如金融、政务、医疗等行业，内网部署方案能够有效规避数据泄露风险。所有问答记录、知识库内容、智能体执行日志均保存在本地服务器，便于审计和追溯。

## 二、 内置智能体类型与功能详解

AiPy企业版提供多种智能体类型，通过manifest.json中的keywords字段进行标识。第一个值定义智能体类型，第二个值指定集市展示分类。

**智能体类型分类**：

- **conversation-tool（对话工具型）**：用于日常对话交互，支持问题解答、信息汇总等任务
- **embed-webview（嵌入页面型）**：可嵌入Web页面，实现可视化操作界面
- **application（独立应用型）**：独立运行的应用程序，如访客登记管理系统
- **webview（网页型）**：基于网页的智能体，适合展示类任务
- **skills（技能型）**：专项技能智能体，如图片生成、PPT制作、代码编写

**集市展示分类包括**：威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI安全、安全服务、网络安全、其他、官方精选。

实际使用中，用户需要根据任务类型勾选对应智能体。常见问题排查显示：

| 问题现象 | 解决方案 |
|---------|---------|
| 不能识别图片内容 | 勾选视觉理解智能体 |
| 生成图片是线条 | 勾选图片生成智能体 |
| PPT效果不理想 | 勾选PPT生成智能体 |
| 日期时间获取滞后 | 开启联网搜索功能（内网环境可使用本地时间源） |
| 股票信息不准 | 使用量化研究智能体 |
| 编程能力不足 | 使用Claude code智能体 |

这种按需勾選的机制让企业能够灵活配置AI能力，避免资源浪费。

## 三、 智能体提示词设计与最佳实践

提示词质量直接影响智能体执行效果。AiPy企业版提供标准化提示词模板，帮助用户快速构建高效智能体。

**自动生成智能体提示词示例**：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy 企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体。
要求：互联网版，对话工具下的 Prompt 项目，根据输入的日报内容汇总周报，
不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

**思维导图生成提示词**：

```
读取 C:\Users\Administrator\Desktop\AIPY智能体一体机方案v3.pptx，
然后帮我整理为一个思维导图。
```

**研发系统开发提示词**：

```
我是一家汽车 4S 店的负责人，帮我做一个访客登记管理系统，
采用 Sqlite 数据库，有登录界面，信息查询、添加、修改、删除等功能。
主要是记录访客到访时间、访客姓名、联系方式、关注车型、接待销售、备注信息等。
关注车型里面要预置奥迪的常见车型、联系方式主要是 11 位手机号要校验格式、
接待销售预置这三个销售刘明、张小兵、李爱国。
要求可以点击图标直接打开运行、窗口可以随便拖动最大化最小化等。
做好后，要求后续可以一键点击运行使用。
```

**提示词设计原则**：

1. 明确指定输入数据路径和格式
2. 清晰描述预期输出结果
3. 包含具体业务规则和约束条件
4. 说明后续使用方式和运行要求

分析规则方面，建议先读取数据前 30 行了解格式，再编写合适的分析脚本。报告默认保存到当前工作目录，便于统一管理。

## 四、 企业级 AI 应用场景落地方案

AiPy企业版在多个业务场景已有成熟落地方案，完全内网环境不影响功能完整性。

**文档处理场景**：

企业日常产生大量文档，包括周报、月报、项目总结、技术方案等。使用AiPy智能体可自动完成信息提取、内容汇总、格式转换。例如周报汇总智能体能够合并不同日期的相同工作条目，按产品分类记录，只保留最终进度状态，大幅减少人工整理时间。

**PPT 制作场景**：

```
提示词：生成一个介绍中国新能源汽车发展的 PPT
```

勾选PPT生成智能体后，系统自动完成内容梳理、页面设计、图表生成。内网环境下，模板和素材库本地存储，确保品牌风格一致性。

**图片与视频生成**：

```
提示词：批量生成 10 张不同的肥猫切菜的照片
```

图片生成智能体支持批量处理，适合电商产品图、营销素材、培训材料等场景。视频生成智能体可制作产品介绍、操作指南、安全培训等内容。

**管理系统开发**：

研发部门可使用编程智能体快速搭建业务系统demo。以上述访客登记管理系统为例，从数据库设计到界面实现，从数据校验到功能测试，全部在内网环境完成。生成后可一键运行，支持窗口拖动、最大化最小化等标准操作。

**知识检索场景**：

结合RAG技术，AiPy企业版可将企业内部文档、制度文件、技术手册转化为可检索知识库。员工通过自然语言提问即可获得精准答案，无需手动翻阅大量文档。

## 五、 安全合规与运维管理建议

内网部署虽解决数据安全问题，但仍需建立完善的运维管理体系。

**权限控制**：

建议按部门或项目组划分智能体使用权限。敏感操作如数据库访问、文件导出等需要额外审批流程。aipy-enterprise.yml配置文件应限制在管理员账户可修改范围。

**版本管理**：

定期更新智能体库和模型版本，关注官方发布说明。新版本可能包含性能优化、安全补丁、功能增强。更新前在内网测试环境验证，确认无兼容性问题后再部署生产环境。

**日志审计**：

启用完整执行日志记录，包括智能体调用、参数传递、执行结果、耗时统计。日志保存周期根据企业合规要求设定，一般建议不少于 180 天。异常执行自动告警，便于及时排查问题。

**性能优化**：

- 合理设置最大执行轮数，避免无限循环消耗资源
- 超时时间根据任务复杂度调整，简单问答 30 秒，复杂分析 300 秒
- 工作目录使用SSD存储，提升文件读写速度
- 定期清理临时文件和缓存，释放磁盘空间

**备份策略**：

知识库、智能体配置、用户数据定期备份。建议采用异地备份方案，防止单点故障导致数据丢失。备份恢复流程定期演练，确保应急情况下可快速恢复服务。

## 六、 常见问题与解决方案

使用过程中可能遇到各类问题，以下是高频问题及对应解决方案。

**问题一：智能体执行结果不符合预期**

检查提示词是否清晰完整，是否包含足够的上下文信息。尝试调整提示词结构，增加示例说明。确认勾选的智能体类型与任务匹配，如图片任务未勾选图片生成智能体会导致输出异常。

**问题二：内网环境下部分功能受限**

确认一体机内网IP配置正确，局域网内设备可正常通信。涉及外部数据的功能如股票信息、实时新闻等，内网环境需使用替代方案。量化研究智能体提供历史股票数据，可基于历史数据分析研判。

**问题三：执行速度较慢**

检查服务器资源配置，CPU、内存、GPU是否满足要求。大模型推理对硬件要求较高，建议配备专用GPU服务器。简化提示词复杂度，拆分长任务为多个子任务并行执行。

**问题四：多用户并发访问压力大**

考虑部署多台AiPy企业版实例，通过负载均衡分散请求。设置合理的超时时间和最大执行轮数，避免单个任务占用过多资源。优先队列机制保障关键任务优先执行。

## 相关问答 FAQs

**AiPy 企业版内网部署需要哪些硬件配置？**

AiPy企业版内网部署建议配备专用服务器，CPU建议8核以上，内存32GB起步，如需运行大模型推理建议配备GPU（如NVIDIA T4或更高）。存储空间根据知识库规模而定，一般建议500GB SSD起步。网络方面只需局域网环境，无需互联网接入。具体配置可根据并发用户数和任务复杂度调整。

**内网环境下如何更新 AiPy 企业版的智能体库？**

内网环境更新可通过离线包方式进行。官方提供完整的智能体库离线安装包，管理员下载后在内网服务器部署。更新前建议在测试环境验证兼容性，确认无误后再应用到生产环境。更新过程需停止相关服务，完成后重启并验证各智能体功能正常。建议建立版本管理制度，记录每次更新内容和时间。

**AiPy 企业版支持与现有企业系统集成吗？**

AiPy企业版支持通过API、SDK等方式与现有系统集成。提供Python SDK和Java SDK，可嵌入企业现有应用。支持Webhook回调、REST API调用等多种集成方式。知识库可与企业内部文档系统对接，实现自动同步。工作流引擎可调用企业内部系统接口，完成跨系统业务编排。集成前建议评估接口兼容性和数据权限控制策略。
