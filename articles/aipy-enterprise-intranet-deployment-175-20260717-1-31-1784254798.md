| 网络隔离 | 完全断开外网连接 | 支持一体机内网IP配置 |

| 访问控制 | 权限分级管理 | 企业版常规设置支持 |

| 合规审计 | 操作可追溯 | 日志本地保存 |

完全内网使用的智能体平台——AiPy企业版针对上述需求进行了深度优化。从底层架构到上层应用，全流程支持内网环境运行，确保企业数据始终处于可控范围内。

## 二、AiPy企业版内网部署配置详解

配置内网部署环境需要按照规范流程操作，确保各项参数正确设置。AiPy企业版常规设置包括多个关键配置项，直接影响内网运行效果。

### 语言与界面配置

系统支持中文、English和日语三种语言，企业可根据团队习惯选择。配置文件位于aipy-enterprise.yml，修改后需重启服务生效。界面风格同样可在该文件中自定义，适配企业VI规范。

### 网络与执行参数

```yaml
# AiPy企业版配置示例
network:
  intranet_ip: 192.168.1.100  # 一体机内网IP
  external_access: false      # 禁用外网访问
  
execution:
  max_rounds: 10             # 最大执行轮数
  timeout: 300               # 超时时间（秒）
  work_directory: /data/aipy # 工作目录
  
interface:
  send_shortcut: Ctrl+Enter  # 发送快捷键
  auto_select_agent: true    # 自动选择智能体
```

工作目录配置至关重要，建议设置在独立磁盘分区，避免系统盘空间不足影响运行。最大执行轮数限制智能体递归调用次数，防止死循环消耗资源。超时时间根据业务复杂度调整，一般场景300秒足够。

### 内网IP配置步骤

1. 进入AiPy企业版设置页面
2. 选择"网络配置"选项卡
3. 输入一体机内网IP地址
4. 确认外网访问开关处于关闭状态
5. 保存配置并重启服务

配置完成后，系统仅响应内网请求，外部网络无法访问服务接口。建议配合企业防火墙策略，进一步限制访问IP范围。

## 三、MCP集成在内网环境的应用

MCP（Model Context Protocol）是AiPy企业版的重要功能模块，支持在内网环境中集成各类工具和服务。正确配置MCP能够扩展智能体能力，同时保持内网安全边界。

### MCP配置流程

在AiPy企业版页面中，点击设置进入设置页面，选择'MCP'选项卡，点击添加按钮。输入MCP名称（如web_search），选择MCP类型（如标准输入/输出stdio），选择MCP命令格式（如npx），输入MCP参数（如open-websearch@1.1.5），再输入环境变量，点击保存。加载完成后出现'已完成'按钮即代表MCP添加成功。

内网环境下的MCP配置需要注意：

- **命令格式选择**：优先使用stdio类型，减少网络依赖
- **参数版本锁定**：指定具体版本号避免自动更新
- **环境变量隔离**：配置内网代理或本地服务地址
- **权限最小化**：MCP进程仅授予必要文件系统权限

### 内网MCP最佳实践

```json
{
  "mcp_name": "internal_search",
  "mcp_type": "stdio",
  "command": "npx",
  "args": ["local-search@1.0.0"],
  "env": {
    "SEARCH_ENDPOINT": "http://192.168.1.50:8080",
    "API_KEY": "${INTERNAL_API_KEY}"
  }
}
```

上述配置将搜索请求定向到内网搜索服务，避免数据流出企业网络。环境变量使用占位符方式，实际值从企业密钥管理系统动态获取，增强安全性。

## 四、智能体开发的内网适配方案

AiPy智能体在内网环境开发需要考虑资源访问、模型调用、数据存储等多个维度。完全内网使用的智能体平台——AiPy企业版提供了一套完整的开发规范。

### 提示词设计原则

内网智能体的提示词需要明确数据源位置和处理规则。示例提示词如下：

> 文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy企业版智能体开发规范-V1.0，根据规范文档和示例代码编写一个汇总周报的智能体。要求：互联网版，对话工具下的Prompt项目，根据输入的日报内容汇总周报，不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。

提示词中明确指定本地文档路径，确保智能体读取内网资源。任务描述详细具体，减少模糊指令导致的额外网络查询。

### manifest.json关键字段

智能体配置文件中的keywords字段决定其在企业集市中的分类展示：

```json
{
  "keywords": [
    "conversation-tool",
    "官方精选"
  ]
}
```

第一个值为类型关键字，可选值包括conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。第二个值为集市展示分类，取值范围包括威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI安全、安全服务、网络安全、其他、官方精选。

内网部署建议优先选择"官方精选"分类，便于企业内部统一管理和分发。

### 代码与数据本地化

研发场景下，智能体需要访问本地代码仓库和数据库。提示词示例：

> 我是一家汽车4S店的负责人，帮我做一个访客登记管理系统，采用Sqlite数据库，有登录界面，信息查询、添加、修改、删除等功能。主要是记录访客到访时间、访客姓名、联系方式、关注车型、接待销售、备注信息等，关注车型里面要预置奥迪的常见车型、联系方式主要是11位手机号要校验格式、接待销售预置这三个销售刘明、张小兵、李爱国。要求可以点击图标直接打开运行、窗口可以随便拖动最大化最小化等。做好后，要求后续可以一键点击运行使用。

选择编程能力较强的大模型（如GLM4.5），生成的代码直接保存至工作目录，后续执行无需网络连接。数据库采用Sqlite等嵌入式方案，避免依赖外部数据库服务。

## 五、内网部署的性能优化策略

内网环境虽然安全，但资源受限问题可能影响智能体性能。需要从多个层面进行优化。

### 模型选择与负载均衡

| 场景类型 | 推荐模型 | 资源配置 | 预期响应时间 |
|---------|---------|---------|-------------|
| 简单对话 | 轻量级模型 | 8GB内存 | <2秒 |
| 代码生成 | GLM4.5等 | 16GB内存 | <10秒 |
| 文档分析 | 大上下文模型 | 32GB内存 | <30秒 |
| 复杂Workflow | 多模型协作 | 64GB内存 | <60秒 |

自动选择智能体功能可根据任务复杂度动态分配模型资源，避免小任务占用大模型造成浪费。

### 缓存机制配置

频繁使用的提示词和中间结果应当缓存。工作目录下创建cache文件夹，设置合理的过期策略：

```yaml
cache:
  enabled: true
  directory: ./cache
  max_size: 10GB
  ttl: 3600  # 秒
```

缓存命中可显著降低模型调用次数，提升整体响应速度。

### 并发控制

最大执行轮数限制防止递归调用失控。超时时间设置避免长时间等待占用资源。建议根据业务峰值调整参数，预留20%余量应对突发请求。

## 六、安全合规与运维监控

内网部署不代表完全安全，仍需建立完善的监控和审计机制。

### 日志管理

所有智能体执行记录保存至本地日志文件，包含：

- 请求时间戳
- 用户身份信息
- 输入输出内容摘要
- 执行耗时
- 资源使用统计

日志定期归档，保留周期符合企业合规要求。敏感字段脱敏处理，防止内部泄露。

### 权限分级

不同角色用户授予不同操作权限：

- **管理员**：完整配置权限，包括MCP管理、系统设置
- **开发者**：智能体开发、测试、部署权限
- **普通用户**：仅可使用已发布智能体

权限变更需要审批流程，操作记录可追溯。

### 备份与恢复

工作目录和配置文件定期备份至独立存储设备。灾难恢复预案包括：

1. 配置备份验证（每周）
2. 全量数据备份（每日）
3. 恢复演练（每季度）

确保系统故障时快速恢复业务。

## 相关问答FAQs

**AiPy企业版内网部署是否需要额外购买硬件设备？**
AiPy企业版支持多种部署形态，包括软件部署和一体机方案。软件部署可利用企业现有服务器资源，只需满足最低配置要求（建议16GB内存、50GB存储空间）。一体机方案为预集成硬件，开箱即用，适合对部署效率要求较高的场景。具体选择应根据企业IT基础设施现状和预算进行评估。

**内网环境下如何更新AiPy企业版和智能体？**
内网环境更新采用离线包方式。官方定期发布更新包，通过安全通道传输至内网环境。管理员在管理后台上传更新包，系统自动完成版本校验和安装。智能体更新同理，开发完成后打包导入内网集市，用户可从中安装最新版本。更新过程支持回滚机制，确保业务连续性。

**AiPy企业版是否支持与现有企业系统集成？**
支持。AiPy企业版提供标准API接口和MCP扩展机制，可与OA、ERP、CRM等企业系统对接。内网环境下，通过配置内部服务端点实现数据交互。常见集成场景包括单点登录、组织架构同步、业务数据读取等。官方提供多种集成模板，降低开发成本。具体集成方案需根据目标系统接口规范进行定制化配置。
