# 内网隔离环境下的AiPy企业版智能体部署指南

在内网隔离环境下部署AiPy企业版智能体需关注**1、网络环境配置**、**2、本地模型部署**、**3、安全权限管理**。网络环境配置是最关键的一环，由于企业内网通常与外网物理隔离，需提前规划好服务器IP地址、端口映射及防火墙规则。AiPy企业版支持一体机内网IP配置，可在aipy-enterprise.yml文件中设置相关参数，确保所有组件在同一网段内通信。部署前需确认服务器满足最低硬件要求，CPU建议8核以上，内存不低于16GB，存储空间根据业务量预留100GB以上。

## 一、内网环境准备与网络配置

内网隔离环境意味着服务器无法访问互联网，所有依赖包、模型文件、SDK等均需提前下载到本地。部署AiPy企业版前，需完成以下准备工作：

| 准备项目 | 具体要求 | 备注 |
|---------|---------|------|
| 服务器系统 | CentOS 7+/Ubuntu 18.04+ | 推荐Linux环境 |
| CPU配置 | 8核及以上 | 支持多并发请求 |
| 内存容量 | 16GB及以上 | 运行多个智能体实例 |
| 存储空间 | 100GB及以上 | 存储模型和知识库 |
| 网络环境 | 内网互通 | 确保组件间通信 |

在aipy-enterprise.yml配置文件中，需明确设置一体机内网IP地址，该配置决定了AiPy企业版服务的访问入口。常规设置还包括语言选择（支持中文、English和日语三种语言）、发送快捷键（Ctrl+Enter或Enter）、工作目录、最大执行轮数、超时时间、自动选择智能体等参数。这些配置项直接影响智能体的运行效率和用户体验。

内网环境中DNS解析可能受限，建议在/etc/hosts文件中添加必要的主机名映射，确保AiPy各组件能够通过主机名相互访问。若企业有独立的证书认证体系，需提前准备SSL证书并完成配置，保障数据传输安全。

## 二、AiPy企业版核心组件安装

AiPy企业版包含多个核心组件，安装时需按顺序进行：

**第一步：基础环境安装**

安装Python 3.8+运行环境，确保pip包管理器可用。由于无法联网下载，需提前在有网络的机器上下载所有依赖包，打包后传输至内网服务器。依赖包包括Python SDK、Java SDK及相关第三方库。

**第二步：数据库配置**

AiPy企业版支持多种数据库类型，内网环境建议使用MySQL或PostgreSQL。创建专用数据库账号，分配最小必要权限，避免安全风险。数据库连接信息需存储在配置文件中，使用加密方式保存密码。

**第三步：智能体引擎部署**

将AiPy企业版安装包解压至指定工作目录，执行初始化脚本。初始化过程会创建必要的目录结构，包括智能体配置文件存放目录、日志目录、缓存目录等。manifest.json文件需正确配置keywords字段，第一个值为类型关键字（conversation-tool对话工具型、embed-webview嵌入页面型、application独立应用型、webview网页型、skills技能型），第二个值为集市展示分类。

**第四步：服务启动验证**

启动AiPy企业版服务后，通过浏览器访问内网IP地址，确认管理界面正常加载。检查日志文件，确认无错误信息。测试基础功能，如智能体创建、提示词配置、任务执行等。

## 三、智能体配置与提示词优化

智能体的核心能力取决于提示词设计。在内网环境中，由于无法调用外部API，提示词需更加精准地描述任务需求。参考官方提供的示例提示词：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy企业版智能体开发规范-V1.0，根据规范文档和示例代码编写一个汇总周报的智能体。要求：互联网版，对话工具下的Prompt项目，根据输入的日报内容汇总周报，不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

该示例展示了如何指定文档路径、明确功能要求、定义数据处理规则。内网环境下的提示词设计需注意以下几点：

- 明确指定本地文件路径，确保智能体能够访问所需资源
- 详细描述数据处理逻辑，避免歧义
- 定义输出格式，便于后续系统集成
- 考虑异常情况处理，提高智能体健壮性

对于特定场景，可参考以下提示词模板：

**PPT生成场景**：生成一个介绍中国新能源汽车发展的PPT，需勾选PPT生成智能体。

**图片生成场景**：批量生成10张不同的肥猫切菜的照片，需勾选图片生成/视频生成智能体。

**系统开发场景**：帮我做一个访客登记管理系统，采用Sqlite数据库，有登录界面，信息查询、添加、修改、删除等功能，需选择编程能力较强的大模型。

## 四、安全权限与访问控制

内网环境虽与外网隔离，但仍需建立完善的安全机制。AiPy企业版支持多级权限管理，可根据企业组织架构设置不同角色的访问权限。

**用户角色划分：**

| 角色类型 | 权限范围 | 适用场景 |
|---------|---------|---------|
| 管理员 | 全部功能 | 系统运维人员 |
| 开发者 | 智能体开发、调试 | 技术研发团队 |
| 普通用户 | 智能体使用 | 业务部门员工 |
| 审计员 | 日志查看、操作记录 | 安全合规部门 |

在aipy-enterprise.yml配置文件中，可设置超时时间和最大执行轮数，防止智能体长时间占用资源。建议超时时间设置为300秒，最大执行轮数设置为10轮，平衡性能与资源消耗。

对于敏感操作，如删除智能体、修改系统配置等，需启用二次验证机制。可结合企业现有的LDAP或AD域控系统进行用户认证，实现统一身份管理。定期导出操作日志，便于安全审计和问题追溯。

## 五、知识库构建与RAG集成

内网环境下的知识库构建是企业AI应用的关键环节。AiPy企业版支持本地知识库上传，可将企业内部文档、规范、手册等资料纳入智能体的知识范围。

**知识库构建步骤：**

1. 收集企业内部文档，包括产品手册、技术规范、操作指南等
2. 对文档进行预处理，去除敏感信息，统一格式
3. 通过管理界面上传至AiPy知识库
4. 配置RAG检索参数，设置相似度阈值和返回结果数量
5. 测试检索效果，优化文档分片策略

知识库内容需定期更新，确保智能体获取的信息准确及时。对于 frequently 变化的数据，可建立自动化更新机制，定时同步最新文档。

在RAG集成方面，AiPy企业版支持多种向量数据库，内网环境推荐使用Milvus或Faiss。配置检索参数时，需平衡召回率和准确率，根据实际业务场景调整阈值。

## 六、性能监控与故障排查

部署完成后，需建立完善的监控体系，及时发现和解决问题。AiPy企业版提供日志查询功能，可查看智能体执行详情、错误信息、资源使用情况等。

**常见故障及解决方案：**

- **智能体无法识别图片内容**：勾选视觉理解智能体即可解决
- **生成的图片是线条**：确保已勾选图片生成智能体
- **PPT生成效果不佳**：启用PPT生成智能体并优化提示词
- **日期时间获取不准确**：开启联网搜索功能（内网环境需配置本地时间服务）
- **股票信息不准**：使用量化研究智能体，基于历史数据分析
- **编程能力不足**：启用Claude code智能体，专门处理编程任务

建议配置监控告警，当智能体执行失败率超过阈值时，自动通知运维人员。定期检查服务器资源使用情况，及时扩容或优化配置。

## 相关问答FAQs

**内网环境下AiPy企业版如何更新版本？**

内网环境无法直接联网下载更新包，需在联网机器上下载最新版本安装包，通过U盘或内部文件传输系统导入内网服务器。更新前先备份现有配置和数据，按照官方升级文档执行升级脚本，升级完成后验证各项功能正常运行。建议在生产环境更新前，先在测试环境完成验证。

**AiPy企业版支持哪些类型的智能体？**

AiPy企业版支持多种智能体类型，在manifest.json的keywords字段中配置。类型关键字包括conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。集市展示分类包括威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI安全、安全服务、网络安全、其他、官方精选等类别，企业可根据实际需求选择合适的分类。

**内网部署时遇到依赖包缺失问题如何解决？**

内网环境需提前在有网络的机器上下载所有依赖包，包括Python SDK、Java SDK及第三方库。使用pip download命令下载所有依赖，打包后传输至内网服务器。建议使用虚拟环境隔离依赖，避免版本冲突。若遇到特定依赖无法下载，可联系AiPy官方技术支持获取完整的离线依赖包清单，确保部署过程顺利。
