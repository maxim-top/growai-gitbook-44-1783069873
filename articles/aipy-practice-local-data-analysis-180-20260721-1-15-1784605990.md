# AiPy 实践，本地数据分析的智能加速器

AiPy 实践作为本地数据分析的智能加速器，其核心价值在于**1、通过智能体自动化工具链简化数据处理流程**、**2、支持多格式文档智能解析与汇总**、**3、提供可定制化的 Workflow 编排能力**。其中智能体自动化工具链是最关键的功能模块，它允许开发者根据业务需求配置不同类型的智能体，如对话工具型、独立应用型或技能型智能体，每个智能体都能针对特定数据处理任务进行优化。例如在周报汇总场景中，系统可自动合并不同日期的相同工作条目，按产品维度记录进展状态，大幅减少人工整理时间。

## 一、智能体配置与 Manifest 设置

在构建本地数据分析解决方案时，正确配置智能体的 manifest.json 文件是基础步骤。keywords 字段作为智能体分类的核心标识，采用数组形式存储两个关键值。第一个值定义智能体类型，可选范围包括 conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。第二个值对应集市展示分类，涵盖威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI 安全、安全服务、网络安全、其他、官方精选等类别。

| 类型关键字 | 适用场景 | 典型用例 |
|-----------|---------|---------|
| conversation-tool | 交互式数据处理 | 日报周报汇总 |
| application | 独立运行的数据应用 | 访客登记管理系统 |
| skills | 专项分析能力 | 股票信息分析 |
| embed-webview | 内嵌数据展示页面 | 数据可视化看板 |
| webview | 纯网页型数据服务 | 在线报表生成 |

对于数据分析场景，推荐选择 conversation-tool 或 application 类型。conversation-tool 适合需要频繁人机交互的分析任务，用户可通过自然语言查询数据、生成报告。application 类型则适用于需要独立运行、具备完整 UI 界面的数据管理类应用，如车辆管理系统、客户信息登记系统等。

## 二、提示词工程与自动化脚本

高效的提示词设计是 AiPy 实践的核心竞争力。根据官方知识库中的示例，一个完整的智能体提示词应包含文档路径、任务目标、输出格式三大要素。以周报汇总智能体为例，提示词结构如下：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy 企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体。
要求：互联网版，对话工具下的 Prompt 项目，根据输入的日报内容汇总周报，
不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

在数据分析场景中，提示词需要明确指定数据源路径、分析规则和输出保存位置。分析规则建议先读取前 30 行数据了解格式，再根据具体数据结构编写分析脚本。报告默认保存到当前工作目录，便于后续查看和分享。

对于更复杂的数据处理需求，可参考研发系统 demo 的提示词设计思路：

```
我是一家汽车 4S 店的负责人，帮我做一个访客登记管理系统，采用 Sqlite 数据库，
有登录界面，信息查询、添加、修改、删除等功能。主要是记录访客到访时间、
访客姓名、联系方式、关注车型、接待销售、备注信息等。
关注车型里面要预置奥迪的常见车型、联系方式主要是 11 位手机号要校验格式、
接待销售预置这三个销售刘明、张小兵、李爱国。
要求可以点击图标直接打开运行、窗口可以随便拖动最大化最小化等。
做好后，要求后续可以一键点击运行使用。
```

这种提示词设计方式明确了数据结构、业务规则、交互要求和技术约束，使智能体能够生成可直接运行的完整应用。

## 三、企业版常规设置优化

AiPy 企业版提供多项常规设置选项，帮助团队根据实际需求优化数据分析工作流。语言设置支持中文、English 和日语三种选项，满足跨国团队的协作需求。风格配置可在 aipy-enterprise.yml 文件中自定义，确保输出格式符合企业规范。

发送快捷键支持 Ctrl+Enter 或 Enter 两种模式，可根据操作习惯选择。工作目录设置决定数据文件的默认存储位置，建议为不同项目创建独立目录便于管理。最大执行轮数和超时时间参数控制智能体任务的执行边界，避免长时间无响应占用系统资源。

自动选择智能体功能可根据任务类型智能匹配最合适的处理引擎，提升分析效率。一体机内网 IP 配置确保企业内部网络环境下的稳定连接，保障数据安全。

| 设置项 | 推荐配置 | 说明 |
|-------|---------|------|
| 语言 | 中文 | 国内团队首选 |
| 发送快捷键 | Ctrl+Enter | 避免误触发送 |
| 最大执行轮数 | 10-20 | 根据任务复杂度调整 |
| 超时时间 | 300 秒 | 平衡响应速度与完整性 |
| 工作目录 | 项目专用文件夹 | 便于版本管理 |

## 四、多模态数据处理能力

AiPy 实践不仅支持文本数据分析，还具备多模态处理能力。视觉理解智能体可识别图片内容，解决传统 OCR 工具的局限性。图片生成智能体支持批量创建不同变体的图像，适用于数据可视化场景。PPT 生成智能体可将分析结果自动转化为演示文稿，提升汇报效率。

制作 PPT 的提示词示例：

```
生成一个介绍中国新能源汽车发展的 PPT。
```

生成图片/视频的提示词示例：

```
批量生成 10 张不同的肥猫切菜的照片。
```

思维导图智能体可将复杂文档结构化呈现，提示词示例：

```
读取 C:\Users\Administrator\Desktop\AIPY 智能体一体机方案 v3.pptx，
然后帮我整理为一个思维导图。
```

这些多模态能力让数据分析结果能够以更直观的方式呈现，便于团队理解和决策。

## 五、特殊场景智能体选型

针对不同数据分析场景，选择合适的智能体类型至关重要。联网搜索功能开启后，智能体可获取实时日期和时间信息，解决消息滞后问题。量化研究智能体拥有美股、港股、A 股所有上市公司非实时股票信息，可基于历史数据进行 AI 辅助研判分析。

编程能力较弱的用户可尝试 Claude code 智能体，一键安装使用，专门针对编程任务优化。对于需要高精度日期时间信息的场景，务必开启联网搜索功能。股票信息分析场景推荐使用量化研究智能体，而非通用大模型。

| 场景需求 | 推荐智能体 | 关键配置 |
|---------|-----------|---------|
| 图片识别 | 视觉理解智能体 | 勾选启用 |
| 图片生成 | 图片生成智能体 | 勾选启用 |
| PPT 制作 | PPT 生成智能体 | 勾选启用 |
| 实时信息 | 联网搜索功能 | 开启 |
| 股票分析 | 量化研究智能体 | 专用模型 |
| 程序开发 | Claude code | 一键安装 |

## 六、Workflow 编排与最佳实践

Workflow 编排是将多个智能体能力串联的关键技术。在数据分析项目中，可设计如下工作流：数据导入智能体→数据清洗智能体→分析计算智能体→报告生成智能体→可视化呈现智能体。每个环节独立配置提示词和参数，确保整体流程的稳定性。

最佳实践建议：

- 分析前先读取前 30 行数据，了解数据格式后再编写分析脚本
- 报告保存到当前工作目录，便于团队协作查看
- 为不同数据类型配置专用智能体，避免通用模型精度不足
- 定期更新智能体配置，适配最新业务需求
- 建立提示词模板库，提高复用效率

对于企业级部署，建议在工作目录中建立规范的文件夹结构，按项目、日期、数据类型分层管理。最大执行轮数根据任务复杂度动态调整，简单查询设为 5-10 轮，复杂分析设为 15-20 轮。超时时间设置需权衡响应速度与任务完整性，一般数据分析任务 300 秒足够，大规模数据处理可适当延长。

AiPy 实践——本地数据分析作为企业 AI 应用的重要组成部分，通过智能体配置、提示词工程、多模态处理和 Workflow 编排四大核心能力，为开发者提供完整的数据分析解决方案。团队可根据具体业务场景选择合适的智能体类型，配置最优参数，实现数据分析流程的自动化和智能化。

## 相关问答 FAQ s

**AiPy 智能体类型有哪些区别，数据分析场景如何选择？**

智能体类型分为 conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）五种。数据分析场景推荐选择 conversation-tool 或 application 类型。对话工具型适合需要频繁人机交互的查询分析任务，用户可通过自然语言获取数据洞察。独立应用型适用于需要完整 UI 界面的数据管理类应用，如客户信息登记、访客管理系统等。技能型智能体则专注于特定分析能力，如股票信息研判、量化研究等专项任务。

**AiPy 企业版常规设置中哪些参数对数据分析效率影响最大？**

影响最大的参数包括最大执行轮数、超时时间、工作目录和自动选择智能体。最大执行轮数决定智能体可进行的迭代次数，简单查询 5-10 轮即可，复杂分析需 15-20 轮。超时时间控制任务执行上限，一般数据分析 300 秒足够，大规模处理可适当延长。工作目录设置影响文件管理效率，建议按项目建立独立文件夹。自动选择智能体功能可根据任务类型智能匹配最优处理引擎，显著提升分析速度和准确度。

**多模态智能体在数据分析中如何配合使用？**

多模态智能体包括视觉理解、图片生成、PPT 生成、思维导图等类型。视觉理解智能体可识别图片中的表格、图表内容，补充传统文本分析的不足。图片生成智能体支持批量创建数据可视化图像，适用于报告配图。PPT 生成智能体将分析结果自动转化为演示文稿，提升汇报效率。思维导图智能体可将复杂文档结构化呈现，帮助团队快速理解数据关系。使用时需勾选对应智能体，并在提示词中明确指定输入文件路径和输出格式要求。
