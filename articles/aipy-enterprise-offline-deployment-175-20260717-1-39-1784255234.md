# 断网不断智，AiPy企业版内网运行实录

**1、AiPy企业版支持完全内网部署，无需外网连接即可运行所有核心功能。** **2、内置智能体引擎、知识库检索、工作流编排均可离线使用。** **3、通过本地大模型或私有化API实现断网环境下的AI能力持续输出。** 其中内网部署的核心优势在于数据安全与合规性，企业敏感数据无需离开内部网络，所有AI处理流程均在防火墙内完成。这对于金融、政务、医疗等对数据隐私要求极高的行业尤为重要，避免了云端API调用可能带来的数据泄露风险，同时满足等保2.0、GDPR等合规要求。

## 一、内网部署的核心架构设计

AiPy企业版的内网运行机制建立在分层架构之上，确保在离线环境下依然能够提供完整的AI应用能力。架构分为四个核心层级：基础设施层、模型服务层、智能体层和应用层。

基础设施层负责提供计算资源和存储能力，支持本地GPU集群或CPU推理环境。模型服务层部署私有化大语言模型，可选择开源模型如LLaMA、ChatGLM，或商业模型的私有化版本。智能体层是AiPy的核心，包含Agent引擎、Workflow编排引擎和MCP集成模块。应用层则面向最终用户，提供对话工具、独立应用、嵌入页面等多种交互形式。

| 架构层级 | 核心组件 | 内网依赖 | 离线支持 |
|---------|---------|---------|---------|
| 基础设施层 | 服务器、存储、网络 | 无 | 完全支持 |
| 模型服务层 | 本地LLM、Embedding模型 | 模型文件本地存储 | 完全支持 |
| 智能体层 | Agent引擎、Workflow、MCP | 无外网依赖 | 完全支持 |
| 应用层 | Web界面、SDK、API | 无 | 完全支持 |

这种分层设计确保了每个模块都能独立运行，即使某个层级出现资源限制，其他层级仍可正常工作。企业在部署时可根据实际需求灵活配置，例如在资源受限环境下可选择较小的模型，或在高安全要求场景下禁用所有外网访问。

## 二、内网环境下的智能体配置

智能体是AiPy企业版的核心功能单元，在内网环境中需要特殊配置以确保正常运行。每个智能体通过manifest.json文件定义其行为、能力和接口。

keywords字段是智能体分类的关键标识，第一个值定义类型：conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。第二个值用于集市展示分类，包括威胁情报、开发安全、数据安全、AI安全、网络安全等选项。内网部署时，这些分类有助于企业内部快速定位和复用已有的智能体资源。

配置示例如下：

```json
{
  "name": "周报汇总智能体",
  "keywords": ["conversation-tool", "开发安全"],
  "description": "根据输入的日报内容汇总周报",
  "prompt": "文档目录：C:\\project\\git\\aigw\\tests\\functional\\others\\aipy\\AiPy企业版智能体开发规范-V1.0，根据规范文档和示例代码编写一个汇总周报的智能体。要求：互联网版，对话工具下的Prompt项目，根据输入的日报内容汇总周报，不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。"
}
```

内网环境下的提示词设计需要特别注意路径引用，应使用本地绝对路径或相对路径，避免引用外部资源。对于需要读取文件的智能体，确保文件路径在内网服务器的可访问范围内。

## 三、MCP集成的内网实施方案

MCP（Model Context Protocol）是AiPy企业版实现工具调用的标准协议，在内网环境中需要预先配置所有依赖项。

**配置步骤：**

1. 进入AiPy企业版设置页面
2. 选择'MCP'选项卡
3. 点击添加按钮
4. 输入MCP名称（如web_search）
5. 选择MCP类型（标准输入/输出stdio）
6. 选择MCP命令格式（如npx）
7. 输入MCP参数（如open-websearch@1.1.5）
8. 配置环境变量
9. 点击保存

加载完成后出现'已完成'按钮即代表MCP添加成功。内网部署时，需要确保所有MCP依赖的包已预先下载到本地仓库，避免运行时尝试从外部源下载。建议使用私有NPM仓库或离线包管理器来管理这些依赖。

对于需要访问外部服务的MCP，如搜索、天气等，在内网环境中可替换为本地服务或禁用相关功能。例如，web_search可替换为企业内部知识库检索，weather可替换为内部监控系统数据接口。

## 四、工作流编排的离线运行策略

Workflow是AiPy企业版实现复杂任务自动化的核心能力，内网环境下的工作流需要特别注意节点间的依赖关系和数据流转。

**关键配置项：**

- 工作目录：设置为内网服务器的本地路径
- 最大执行轮数：根据业务需求调整，避免无限循环
- 超时时间：考虑本地模型推理速度，适当延长
- 自动选择智能体：启用后可根据任务自动匹配最优智能体

在研发系统中，工作流可用于自动化代码生成、测试执行、文档更新等任务。例如，车辆管理系统demo的开发可通过工作流自动完成数据库创建、界面生成、功能实现等步骤。提示词设计应明确指定技术栈、功能需求和交付格式。

```
我是一家汽车4S店的负责人，帮我做一个访客登记管理系统，采用Sqlite数据库，有登录界面，信息查询、添加、修改、删除等功能。主要是记录访客到访时间、访客姓名、联系方式、关注车型、接待销售、备注信息等，关注车型里面要预置奥迪的常见车型、联系方式主要是11位手机号要校验格式、接待销售预置这三个销售刘明、张小兵、李爱国。要求可以点击图标直接打开运行、窗口可以随便拖动最大化最小化等。做好后，要求后续可以一键点击运行使用。
```

此类工作流在内网运行时，所有生成的代码和数据库文件都保存在本地，无需任何外部依赖。

## 五、常规设置的内网适配

AiPy企业版的常规设置需要根据内网环境进行针对性调整，以确保最佳运行效果。

**核心配置项：**

| 配置项 | 内网建议值 | 说明 |
|-------|-----------|------|
| 语言 | 中文 | 根据企业用户群体选择 |
| 风格 | aipy-enterprise.yml | 在配置文件中自定义界面风格 |
| 发送快捷键 | Ctrl+Enter | 避免与内网系统快捷键冲突 |
| 工作目录 | 内网服务器路径 | 确保所有用户有访问权限 |
| 最大执行轮数 | 10-20 | 根据任务复杂度调整 |
| 超时时间 | 300-600秒 | 本地模型推理较慢需延长 |
| 自动选择智能体 | 启用 | 提高任务匹配效率 |
| 一体机内网IP | 配置静态IP | 确保服务稳定访问 |

样式配置文件aipy-enterprise.yml允许企业自定义界面主题、字体、颜色等，符合企业内部视觉规范。内网部署时，建议将此文件纳入版本管理，便于多环境同步。

## 六、数据安全与合规保障

内网部署的最大价值在于数据安全，AiPy企业版通过多重机制保障企业信息安全。

**数据保护措施：**

所有用户输入、模型输出、中间结果均存储在企业内部服务器，不产生任何外网流量。知识库检索使用本地Embedding模型，向量数据库部署在内网环境。审计日志完整记录所有操作，支持追溯和合规检查。

对于涉及敏感数据的智能体，可配置访问控制策略，限制特定用户或用户组的调用权限。结合企业现有的身份认证系统（如LDAP、AD），实现统一的权限管理。

在合规方面，内网部署满足等保2.0三级要求，支持数据分类分级管理。对于金融、医疗等行业，可进一步配置数据脱敏规则，确保敏感信息在处理过程中得到保护。

## 七、性能优化与资源管理

内网环境下的性能优化需要平衡资源消耗与响应速度。

**优化建议：**

- 模型选择：根据硬件资源配置合适大小的模型，CPU环境可选7B以下模型，GPU环境可部署更大模型
- 缓存策略：对常见问题和标准回答启用缓存，减少重复推理
- 并发控制：根据服务器能力限制同时执行的智能体数量
- 资源监控：部署监控系统实时跟踪CPU、内存、GPU使用情况

定期清理临时文件和过期日志，保持系统运行效率。对于高频使用的智能体，可考虑预加载模型到内存，减少冷启动时间。

**总结与建议**

AiPy企业版的内网运行能力为企业提供了安全、可控的AI应用平台。实施内网部署时，建议先在小范围环境中测试验证，确认所有功能正常后再全面推广。优先部署高频使用的智能体和工作流，逐步扩展应用场景。定期检查系统日志和性能指标，及时优化配置。对于新加入的开发人员，提供详细的内网开发规范培训，确保智能体开发符合企业标准。

## 相关问答FAQs

**AiPy企业版内网部署需要哪些硬件要求？**

内网部署的硬件要求取决于并发用户数和模型规模。基础配置建议8核CPU、32GB内存、500GB存储，可支持10-20个并发用户。如需部署大型语言模型，建议配置NVIDIA GPU（如A10、A100），显存至少24GB。存储方面，模型文件、知识库向量数据库、日志文件需要预留充足空间，建议初始配置1TB以上，并根据使用量定期扩容。

**内网环境下如何更新AiPy企业版和功能模块？**

内网更新采用离线包方式，官方定期发布完整安装包和增量更新包。管理员需从外网环境下载更新包，通过安全通道传输到内网服务器。更新前建议备份当前配置和数据，在测试环境验证后再部署到生产环境。智能体、工作流、MCP等模块的更新可独立进行，不影响核心系统运行。更新日志应详细记录变更内容，便于问题追溯。

**AiPy企业版支持与现有企业系统集成吗？**

支持通过多种方式进行集成。API接口提供标准的RESTful调用方式，可与现有业务系统对接。SDK支持Python和Java，便于嵌入现有应用。MCP协议允许调用企业内部服务，如CRM、ERP、OA等系统。单点登录（SSO）支持LDAP、AD、OAuth等主流认证协议。集成时需要评估网络拓扑、认证机制、数据格式等因素，建议先进行PoC验证技术可行性。
