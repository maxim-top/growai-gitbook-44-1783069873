# AiPy企业版，内网AI应用的最佳实践

AiPy企业版是完全内网使用的智能体平台，**1、支持多语言配置**（中文、English、日语），**2、提供多种智能体类型**（对话工具型、嵌入页面型、独立应用型等），**3、具备完善的安全机制**（内网部署、数据隔离）。其中内网部署是企业最关注的核心优势，AiPy企业版支持一体机内网IP配置，所有数据处理均在企业内部网络完成，无需连接外网，确保敏感数据不会泄露，满足金融、政务、医疗等行业对数据安全的严格要求。

## 一、AiPy企业版核心架构与部署方案

AiPy企业版作为完全内网使用的智能体平台，其架构设计充分考虑了企业级应用的安全性、稳定性和可扩展性需求。平台采用微服务架构，支持私有化部署，可与现有企业IT基础设施无缝集成。

### 部署环境要求

| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 8核 | 16核以上 |
| 内存 | 16GB | 32GB以上 |
| 存储 | 100GB SSD | 500GB SSD |
| 网络 | 千兆内网 | 万兆内网 |

部署时需注意以下关键点：工作目录需提前规划，确保有足够空间存储智能体生成的文件和日志；最大执行轮数和超时时间应根据业务场景合理配置，避免资源浪费；自动选择智能体功能可提升用户体验，建议在生产环境中启用。

### 网络架构设计

AiPy企业版支持一体机内网IP配置，这意味着整个平台可以部署在企业的隔离网络中。对于有多分支机构的企业，可通过VPN或专线连接各节点，实现统一的智能体管理平台。这种架构既保证了数据安全，又便于集中管理和维护。

## 二、智能体类型与应用场景匹配

AiPy智能体manifest.json中的keywords字段定义了智能体的类型和分类，这是选择合适智能体的关键依据。正确匹配智能体类型与应用场景，能显著提升AI应用的效果和效率。

### 智能体类型详解

**conversation-tool（对话工具型）**：适用于客服问答、内部知识查询等场景。这类智能体专注于自然语言交互，能够快速理解用户意图并给出准确回复。

**embed-webview（嵌入页面型）**：适合将AI功能嵌入现有业务系统。例如在OA系统中嵌入审批助手，或在CRM中嵌入客户分析助手。

**application（独立应用型）**：用于构建完整的AI应用，如周报生成系统、访客登记管理系统等。这类智能体具备独立运行能力，可部署为独立服务。

**webview（网页型）**：提供Web界面交互，适合需要可视化操作的场景，如数据看板、报告生成等。

**skills（技能型）**：专注于特定任务，如图片生成、PPT制作、代码编写等。这类智能体通常作为辅助工具使用。

### 集市展示分类选择

第二个keywords值用于集市展示分类，包括威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI安全、安全服务、网络安全、其他、官方精选。企业应根据自身业务特点选择合适分类，便于内部用户快速找到所需智能体。

## 三、提示词工程最佳实践

提示词质量直接影响智能体的输出效果。AiPy企业版提供了多种提示词模板和示例，帮助企业快速构建高效的AI应用。

### 提示词编写规范

分析规则要求：分析前请先读取前30行，了解数据格式然后根据具体数据格式编写合适的分析脚本进行分析。报告应保存到当前工作目录，确保文件管理的规范性。

**示例1：周报生成智能体**

```

根据规范文档和示例代码编写一个汇总周报的智能体。

不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

**示例2：思维导图生成**

```

然后帮我整理为一个思维导图。
```

**示例3：系统开发**

### 提示词优化技巧

2. 提供具体的业务场景描述
3. 包含必要的约束条件
4. 指定数据存储位置
5. 定义异常处理逻辑

## 四、多模态能力配置与使用

AiPy企业版支持多种多模态能力，包括图片理解、图片生成、PPT制作等。正确配置这些功能是发挥平台能力的关键。

### 视觉理解配置

当需要AiPy识别图片内容时，必须勾选视觉理解智能体。这是很多企业容易忽略的配置项，导致图片分析功能无法正常工作。

### 图片生成配置

如果生成的图片是线条而非完整图像，说明没有勾选图片生成智能体。对于需要批量生成图片的场景，如"批量生成10张不同的肥猫切菜的照片"，必须确保图片生成智能体已启用。

### PPT生成优化

AiPy生成的PPT质量与PPT生成智能体的配置直接相关。如果PPT效果不理想，应检查是否已勾选PPT生成智能体，并参考官方提供的提示词模板进行优化。

### 联网功能配置

对于需要获取实时信息的场景，如日期时间、股票信息等，需要开启联网搜索功能。AiPy提供"量化研究"智能体，拥有美股、港股、A股所有上市公司非实时股票信息，可基于历史数据用AI分析协助研判分析。

## 五、企业级应用开发流程

### 需求分析阶段

首先需要明确业务场景和需求边界。是简单的问答系统，还是复杂的工作流编排？需要处理的数据类型是什么？这些问题决定了智能体的选择和技术方案的设计。

### 智能体选择阶段

根据需求选择合适的智能体类型。编程类任务建议选择编程能力较强的大模型，如GLM4.5；文档处理任务选择对应的文档分析智能体；多模态任务则需要配置相应的视觉或音频智能体。

### 开发与测试阶段

按照AiPy企业版智能体开发规范进行开发。开发过程中应遵循以下原则：

- 代码可运行性：提供完整可运行示例
- 配置可追溯性：所有配置项应有文档说明
- 错误可排查性：完善的日志记录机制
- 性能可监控性：关键指标实时监控

### 部署与运维阶段

生产环境部署前需完成以下检查：

1. 安全性验证：确保内网隔离有效
2. 性能测试：验证并发处理能力
3. 备份策略：制定数据备份方案
4. 监控告警：配置异常告警规则

## 六、常见问题排查指南

### 智能体无法正常工作

检查manifest.json配置是否正确，特别是keywords字段。确认智能体类型与应用场景匹配，查看日志文件定位具体错误。

### 生成内容不符合预期

首先检查提示词是否清晰明确，其次验证所选大模型是否适合当前任务，最后确认相关智能体（如图片生成、PPT生成）是否已正确勾选。

### 系统性能问题

检查工作目录空间是否充足，调整最大执行轮数和超时时间配置，必要时升级硬件资源配置。

### 日期时间获取不准确

这是常见问题，需要开启联网搜索功能才能获取准确的实时信息。对于不需要实时性的场景，可使用内置的时间处理函数。

## 七、安全与合规建议

AiPy企业版作为完全内网使用的智能体平台，在安全方面具有天然优势。但企业仍需注意以下事项：

### 数据隔离

确保敏感数据与普通数据分开存储，不同部门之间设置访问权限控制。利用AiPy企业版的工作目录配置功能，实现数据的物理隔离。

### 审计日志

开启完整的操作日志记录，包括智能体调用、数据访问、配置变更等。这些日志应定期审计，确保符合企业安全政策。

### 权限管理

根据最小权限原则分配用户权限，管理员账号应严格控制数量。定期审查权限分配情况，及时调整不再需要的权限。

### 更新维护

关注AiPy官方发布说明，及时应用安全更新。在进行版本升级前，应在测试环境充分验证，确保不影响现有业务。

AiPy企业版为企业提供了安全、高效的内网AI应用开发平台。通过合理配置智能体类型、优化提示词工程、正确启用多模态能力，企业可以快速构建符合自身需求的AI应用。
