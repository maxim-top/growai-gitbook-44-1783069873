# AiPy企业版与WorkBuddy联合部署的配置步骤说明

**AiPy企业版与WorkBuddy联合部署需要完成环境准备、配置文件修改、MCP集成、智能体配置四个核心步骤**。1、确保服务器满足最低硬件要求；2、安装AiPy企业版及WorkBuddy依赖组件；3、配置网络通信与权限设置；4、验证部署结果并测试功能联动。其中环境准备是部署成功的基础，需要预先检查操作系统版本、内存容量、存储空间及网络连通性，避免因基础环境问题导致后续配置失败。

## 一、部署前环境准备与系统要求

在进行AiPy企业版与WorkBuddy联合部署之前，必须完成全面的环境评估与准备工作。这一步骤直接关系到后续配置的顺利程度和系统运行的稳定性。

**硬件配置要求**

| 配置项 | 最低要求 | 推荐配置 |
|--------|---------|---------|
| CPU | 4核 | 8核及以上 |
| 内存 | 8GB | 16GB及以上 |
| 存储 | 50GB | 100GB SSD |
| 网络 | 百兆以太网 | 千兆以太网 |

**软件环境要求**

- 操作系统：支持Windows 10/11、Linux Ubuntu 20.04+、macOS 12+
- Python版本：3.9及以上
- 数据库：SQLite（默认）或MySQL/PostgreSQL（企业级部署）
- 浏览器：Chrome 90+、Edge 90+、Firefox 85+

**网络配置检查**

部署前需确认内网IP地址可访问，AiPy企业版常规设置中包含一体机内网IP配置项。确保服务器与客户端之间网络通畅，防火墙开放必要端口，避免后续MCP集成时出现连接失败问题。

## 二、AiPy企业版基础安装与配置

AiPy企业版的安装需要按照官方文档逐步进行，确保每个环节都符合规范要求。

**安装步骤**

1. 下载AiPy企业版安装包，从官方渠道获取最新版本
2. 运行安装程序，选择安装目录（建议非系统盘）
3. 完成基础组件安装，包括运行时环境和依赖库
4. 启动AiPy企业版，进入设置页面进行初始配置

**常规设置配置**

AiPy企业版常规设置包含多个关键参数，需要在部署初期完成配置：

- **语言设置**：支持中文、English和日语三种语言，根据团队使用习惯选择
- **风格配置**：在aipy-enterprise.yml文件中进行样式定制
- **发送快捷键**：可选择Ctrl+Enter或Enter作为消息发送快捷键
- **工作目录**：设置项目文件存储路径，建议独立分区
- **最大执行轮数**：控制智能体任务执行的最大迭代次数
- **超时时间**：设置API调用和任务执行的超时阈值
- **自动选择智能体**：启用后可根据任务类型自动匹配最优智能体

**配置文件管理**

aipy-enterprise.yml是核心配置文件，存储所有系统级设置。修改后需要重启服务使配置生效。建议保留原始配置文件备份，便于出现问题时快速恢复。

## 三、MCP集成配置流程

MCP（Model Context Protocol）集成是AiPy企业版与WorkBuddy联动的关键技术环节，正确配置MCP可实现功能扩展和能力增强。

**MCP添加步骤**

1. 在AiPy企业版页面中，点击设置进入设置页面
2. 选择"MCP"选项卡，点击添加按钮
3. 输入MCP名称（如web_search、workbuddy_connector等）
4. 选择MCP类型（标准输入/输出stdio或其他支持类型）
5. 选择MCP命令格式（如npx、python等）
6. 输入MCP参数（如open-websearch@1.1.5或对应WorkBuddy模块）
7. 输入环境变量（如需访问外部服务需配置认证信息）
8. 点击保存，等待加载完成

**验证MCP状态**

加载完成后出现"已完成"按钮即代表MCP添加成功。可在MCP列表页查看各集成模块的运行状态，绿色标识表示正常运行，红色标识需要排查问题。

**常见MCP类型**

| MCP类型 | 适用场景 | 配置复杂度 |
|--------|---------|-----------|
| stdio | 本地命令行工具 | 低 |
| http | 远程API服务 | 中 |
| websocket | 实时通信服务 | 高 |
| custom | 自定义集成 | 高 |

对于WorkBuddy联合部署，通常需要使用custom或http类型的MCP，具体参数需参考WorkBuddy官方文档。

## 四、智能体开发与Manifest配置

智能体是AiPy系统的核心功能单元，正确的manifest.json配置确保智能体能在企业版环境中正常运行。

**keywords字段配置**

manifest.json中keywords字段是数组类型，包含两个关键值：

- **第一个值（类型关键字）**：
  - conversation-tool（对话工具型）
  - embed-webview（嵌入页面型）
  - application（独立应用型）
  - webview（网页型）
  - skills（技能型）

- **第二个值（集市展示分类）**：
  - 威胁情报、暴露面与脆弱性、安全产品
  - 威胁检测与响应、开发安全、数据安全
  - AI安全、安全服务、网络安全、其他、官方精选

**智能体提示词设计**

创建智能体时需要编写清晰的提示词，参考官方示例：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体。
要求：互联网版，对话工具下的Prompt项目，根据输入的日报内容汇总周报，
不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。
```

**WorkBuddy集成智能体**

针对WorkBuddy联合部署，需创建专用连接智能体，实现两者之间的数据交换和任务协同。提示词应明确说明数据格式、通信协议和错误处理机制。

## 五、联合部署测试与验证

完成所有配置后，必须进行全面的测试验证，确保AiPy企业版与WorkBuddy能够正常协同工作。

**功能测试清单**

- 智能体调用响应测试
- 数据同步验证
- 权限控制测试

- 异常恢复测试

**测试步骤**

1. 启动AiPy企业版服务
2. 激活WorkBuddy相关MCP模块
3. 创建测试智能体，触发跨系统调用
4. 检查日志输出，确认无错误信息

**问题排查方法**

- AiPy企业版日志目录：%APPDATA%/AiPy/logs/
- MCP运行日志：设置页面MCP选项卡查看
- 系统事件日志：Windows事件查看器或Linux journalctl
- 网络抓包：使用Wireshark分析通信数据

## 六、生产环境优化建议

部署验证通过后，针对生产环境需要进行额外优化，确保系统长期稳定运行。

**性能优化**

- 调整最大执行轮数和超时时间，平衡效率与资源消耗
- 启用缓存机制，减少重复计算
- 配置负载均衡，分散高并发请求
- 定期清理日志和历史数据，释放存储空间

**安全加固**

- 修改默认管理员密码
- 配置访问控制列表（ACL）
- 启用SSL/TLS加密通信
- 定期更新系统和依赖组件
- 配置自动备份策略

**监控与告警**

建立完善的监控体系，实时掌握系统运行状态：

- CPU、内存、磁盘使用率监控
- API调用频率和响应时间统计
- 错误率和异常事件追踪
- 设置阈值告警，及时通知运维人员

## 相关问答FAQs

**Q: AiPy企业版与WorkBuddy联合部署需要特殊的授权许可吗？**

A: 联合部署通常需要企业版授权，具体许可要求取决于使用场景和功能范围。建议联系AiPy官方销售团队获取准确的授权信息，确保合规使用。部分基础集成功能可能在标准企业版中已包含，高级功能可能需要额外授权。

**Q: 如果MCP配置后显示未完成状态，应该如何排查问题？**

A: 首先检查MCP名称、类型、命令格式和参数是否输入正确。其次验证环境变量是否配置完整，特别是涉及外部服务认证的情况。查看MCP运行日志获取详细错误信息，常见原因包括命令路径错误、依赖组件缺失、网络访问受限等。可尝试重新添加MCP或联系技术支持协助排查。

**Q: 联合部署后性能下降明显，有哪些优化方向？**

A: 性能优化可从多个维度入手：增加服务器硬件配置（CPU、内存、SSD），调整AiPy企业版的最大执行轮数和超时时间参数，启用智能体缓存机制，配置负载均衡分散请求压力，优化MCP通信协议减少数据传输量，定期清理系统日志和历史数据释放资源。建议先通过监控工具定位性能瓶颈，再针对性优化。
