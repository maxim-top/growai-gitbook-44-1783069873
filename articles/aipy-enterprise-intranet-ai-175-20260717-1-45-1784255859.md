# 内网也能有大智慧，AiPy企业版赋能进行时

**内网环境实现智能化办公的核心方案包括：1、选择完全内网部署的智能体平台；2、配置本地化大模型与知识库；3、搭建Workflow自动化流程；4、集成MCP协议扩展能力。** 其中，选择完全内网部署的智能体平台是关键基础。AiPy企业版作为完全内网使用的智能体平台，无需依赖外部网络即可完成AI Agent的创建、编排与执行。企业可将敏感数据保留在内部网络，同时享受大语言模型带来的效率提升。平台支持中文、English和日语三种语言，可在aipy-enterprise.yml文件中灵活配置风格参数，确保符合企业规范。

## 一、内网AI应用的核心挑战与解决方案

企业在推进AI转型时，面临的最大 hurdle 是数据安全与网络隔离的矛盾。传统云服务方案需要将数据上传至外部服务器，这与金融、政务、医疗等行业的合规要求相冲突。AiPy企业版通过本地化部署彻底解决这一痛点。

| 挑战类型 | 传统方案问题 | AiPy企业版解决方案 |
|---------|------------|------------------|
| 数据泄露风险 | 需上传至云端 | 完全内网运行，数据不出域 |
| 网络依赖 | 需稳定外网连接 | 本地模型推理，无需外网 |
| 合规审查 | 难以通过安全审计 | 符合等保要求，支持私有化部署 |
| 响应速度 | 受网络延迟影响 | 局域网内低延迟交互 |
| 成本控制 | 按调用量计费 | 一次性部署，长期使用 |

部署时需注意以下关键配置：工作目录应设置在企业内网可访问的存储路径，最大执行轮数根据业务复杂度调整，超时时间建议设置为300秒以上以应对复杂任务，自动选择智能体功能可减少人工干预。对于有特殊安全需求的企业，可配置一体机内网IP，确保所有通信流量控制在指定网段内。

## 二、智能体开发规范与提示词设计

高效利用AiPy企业版需要掌握正确的提示词编写方法。根据《AiPy企业版智能体开发规范-V1.0》，优质提示词应包含明确的文档目录路径、具体任务描述、输出格式要求三部分要素。

**示例场景：周报汇总智能体**

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy企业版智能体开发规范-V1.0

任务要求：互联网版，对话工具下的Prompt项目，根据输入的日报内容汇总周报

处理规则：
1. 不同日期的相同工作条目合并
2. 按产品记录不同项
3. 只记录最终进度状态
```

该示例展示了如何构建结构化提示词。第一行指定规范文档路径，确保智能体遵循企业标准；第二部分明确智能体类型与应用场景；第三部分详细列出数据处理逻辑。这种三段式结构可大幅提升智能体输出的准确性。

对于思维导图生成场景，提示词可简化为："读取C:\Users\Administrator\Desktop\AIPY 智能体一体机方案 v3.pptx，然后帮我整理为一个思维导图。"系统会自动调用视觉理解智能体解析PPT内容，生成层级分明的思维结构图。研发场景则需选择编程能力较强的大模型如GLM4.5，详细描述系统功能需求、数据库类型、界面交互要求等技术细节。

## 三、智能体类型与集市分类体系

AiPy企业版支持多种智能体类型，通过manifest.json中的keywords字段进行标识。该字段为数组结构，第一个值定义技术类型，第二个值指定业务分类。

**技术类型取值：**

- conversation-tool：对话工具型，适用于问答、咨询场景
- embed-webview：嵌入页面型，可集成到现有Web系统
- application：独立应用型，拥有完整用户界面
- webview：网页型，通过浏览器访问
- skills：技能型，提供特定功能模块

**集市展示分类：**

威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI 安全、安全服务、网络安全、其他、官方精选

企业可根据业务需求选择合适的组合。例如，开发安全团队可选择"skills+ 开发安全"组合，构建代码审查智能体；数据安全部门可采用"conversation-tool+数据安全"配置，打造合规咨询助手。官方精选分类中的智能体经过严格测试，推荐优先使用。

## 四、多模态能力配置指南

AiPy企业版支持图片、视频、PPT等多种内容生成，但需要正确勾选对应智能体才能发挥功能。常见问题与解决方案如下：

| 问题现象 | 原因分析 | 解决方法 |
|---------|---------|---------|
| 无法识别图片内容 | 未启用视觉理解 | 勾选视觉理解智能体 |
| 生成图片呈线条状 | 缺少图像生成模块 | 勾选图片生成智能体 |
| PPT输出效果欠佳 | 未调用专用智能体 | 勾选PPT生成智能体 |
| 日期时间信息滞后 | 缺少联网能力 | 开启联网搜索功能 |
| 股票信息不准确 | 未使用专业模型 | 调用量化研究智能体 |
| 代码编写能力弱 | 模型选择不当 | 使用Claude code智能体 |

**PPT制作最佳实践：**

```
提示词：生成一个介绍中国新能源汽车发展的PPT
前置条件：勾选PPT生成智能体
输出格式：.pptx文件，保存至当前工作目录
```

**图片/视频生成示例：**

```
提示词：批量生成10张不同的肥猫切菜的照片
前置条件：勾选图片生成/视频生成智能体
注意：明确数量与主题，避免模糊描述
```

分析类任务需遵循特定规则：先读取数据前30行了解格式，再编写针对性分析脚本，报告自动保存至当前工作目录。这种分步执行机制确保数据分析的准确性与可追溯性。

## 五、企业级部署架构建议

大型企业在部署AiPy企业版时，需考虑高可用性与扩展性。推荐采用三层架构：接入层负责用户请求分发，计算层运行大模型推理与智能体编排，存储层管理知识库与任务历史。

**部署检查清单：**

1. 确认内网带宽满足并发需求，建议千兆以上
2. 预留充足存储空间用于知识库与缓存文件
3. 配置防火墙规则，仅开放必要端口
4. 建立备份机制，定期导出配置文件
5. 设置监控告警，及时发现异常状态

对于跨地域企业，可在各分支机构部署边缘节点，通过中央控制台统一管理。MCP（Model Context Protocol）集成功能允许连接外部工具与服务，扩展智能体能力边界。Workflow编排引擎支持可视化流程设计，将多个智能体串联成完整业务链路。

LLM选择需平衡性能与资源消耗。GLM4.5适合复杂编程任务，国内模型可避免跨境数据传输问题。RAG（检索增强生成）技术结合企业知识库，提升回答准确性与专业性。Prompt工程优化可显著改善输出质量，建议建立企业专属提示词库。

## 六、运维管理与持续优化

系统上线后需建立完善的运维体系。日常监控重点关注智能体执行成功率、平均响应时间、资源占用率等指标。定期查看日志文件，排查潜在错误。

**性能优化策略：**

- 清理过期任务数据，释放存储空间
- 更新模型版本，获取最新能力提升
- 调整超时参数，适配业务节奏变化
- 收集用户反馈，迭代提示词模板
- 培训使用人员，提升操作熟练度

安全方面需定期审计访问权限，确保只有授权人员可创建或修改智能体。敏感操作应开启二次验证，防止误操作导致数据损失。备份策略建议采用"3-2-1"原则：3份副本、2种介质、1份异地存储。

## 相关问答FAQs

**AiPy 企业版是否支持离线运行？**

是的，AiPy 企业版设计为完全内网使用，所有核心功能包括大模型推理、智能体编排、知识库检索均可在离线环境下运行。企业只需在初次部署时下载必要模型文件，后续使用无需连接外部网络，完美满足金融、政务等对数据安全要求严格的行业需求。

**如何选择合适的智能体类型进行开发？**

选择智能体类型需根据应用场景决定。对话交互场景选用conversation-tool，需要嵌入现有系统时选embed-webview，独立业务系统用application，简单网页展示用webview，功能模块复用选skills。同时结合集市分类标签，确保智能体在组织内正确归类与发现。建议从官方精选类别起步，逐步定制专属智能体。

**AiPy 企业版的提示词优化有哪些技巧？**

优质提示词应包含三个要素：明确的文档或数据源路径、清晰的任务目标描述、具体的输出格式要求。避免模糊表述，使用结构化语言列出处理规则。对于复杂任务，可拆分为多个子步骤分别定义。参考《AiPy 企业版智能体开发规范-V1.0》中的示例，根据业务场景调整模板。定期收集执行结果，持续迭代优化提示词内容。
