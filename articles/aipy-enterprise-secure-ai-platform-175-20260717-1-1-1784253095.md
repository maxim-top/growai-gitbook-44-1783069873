# AiPy企业版，数据不出门智能已到家

**AiPy企业版如何实现数据不出门智能已到家？1、完全内网部署保障数据安全；2、本地化智能体自主运行；3、无需外联即可调用AI能力。**其中完全内网部署是核心优势，AiPy企业版支持在一体机或私有服务器上完整部署，所有数据处理、模型推理、智能体执行均在企业内部网络完成，从根本上避免敏感数据泄露风险。企业无需担心代码、文档、业务数据上传至公有云，满足金融、政务、医疗等行业的合规要求。

## 一、AiPy企业版的核心安全架构

AiPy企业版作为完全内网使用的智能体平台，其安全架构设计围绕"数据不出门"原则展开。传统AI应用往往需要将数据发送至云端API进行处理，这种方式存在数据泄露、合规风险等多重隐患。AiPy企业版通过本地化部署方案，将LLM、Agent、Workflow等核心组件全部内置于企业内网环境。

内网部署模式带来三大安全保障：

| 安全维度 | 传统云端AI | AiPy企业版 |
|---------|-----------|-----------|
| 数据传输 | 需上传至公有云 | 全程内网传输 |
| 数据存储 | 第三方服务器存储 | 企业自有存储 |
| 访问控制 | 依赖云服务商策略 | 企业自主管控 |

企业可在aipy-enterprise.yml配置文件中设定工作目录、最大执行轮数、超时时间等参数，确保所有操作在可控范围内执行。一体机内网IP配置功能进一步隔离外部网络访问，形成完整的安全边界。

## 二、智能体本地化运行机制

AiPy企业版支持多种类型智能体在内网环境中独立运行。根据manifest.json配置文件，智能体可分为对话工具型、嵌入页面型、独立应用型、网页型、技能型五大类别。每种类型均能在不依赖外部服务的情况下完成既定任务。

以周报汇总智能体为例，提示词配置如下：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体
要求：互联网版，对话工具下的Prompt项目
根据输入的日报内容汇总周报
不同日期的相同工作条目合并
按产品记录不同项，只记录最终进度状态
```

该智能体读取本地文档目录，分析日报数据，生成周报汇总，全程无需访问外部网络。企业可根据业务需求定制各类智能体，如访客登记管理系统、数据分析脚本生成、思维导图制作等，所有代码和执行结果均保存在本地工作目录。

## 三、多模态能力的内网实现

AiPy企业版在内网环境下同样支持丰富的多模态功能。用户只需勾选对应智能体即可启用相关能力，无需额外配置外部API。

常见多模态场景及配置方式：

- **图片识别**：勾选视觉理解智能体，可分析上传图片内容
- **图片生成**：勾选图片生成智能体，提示词示例"批量生成10张不同的肥猫切菜的照片"
- **PPT制作**：勾选PPT生成智能体，提示词示例"生成一个介绍中国新能源汽车发展的PPT"
- **视频生成**：勾选视频生成智能体，支持视频内容创作
- **联网搜索**：开启联网搜索功能，获取实时日期、时间、股票等信息

对于股票信息分析，AiPy提供"量化研究"智能体，拥有美股、港股、A股所有上市公司非实时股票信息，可基于历史数据用AI协助研判分析。编程任务可调用Claude code智能体，一键安装使用，专门处理代码编写需求。

## 四、Workflow编排与自动化

AiPy企业版的Workflow功能支持复杂业务流程的自动化编排。企业可将多个智能体串联，形成完整的工作流，实现从数据输入到结果输出的全自动处理。

典型Workflow应用场景：

1. **文档处理流程**：上传文档→内容提取→信息分析→报告生成→存档
2. **客户服务流程**：问题接收→智能匹配→方案生成→回复发送→记录归档
3. **研发辅助流程**：需求分析→代码生成→测试用例→部署脚本→文档更新

每个环节均可配置独立的智能体，设置执行条件和参数传递规则。Workflow支持条件分支、循环执行、错误处理等高级功能，满足企业级应用的复杂性需求。流程执行日志完整记录在本地，便于审计和追溯。

## 五、MCP集成与扩展能力

AiPy企业版支持MCP（Model Context Protocol）集成，可与现有企业系统无缝对接。通过MCP协议，AiPy能够调用企业内部数据库、API服务、文件系统等资源，扩展智能体的能力边界。

MCP集成优势：

| 集成类型 | 功能说明 | 应用场景 |
|---------|---------|---------|
| 数据库连接 | 读取/写入企业数据库 | 客户信息查询、订单处理 |
| API调用 | 调用内部REST API | 系统状态监控、数据同步 |
| 文件系统 | 访问企业文件服务器 | 文档管理、备份归档 |
| 消息队列 | 集成企业消息中间件 | 异步任务处理、通知推送 |

配置MCP集成时，需在智能体manifest.json中声明所需权限和资源类型，确保最小权限原则。企业IT部门可审核并批准每个智能体的访问范围，实现细粒度的安全管控。

## 六、Prompt Engineering最佳实践

在AiPy企业版中，Prompt设计直接影响智能体的执行效果。针对不同任务类型，应采用差异化的Prompt策略。

**数据分析类任务**：分析前请先读取前30行，了解数据格式然后根据具体数据格式编写合适的分析脚本进行分析。报告保存到当前工作目录。

**系统开发类任务**：明确说明技术栈、功能需求、界面要求、数据校验规则等。例如车辆管理系统demo开发，需指定Sqlite数据库、登录界面、信息查询添加修改删除功能、访客信息字段、手机号格式校验、预置销售名单等细节。

**文档处理类任务**：指定输入文档路径、处理规则、输出格式。例如生成思维导图，提示词为"读取C:\Users\Administrator\Desktop\AIPY 智能体一体机方案 v3.pptx，然后帮我整理为一个思维导图"。

Prompt设计应遵循清晰、具体、可执行原则，避免模糊描述导致智能体理解偏差。

## 七、企业部署与运维指南

AiPy企业版部署分为一体机部署和服务器部署两种模式。一体机模式即插即用，适合中小型企业快速上线。服务器部署模式支持集群化扩展，适合大型企业的分布式部署需求。

部署前准备工作：

- 确认硬件配置满足最低要求
- 规划内网IP地址分配
- 准备SSL证书（如需HTTPS访问）
- 配置防火墙规则限制外部访问
- 备份现有业务系统数据

部署完成后，进入常规设置界面配置语言（支持中文、English和日语三种语言）、发送快捷键（Ctrl+Enter或Enter）、自动选择智能体等参数。建议定期更新智能体版本，关注官方发布说明获取新功能和安全补丁。

运维期间需监控系统资源使用情况，包括CPU、内存、存储空间等指标。设置告警阈值，当资源使用超过预设值时及时通知管理员。定期清理临时文件和过期日志，保持系统运行效率。

## 八、常见问题排查方法

使用AiPy企业版过程中可能遇到各类问题，掌握排查方法可快速定位并解决。

**智能体无法执行**：检查manifest.json配置是否正确，keywords字段是否包含正确的类型关键字（conversation-tool、embed-webview、application、webview、skills）和集市展示分类。

**功能无法启用**：确认已勾选对应智能体，如图片生成需勾选图片生成智能体，PPT制作需勾选PPT生成智能体。

**日期时间不准确**：开启联网搜索功能获取实时信息。

**API调用失败**：检查MCP集成配置，确认权限设置和网络连通性。

**性能下降**：调整最大执行轮数和超时时间参数，优化Workflow流程设计，减少不必要的智能体调用。

建立问题记录文档，积累排查经验，形成企业内部的AiPy使用知识库。
