# 老系统继续服役，AiPy给它装上数据共享的翅膀

**企业老旧系统无需推倒重来，通过 AiPy 可实现数据无缝共享。** 核心方案包括：**1、** 利用 AiPy 智能体实现系统间数据自动采集与整合；**2、** 通过 Workflow 编排构建数据流转管道；**3、** 采用 MCP 集成协议打通异构系统接口；**4、** 借助 RAG 技术增强知识库检索能力。其中，Workflow 编排是关键环节，用户可根据业务需求自定义数据流转规则，设置触发条件、数据处理逻辑和输出目标，无需编写复杂代码即可完成系统间的数据桥接，大幅降低集成成本和时间周期。

## 一、老旧系统集成的现实困境

企业在数字化转型过程中，普遍面临一个棘手问题：大量核心业务系统建设于多年前，技术架构陈旧，但承载关键业务数据，无法轻易替换。这些系统往往存在以下特征：

| 问题类型 | 具体表现 | 影响程度 |
|---------|---------|---------|
| 技术栈落后 | 使用过时的编程语言和数据库 | 高 |
| 接口不统一 | 缺乏标准化 API 接口 | 高 |
| 数据孤岛 | 各系统数据无法互通 | 极高 |
| 维护成本高 | 专业技术人员稀缺 | 中 |
| 扩展性差 | 难以对接新业务系统 | 高 |

传统集成方案需要投入大量开发资源进行接口改造，甚至需要重构 entire 系统架构，成本高、周期长、风险大。AiPy 企业版提供了另一种思路——不改造现有系统，通过 AI 智能体实现数据集成。

AiPy 实践表明，不改造现有系统实现数据集成已成为企业 AI 应用的主流方向。通过智能体技术，可以在不触动原有系统代码的前提下，实现数据的自动采集、处理和分发，让老系统焕发新生。

## 二、AiPy 智能体的核心能力解析

AiPy 智能体作为数据集成的核心组件，具备多项关键能力。智能体类型在 manifest.json 中通过 keywords 字段定义，第一个值为类型关键字，包括 conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。

### 2.1 智能体类型选择

根据不同集成场景，需选择合适的智能体类型：

- **对话工具型**：适用于人机交互场景，如客服系统数据查询
- **独立应用型**：适用于后台数据处理任务，如定时数据同步
- **技能型**：适用于特定功能模块，如数据格式转换、校验

### 2.2 Workflow 编排能力

Workflow 是 AiPy 实现数据流转的核心机制。用户可通过可视化界面或配置文件定义数据处理的完整流程，包括：

1. 数据源连接配置
2. 数据抽取规则设定
3. 数据转换逻辑编写
4. 目标系统对接设置
5. 异常处理机制定义

Workflow 支持条件分支、循环处理、并行执行等高级功能，可满足复杂业务场景需求。最大执行轮数和超时时间可在常规设置中调整，确保任务执行的可靠性。

### 2.3 MCP 集成协议

MCP（Model Context Protocol）是 AiPy 提供的标准化集成协议，用于统一不同系统间的通信方式。通过 MCP 集成，可实现：

- 统一的身份认证机制
- 标准化的数据格式转换
- 自动化的错误重试策略
- 完整的操作日志记录

## 三、实际部署方案与步骤

### 3.1 环境准备

部署前需完成以下准备工作：

| 步骤 | 操作内容 | 注意事项 |
|-----|---------|---------|
| 1 | 安装 AiPy 企业版客户端 | 确保系统兼容性 |
| 2 | 配置工作目录 | 建议选择专用目录 |
| 3 | 设置语言风格 | 支持中文、English 和日语 |
| 4 | 配置发送快捷键 | Ctrl+Enter 或 Enter |
| 5 | 设置一体机内网 IP | 确保网络连通性 |

### 3.2 智能体配置

在 aipy-enterprise.yml 文件中配置智能体参数，包括：

```yaml
agent:
  type: application
  keywords:
    - application
    - 数据安全
  workflow:
    max_rounds: 10
    timeout: 300
    auto_select: true
```

keywords 字段的第二个值为集市展示分类，可选范围包括：威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI 安全、安全服务、网络安全、其他、官方精选。

### 3.3 数据源连接

连接老系统数据源时，需遵循以下原则：

- 分析前读取前 30 行，了解数据格式
- 根据具体数据格式编写合适的分析脚本
- 报告保存到当前工作目录
- 确保数据库连接凭证安全存储

### 3.4 测试验证

部署完成后，需进行完整的功能测试：

1. 单条数据流转测试
2. 批量数据处理测试
3. 异常场景容错测试
4. 性能压力测试
5. 安全合规性检查

### 5.1 安全 considerations

数据集成过程中需重点关注：

- **权限控制**：严格执行最小权限原则，智能体仅访问必要数据
- **数据加密**：传输和存储过程均采用加密保护

## 相关问答 FAQs

**AiPy 智能体支持哪些类型的数据源连接？**

AiPy 智能体支持多种数据源类型，包括关系型数据库（MySQL、Oracle、SQL Server 等）、文件系统（CSV、Excel、JSON 等）、API 接口、消息队列等。具体支持的数据源类型取决于所选智能体的配置和能力，建议在 manifest.json 中明确声明所需的数据源类型，并在 keywords 字段中选择合适的分类标识。

**Workflow 执行失败时如何处理？**

Workflow 执行失败时，系统会自动记录错误日志并触发预设的异常处理机制。用户可在常规设置中配置最大执行轮数和超时时间，避免任务无限挂起。同时建议设置自动重试策略和告警通知，便于及时发现问题。对于频繁失败的节点，需检查数据源连接、权限配置和处理逻辑是否正确。

**AiPy 企业版的常规设置包含哪些可配置项？**

AiPy 企业版常规设置包括语言（支持中文、English 和日语三种语言）、风格（可在 aipy-enterprise.yml 文件中配置）、发送快捷键（Ctrl+Enter 或 Enter）、工作目录、最大执行轮数、超时时间、自动选择智能体、一体机内网 IP 等。这些设置可根据企业实际需求进行调整，以优化使用体验和系统性能。
