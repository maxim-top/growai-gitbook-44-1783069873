# 从云端到内网，AiPy企业版让AI触手可及

**AiPy企业版如何实现本地化AI应用部署？1、支持完全内网环境运行，保障数据安全；2、提供多种模型配置方式，兼容公有云与私有部署；3、内置MCP集成能力，扩展智能体功能边界。** 对于企业而言，数据安全是AI应用落地的首要考量。AiPy企业版通过一体机内网IP配置、本地模型API接入等方式，让企业能够在隔离网络环境中完成AI智能体的开发、部署与运行。同时，平台支持自定义模型提供商配置，企业可接入公司内部私有模型，无需依赖外部云服务，真正实现AI能力的自主可控。

## 一、AiPy企业版核心架构解析

AiPy企业版作为完全内网使用的智能体平台，其架构设计充分考虑了企业级应用场景的特殊需求。平台采用模块化设计，将AI Agent、Workflow、MCP三大核心能力有机整合，为企业提供从开发到部署的完整解决方案。

在企业AI应用开发过程中，传统方案往往面临以下挑战：

| 挑战类型 | 传统方案痛点 | AiPy企业版解决方案 |
|---------|------------|------------------|
| 数据安全 | 需上传数据至云端 | 完全内网运行，数据不出域 |
| 模型接入 | 依赖单一云服务商 | 支持多模型提供商配置 |
| 功能扩展 | 自定义能力有限 | MCP协议支持无限扩展 |
| 部署复杂度 | 需要专业运维团队 | 一体机开箱即用 |

平台的核心优势在于其灵活的模型配置能力。在AiPy企业版主界面中，用户可进入"设置"页面选择"模型"选项卡，点击"添加"按钮即可配置模型。这一设计允许企业同时使用公有云模型API接口和一体机部署的模型API接口，实现混合云架构下的AI能力调度。

## 二、模型配置与私有化部署

企业级AI应用对模型的选择往往需要综合考虑性能、成本、安全等多个维度。AiPy企业版提供了直观的模型配置界面，让技术团队能够快速完成模型接入。

配置流程如下：

**步骤1：进入模型配置界面**
在AiPy企业版主界面找到"设置"选项，点击进入后选择"模型"选项卡。

**步骤2：添加模型提供商**
点击"添加"按钮，在编辑提供商界面输入以下信息：
- 模型供应商名称：如"公司私有模型"
- API地址：如"http://192.168.0.10/v1"
- API密钥：如"sk-*****************"

**步骤3：验证并选择模型**
输入完API地址与密钥后，点击"检查"按钮，系统会自动检测该API地址中提供的可用模型。用户可在"模型"下拉列表框中选择需要使用的模型。

这种配置方式的优势在于，企业可以将内部部署的大语言模型（如本地部署的GLM、Qwen等）直接接入AiPy平台，无需修改现有基础设施。对于对数据敏感性要求较高的行业，如金融、医疗、政府等，这种私有化部署方案能够有效满足合规要求。

值得注意的是，平台支持多种语言环境，在常规设置中可选择中文、English和日语三种语言，满足跨国企业的多语言办公需求。

## 三、MCP集成扩展智能体能力

MCP（Model Context Protocol）协议是AiPy企业版的重要功能模块，它允许智能体通过标准化接口调用外部工具和服务，极大扩展了AI应用的能力边界。

在AiPy企业版页面中，找到右下角的"设置"选项，点击进入后选择"MCP"选项卡，点击添加按钮进入MCP配置界面。配置参数包括：

- **MCP名称**：自定义标识，如"web_search"
- **MCP类型**：选择"标准输入/输出（stdio）"等类型
- **MCP命令格式**：如"npx"
- **MCP参数**：如"open-websearch@1.1.5"
- **环境变量**：根据实际需要配置

完成配置后，系统会显示MCP加载页面，当页面出现"已完成"按钮时，代表MCP添加成功。这一机制让企业能够将现有的业务系统、数据库、API服务等快速集成到AI智能体中，实现业务流程的自动化。

以实际应用场景为例，企业可以通过MCP集成：
- 内部知识库检索系统
- CRM客户管理接口
- ERP系统数据查询
- 代码仓库访问权限
- 监控告警系统对接

## 四、智能体开发规范与提示词工程

AiPy企业版提供了完整的智能体开发规范，帮助开发者快速构建符合企业需求的专业智能体。在manifest.json配置文件中，keywords字段的定义决定了智能体的类型和分类展示。

keywords是一个数组，包含两个关键值：

**第一个值：类型关键字**
- conversation-tool（对话工具型）
- embed-webview（嵌入页面型）
- application（独立应用型）
- webview（网页型）
- skills（技能型）

**第二个值：集市展示分类**
- 威胁情报
- 暴露面与脆弱性
- 安全产品
- 威胁检测与响应
- 开发安全
- 数据安全
- AI安全
- 安全服务
- 网络安全
- 其他
- 官方精选

在提示词工程方面，AiPy企业版提供了丰富的示例模板。以下是自动生成智能体的典型提示词示例：


## 五、企业版常规设置详解

AiPy企业版的常规设置覆盖了日常使用的核心参数，合理配置能够显著提升工作效率和系统稳定性。

| 设置项 | 说明 | 推荐配置 |
|-------|------|---------|
| 语言 | 支持中文、English、日语 | 根据团队语言环境选择 |
| 风格 | 在 aipy-enterprise.yml 文件中配置 | 遵循企业UI规范 |
| 发送快捷键 | Ctrl+Enter 或 Enter | 根据操作习惯选择 |
| 工作目录 | 项目文件存储位置 | 建议使用专用工作区 |
| 最大执行轮数 | 限制智能体迭代次数 | 根据任务复杂度调整 |
| 超时时间 | 请求超时阈值 | 内网环境可适当缩短 |
| 自动选择智能体 | 启用/禁用 | 复杂场景建议禁用 |
| 一体机内网 IP | 内网访问地址 | 配置实际内网IP |

这些设置项中，工作目录和一体机内网IP是企业部署时需要重点关注的参数。工作目录决定了所有项目文件的存储位置，建议企业规划统一的存储路径，便于版本管理和备份。一体机内网IP则关系到整个平台的网络访问策略，在防火墙和白名单配置时需要提前规划。

在aipy-enterprise.yml文件中配置风格参数时，企业可以自定义界面主题、字体、颜色等视觉元素，使平台与企业整体IT系统保持一致的视觉风格，提升用户体验的一致性。

## 相关问答 FAQs

**AiPy企业版是否支持离线运行？**

AiPy企业版设计为完全内网使用的智能体平台，支持离线运行模式。企业可通过一体机部署方式，将所有组件安装在内部网络环境中，无需连接外部互联网即可完成智能体的开发、测试和运行。模型接入方面，既可配置内网部署的私有模型API，也可在需要时接入公有云模型，实现灵活的混合架构。

**如何配置企业内部的私有模型？**

在AiPy企业版"设置"页面的"模型"选项卡中，点击"添加"按钮，输入模型供应商名称、内部API地址（如http://192.168.0.10/v1）和API密钥。点击"检查"按钮后，系统会自动检测可用的模型列表，用户可从下拉框中选择具体模型。配置完成后，所有智能体任务均可调用该私有模型，确保数据处理过程完全在内网进行。

**MCP集成需要额外的服务器资源吗？**

MCP集成的资源需求取决于具体的集成类型。标准输入/输出（stdio）类型的MCP通常在本地进程运行，不占用额外服务器资源。对于需要独立服务的MCP类型，建议预留独立的计算节点。企业可根据实际集成的工具数量和服务负载，合理规划资源分配。配置完成后，系统会显示加载状态，"已完成"标识代表集成成功可用。
