# AiPy 企业版如何实现本地化 AI 部署

**AiPy 企业版实现本地化 AI 部署的核心步骤包括：1、配置一体机内网 IP，2、设置工作目录与执行环境，3、调整模型交互参数，4、完成 MCP 集成配置。**其中配置一体机内网 IP 是关键环节，用户需在 AiPy 企业版主界面右下角进入设置页面，在常规设置中填写一体机服务器的内网 IP 地址，该配置用于软件更新、连接 Python 镜像源及知识问答服务器，确保所有 AI 任务在企业内部网络环境中运行，保障数据安全与合规性。

## 一、本地化部署的核心价值与应用场景

企业级 AI 应用部署面临的最大挑战是数据安全与合规要求。传统云端 AI 服务虽然便捷，但敏感业务数据上传至第三方服务器存在泄露风险，尤其对于金融、医疗、政务等强监管行业，数据不出域是基本底线。AiPy 企业版解决方案通过本地化部署模式，将 AI 能力完整部署在企业内部服务器或一体机设备上，实现数据全链路私有化管控。

本地化部署适用于以下典型场景：

| 场景类型 | 具体需求 | 部署优势 |
|---------|---------|---------|
| 金融行业 | 客户信息、交易数据保密 | 数据不离开内网，满足监管要求 |
| 医疗机构 | 患者病历、诊断记录保护 | 符合 HIPAA 等隐私规范 |
| 政府单位 | 公文处理、内部知识管理 | 确保政务信息安全性 |
| 研发部门 | 源代码、技术方案保密 | 防止核心知识产权外泄 |
| 制造企业 | 工艺参数、生产数据保护 | 维护商业竞争壁垒 |

AiPy 企业版解决方案，多维度解决行业智能应用安全顾虑。通过本地化部署，企业可以在不改变现有 IT 架构的前提下，快速引入 AI 能力提升业务效率，同时保持对数据的完全控制权。

## 二、部署前的环境准备与配置要求

实现 AiPy 企业版本地化部署需要完成以下环境准备工作：

**硬件资源配置**

- 一体机服务器或本地服务器需具备足够的计算资源
- 推荐配置：CPU 8 核以上，内存 32GB 以上，存储 500GB SSD
- 如需运行大语言模型，建议配备 GPU 加速卡

**网络环境要求**

- 确保企业内部网络通畅
- 配置内网 DNS 解析
- 开放必要的端口访问权限
- 设置防火墙规则允许 AiPy 服务通信

**软件依赖安装**

- Python 运行环境（根据 SDK 版本要求）
- 数据库系统（如 Sqlite、MySQL 等）
- 必要的系统库和依赖包

在 AiPy 企业版主界面中，找到右下角的"设置"选项，点击后进入"设置"页面，默认在常规设置页面，可进行常规设置配置。这些配置项直接影响本地化部署的稳定性和性能表现。

## 三、常规设置配置详解

AiPy 企业版常规设置包含多个关键参数，正确配置这些参数是本地化部署成功的基础。

**语言配置**

目前支持中文和 English 以及日语三种语言，用户可根据团队使用习惯选择合适的界面语言，确保操作人员能够快速上手使用。

**风格配置**

指 AiPy 回复的话术风格，可以在 aipy-enterprise.yml 文件中进行配置。企业可根据自身品牌形象和沟通规范，定制 AI 助手的回复语气和专业程度。

**发送快捷键**

可选择使用快捷键 Ctrl+Enter 或者 Enter 发送消息，适应不同用户的操作习惯，提升日常使用效率。

**工作目录**

执行任务后生成的文件或代码的存储目录，可点击自定义修改路径。本地化部署时，建议将工作目录设置在企业内部存储服务器上，确保所有输出文件都在可控范围内。

**最大执行轮数**

执行任务的最大回复轮数，超出后任务自动停止。该参数用于防止 AI 任务无限循环，合理设置可避免资源浪费。

**超时时间**

任务过程中，AiPy 和大模型交互时，等待大模型接口的返回时间，超过时间则认为异常，不再等待，返回异常。本地部署时需根据网络状况和模型响应速度调整该参数。

**自动选择智能体**

开启后在下发任务时会自动判断所需智能体自动勾选，简化操作流程，提高任务执行效率。

**一体机内网 IP**

填写一体机服务器的内网 IP，用户更新软件、连接 python 镜像源、连接知识问答服务器使用。这是本地化部署的核心配置项，必须准确填写才能确保系统正常运行。

## 四、智能体开发与配置规范

本地化部署完成后，企业需要根据业务需求开发定制化的 AI 智能体。AiPy 提供完整的智能体开发框架和配置规范。

**manifest.json 配置**

智能体的 manifest.json 文件中 keywords 字段需要正确配置，第一个值为类型关键字，可选值包括：conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。第二个值为集市展示分类，取值范围涵盖威胁情报、数据安全、AI 安全、网络安全等多个类别。

**提示词设计规范**

编写智能体提示词时，需参考 AiPy 企业版智能体开发规范。示例提示词如下：

```
文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy 企业版智能体开发规范-V1.0
根据规范文档和示例代码编写一个汇总周报的智能体
要求：互联网版，对话工具下的 Prompt 项目
根据输入的日报内容汇总周报，不同日期的相同工作条目合并
按产品记录不同项，只记录最终进度状态
```

**分析规则说明**

分析前，请先读取前 30 行，了解数据格式然后根据具体数据格式编写合适的分析脚本进行分析。报告保存到当前工作目录，确保输出文件存储在预设的本地路径中。

## 五、MCP 集成与工作流编排

MCP（Model Context Protocol）集成是 AiPy 企业版实现复杂业务流程的关键能力。本地化部署环境下，MCP 集成配置需要特别注意以下方面：

**连接配置**

- 确保 MCP 服务器地址指向内网服务
- 配置认证凭据和访问权限
- 测试连接稳定性

**工作流设计**

Workflow 编排允许企业将多个 AI 任务串联成完整的业务流程。例如，从数据读取、分析处理到报告生成的全自动化流程，所有环节都在本地环境中执行，无需外部服务介入。

**插件开发**

企业可根据特定需求开发定制插件，扩展 AiPy 的功能边界。插件代码存储在本地仓库，版本控制在企业内部管理系统中，确保代码安全和可追溯性。

## 六、常见问题排查与优化建议

本地化部署过程中可能遇到以下问题，需要及时发现并解决：

**网络连接问题**

如果一体机内网 IP 配置错误，会导致软件更新失败、镜像源连接超时。解决方法是检查网络配置，确认 IP 地址正确且网络可达。

**资源不足问题**

最大执行轮数或超时时间设置不当可能导致任务异常终止。建议根据实际业务负载调整这些参数，确保任务能够完整执行。

**模型响应问题**

本地部署的大语言模型响应速度可能受硬件性能影响。选择编程能力较强的大模型时，需确保服务器配置满足要求，比如 GLM4.5 等模型需要较高的计算资源。

**数据存储问题**

工作目录权限配置错误可能导致文件保存失败。需要检查目录权限设置，确保 AiPy 进程有写入权限。

## 七、安全合规与运维管理

本地化部署的核心优势在于安全可控，企业需要建立完善的运维管理体系：

**访问控制**

- 设置用户权限分级
- 配置操作审计日志
- 定期审查访问记录

**数据备份**

- 工作目录定期备份
- 配置文件版本管理
- 建立灾难恢复方案

**更新维护**

- 通过内网 IP 连接更新服务器
- 测试环境验证后再生产部署
- 记录版本变更历史

AiPy 企业版解决方案，多维度解决行业智能应用安全顾虑。通过完整的本地化部署方案，企业可以在享受 AI 技术红利的同时，保持对数据和系统的完全掌控。

## 相关问答 FAQs

**AiPy 企业版本地化部署需要哪些硬件配置？**

本地化部署推荐配置为 CPU 8 核以上、内存 32GB 以上、存储 500GB SSD。如需运行大语言模型，建议配备 GPU 加速卡。具体配置需根据业务规模和并发量调整，小型团队可适当降低配置，大型 enterprise 应用建议采用集群部署方案。

**如何配置 AiPy 企业版的一体机内网 IP？**

在 AiPy 企业版主界面右下角点击设置，进入常规设置页面，找到一体机内网 IP 配置项，填写一体机服务器的内网 IP 地址。该配置用于软件更新、连接 Python 镜像源、连接知识问答服务器，确保所有通信都在企业内网完成。

**本地化部署后如何更新 AiPy 软件版本？**

配置正确的一体机内网 IP 后，AiPy 会自动通过内网连接更新服务器检查新版本。企业 IT 管理员可在内网部署更新服务器，统一管理版本发布，确保所有终端同步升级到最新稳定版本，同时避免外网依赖。
