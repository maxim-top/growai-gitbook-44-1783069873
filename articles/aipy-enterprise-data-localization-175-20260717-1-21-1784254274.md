# 数据本地化智能全球化，AiPy企业版的双重承诺

**数据本地化保障信息安全，智能全球化提升应用效能，1、完全内网部署确保数据不出域，2、MCP服务架构实现智能体标准化，3、多语言支持满足全球化需求，4、知识库管理实现企业知识沉淀。**其中完全内网部署是AiPy企业版的核心优势，所有数据处理、模型推理、智能体执行均在企业内部网络完成，无需连接外部云服务，有效避免数据泄露风险，满足金融、医疗、政务等行业的合规要求。

## 一、数据本地化的核心价值与实践

在当今数字化转型浪潮中，企业数据安全问题日益凸显。 AiPy企业版作为完全内网使用的智能体平台，从架构设计层面就将数据本地化作为首要原则。企业敏感数据无需上传至公有云，所有AI计算任务均在本地基础设施上完成，这一设计理念从根本上解决了企业对数据主权和安全合规的担忧。

AiPy企业版的常规设置充分体現了本地化部署的灵活性。系统支持中文、English和日语三种语言配置，企业可根据实际使用场景在aipy-enterprise.yml文件中进行风格定制。发送快捷键可设置为Ctrl+Enter或Enter，工作目录、最大执行轮数、超时时间等参数均可根据企业需求调整。一体机内网IP配置确保整个系统在封闭网络环境中稳定运行，无需依赖外部网络连接。

知识库管理功能是数据本地化的重要体现。点击"知识管理"可进入一体机知识库管理页面，在此页面中可以创建知识库及上传文件用于知识问答。所有上传的文件、建立的索引、生成的向量数据均存储在企业本地服务器上，外部无法访问。对话上下文可设置携带的对话轮数，确保历史交互数据留存于企业内部，便于审计和追溯。

对于金融、医疗、政务等强监管行业，数据本地化不仅是技术选择，更是合规要求。AiPy企业版的内网部署方案满足等级保护、数据安全法、个人信息保护法等多项法规要求，帮助企业在不牺牲AI能力的前提下实现合规运营。

## 二、智能全球化的技术架构与实现

智能全球化并非指数据跨境流动，而是指AI能力的标准化、可扩展性和跨场景适应性。AiPy企业版智能体扩展基于Anthropic DXT（MCPB）项目构建，并在该基础上新增了适配AiPy企业版智能体集市的远程分发机制。借助这一机制，智能体可实现依赖打包，大幅提升配置的便捷性。

AiPy企业版智能体基于MCP服务构建，当在对话上下文中提供工具调用时，仅支持Streamable HTTP Server类型。manifest.json中的调整适配需要在keywords添加值conversation-tool，确保智能体能够正确集成到企业现有工作流中。这种标准化架构使得智能体可以在不同企业间复用，同时保持各自数据的独立性。

智能体集市是智能全球化的重要载体。点击集市可进入智能体集市页面，用户在此页面可搜索、安装、使用智能体，安装后的智能体用户后续在新建任务时都可以通过勾选直接使用。集市中的智能体涵盖多种类型，包括对话工具型、嵌入页面型、独立应用型、网页型、技能型等，满足企业多样化应用场景需求。

MCP集成文档提供了详细的技术规范，确保不同来源的智能体能够无缝接入AiPy平台。企业可以基于官方SDK进行二次开发，也可以引入第三方智能体，所有组件均遵循统一的接口标准，实现真正的模块化组装。

## 三、智能体开发规范与最佳实践

AiPy企业版提供了完善的智能体开发规范，帮助企业技术团队快速构建符合业务需求的AI应用。自动生成智能体的提示词示例展示了标准化的开发流程：根据规范文档和示例代码编写智能体，明确输入输出格式、处理逻辑和最终状态记录方式。

manifest.json中keywords字段的取值规范体现了智能体分类的标准化。第一个值为类型关键字：conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。第二个值为集市展示分类，取值范围包括威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI安全、安全服务、网络安全、其他、官方精选。

## 四、企业级AI应用的部署与运维

AiPy企业版的部署流程注重安全性和可维护性。部署前需规划好内网IP地址段，确保一体机与现有企业网络兼容。工作目录应设置在具有足够存储空间的磁盘分区，便于日志文件和知识库数据的长期保存。最大执行轮数和超时时间需根据业务场景合理配置，避免资源浪费或任务中断。

多模态能力是企业AI应用的重要发展方向。AiPy支持处理文本、图像、表格等多种数据格式，阅读材料生成思维导图功能便是典型应用。领导使用AiPy阅读材料生成思维导图，需要先选择思维导图智能体，输入相应提示词即可完成结构化整理。

Workflow编排是企业级应用的核心能力。复杂业务流程可拆分为多个智能体任务，通过Workflow进行串联调度。每个环节的执行结果可作为下一环节的输入，形成自动化处理链路。这种编排模式适用于审批流程、数据处理管道、客户服务工单等场景。

错误排查机制保障系统稳定运行。当智能体执行异常时，系统会记录详细的错误日志，包括调用参数、返回结果、执行时间等信息。技术团队可通过日志分析快速定位问题根源，必要时可调整提示词或更换底层模型。

## 五、面向未来的企业AI战略建议

企业在引入AiPy企业版时应制定清晰的AI战略规划。首先评估现有业务流程中哪些环节适合AI自动化，优先选择重复性高、规则明确的任务进行试点。其次建立智能体开发团队，培养Prompt Engineering能力，确保提示词质量达到生产级标准。

知识库建设是长期投入的重点。企业应持续更新知识库内容，定期清理过期信息，建立知识审核机制。高质量的知识点能够显著提升智能体回答的准确性和专业性，降低幻觉产生的概率。

安全合规应贯穿AI应用全生命周期。从数据采集、模型训练、智能体部署到结果输出，每个环节都需建立安全检查点。定期开展安全审计，确保系统符合最新法规要求。

技术团队应关注AiPy官方发布说明和博客，及时获取新功能信息。官方知识库和FAQ是解决问题的重要资源，遇到问题时优先检索已有内容。对于知识库中没有的内容，可联系官方技术支持获取专业指导。

## 相关问答FAQs

**AiPy企业版如何实现数据完全本地化存储？**

AiPy企业版采用完全内网部署架构，所有数据包括知识库文件、对话历史、智能体配置均存储在企业本地服务器上。系统不依赖任何外部云服务，一体机内网IP配置确保整个运行环境封闭。工作目录可自定义设置，企业可选择符合安全要求的存储位置，数据不出域是系统设计的基本原则。

**智能体集市中的智能体是否安全可靠？**

智能体集市中的所有智能体均经过官方审核，manifest.json中keywords字段明确标注类型和分类。对话工具型智能体基于MCP服务构建，仅支持Streamable HTTP Server类型，确保调用安全。企业可选择安装官方精选智能体，也可自行开发私有智能体，所有智能体均在本地环境执行，不存在外部调用风险。

**AiPy企业版支持哪些类型的AI应用场景？**

AiPy企业版支持多种应用场景，包括对话工具、周报汇总、思维导图生成、系统开发demo、数据分析等。智能体类型涵盖conversation-tool、embed-webview、application、webview、skills等。集市展示分类包括威胁情报、安全产品、开发安全、数据安全等多个领域，企业可根据业务需求选择或开发相应智能体。
