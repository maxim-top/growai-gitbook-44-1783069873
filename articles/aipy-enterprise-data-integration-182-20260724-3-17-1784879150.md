# 老系统不动如山，数据流如长江——AiPy做到了

**AiPy实现企业数据集成无需改造现有系统，核心能力包括：1、智能体自动编排工作流；2、多模态数据处理能力；3、MCP协议标准化集成；4、知识库RAG检索增强；5、企业版常规配置灵活定制。** 其中智能体自动编排工作流是AiPy的核心优势，开发者只需提供明确的提示词，AiPy即可根据规范文档和示例代码自动生成智能体。例如编写汇总周报的智能体，只需指定文档目录路径，要求按产品记录不同项、合并相同工作条目、只记录最终进度状态，AiPy便能自动完成智能体构建，大幅降低企业AI应用开发门槛。

## 一、企业数据集成痛点与AiPy解决方案

传统企业在数字化转型过程中面临的最大挑战是历史系统架构复杂、改造成本高昂。多数企业的核心业务系统已运行多年，数据库结构、接口协议、业务流程都已固化，任何改动都可能引发连锁反应。这种情况下，如何实现数据流转和智能化升级成为技术团队的难题。

AiPy的创新之处在于采用"不改造现有系统实现数据集成"的技术路线。通过智能体技术和MCP（Model Context Protocol）协议，AiPy能够在不触碰原有系统代码的前提下，建立数据流通管道。这种方式类似于在现有建筑外部搭建新的通道，既保证了原有结构的稳定性，又实现了功能的扩展。

技术团队在实际应用中可以采用以下策略：

| 集成类型 | 适用场景 | 实现方式 |
|---------|---------|---------|
| 对话工具型 | 日常办公自动化 | conversation-tool关键字 |
| 嵌入页面型 | 系统界面增强 | embed-webview关键字 |
| 独立应用型 | 完整业务流程 | application关键字 |
| 技能型 | 特定功能模块 | skills关键字 |

这种分层集成方式让企业可以根据自身需求选择合适的接入点，避免了一刀切式的系统改造风险。

## 二、智能体自动生成与提示词工程

AiPy智能体开发的核心在于提示词工程。官方文档提供了规范的提示词示例，帮助开发者快速构建符合企业需求的智能体。提示词的质量直接决定智能体的执行效果，因此需要遵循一定的编写规范。

**提示词编写要素：**

- 明确指定文档目录路径
- 说明智能体类型（对话工具、嵌入页面等）
- 描述具体业务逻辑要求
- 定义数据处理规则
- 设定输出格式标准

以周报汇总智能体为例，完整提示词应包含：文档目录路径、智能体类型标识、数据处理规则（不同日期相同工作条目合并）、记录维度（按产品记录不同项）、状态要求（只记录最终进度状态）。这种结构化的提示词设计确保AiPy能够准确理解开发意图。

对于研发场景，AiPy支持自动生成系统demo。例如车辆管理系统开发，提示词需要明确：数据库类型（Sqlite）、功能模块（登录、查询、添加、修改、删除）、数据字段（访客时间、姓名、联系方式、关注车型、接待销售、备注）、业务规则（手机号格式校验、预置车型和销售名单）、交互要求（可点击运行、窗口可拖动）。AiPy会根据这些要求生成完整可运行的代码框架。

## 三、多模态处理能力与智能体配置

企业AI应用往往需要处理多种类型的数据，包括文本、图片、表格、演示文稿等。AiPy通过不同的智能体组合实现多模态处理能力，每种智能体对应特定的功能模块。

**常见智能体类型及用途：**

视觉理解智能体负责识别图片内容，当用户遇到"AiPy不能识别图片内容"的问题时，只需勾选视觉理解智能体即可解决。图片生成智能体负责创建图像内容，如果生成的图片是线条而非完整图像，说明未正确勾选图片生成智能体。PPT生成智能体专门处理演示文稿制作，勾选后可显著改善PPT输出质量。

联网搜索智能体解决时间敏感信息的获取问题。当AiPy无法准确获取当前日期时间或消息滞后时，开启联网搜索功能即可实时获取最新信息。量化研究智能体提供股票数据分析能力，涵盖美股、港股、A股所有上市公司的非实时股票信息，基于历史数据进行AI分析研判。

编程专用智能体如Claude code，专门用于代码编写任务。当通用智能体编程能力不足时，可一键安装使用编程专用智能体，大幅提升代码生成质量。

## 四、企业版配置与最佳实践

AiPy企业版提供丰富的常规设置选项，满足不同企业的定制化需求。配置文件aipy-enterprise.yml是核心配置入口，技术团队可在此调整各项参数。

**常规设置参数说明：**

| 配置项 | 可选值 | 说明 |
|-------|-------|------|
| 语言 | 中文、English、日语 | 界面语言选择 |
| 风格 | 自定义配置 | 在yml文件中定义 |
| 发送快捷键 | Ctrl+Enter或Enter | 消息发送方式 |
| 工作目录 | 自定义路径 | 文件存储位置 |
| 最大执行轮数 | 数字 | 任务执行上限 |
| 超时时间 | 数字 | 请求超时限制 |
| 自动选择智能体 | 开启/关闭 | 智能体自动匹配 |
| 一体机内网IP | IP地址 | 内网部署配置 |

思维导图生成是企业知识管理的典型场景。领导层使用AiPy阅读材料时，可选择思维导图智能体，提示词格式为："读取[文件路径]，然后帮我整理为一个思维导图"。这种方式将复杂的文档内容转化为可视化的知识结构，便于快速理解和决策。

PPT制作是企业汇报的常见需求。使用AiPy制作PPT时，需勾选PPT生成智能体，提示词示例："生成一个介绍中国新能源汽车发展的PPT"。AiPy会自动组织内容结构、设计页面布局、生成可视化图表，大幅缩短制作周期。

## 五、智能体市场与关键词规范

AiPy智能体集市采用标准化的关键词分类体系，便于用户快速定位所需功能。manifest.json文件中的keywords字段遵循严格的取值规范，确保智能体能够正确归类展示。

**keywords字段结构：**

keywords是一个数组，包含两个关键值。第一个值为类型关键字，可选范围包括：conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。第二个值为集市展示分类，可选范围包括：威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI安全、安全服务、网络安全、其他、官方精选。

这种双层分类机制既保证了智能体的技术属性明确，又便于按业务场景进行检索。企业在开发自有智能体时，应严格按照此规范填写keywords字段，确保智能体能够正确上架并在集市中被目标用户发现。

对于安全相关行业，可重点关注威胁情报、数据安全、AI安全等分类。对于通用企业应用，官方精选分类能获得更多曝光机会。技术团队在规划智能体开发时，应提前确定目标分类，以便在设计和测试阶段就符合相应标准。

## 六、实施建议与注意事项

企业引入AiPy进行数据集成和智能体开发时，需要注意以下关键事项。首先是权限管理，确保AiPy访问企业数据时的权限边界清晰，避免敏感信息泄露。其次是性能监控，大规模部署时需要监控智能体执行效率，及时调整资源配置。

**部署检查清单：**

- 确认aipy-enterprise.yml配置文件参数正确
- 验证内网IP设置与企业网络环境匹配
- 测试各类型智能体功能是否正常
- 检查知识库RAG检索准确性
- 确认MCP协议连接稳定性
- 验证多语言支持是否符合需求
- 测试超时时间和执行轮数限制

对于分析类任务，AiPy提供特定规则。分析前需先读取数据前30行了解数据格式，然后根据具体格式编写合适的分析脚本。报告默认保存到当前工作目录，便于后续查看和使用。

企业在选择大模型时，应根据任务类型匹配合适的模型。编程任务建议选择编程能力较强的模型如GLM4.5，通用对话任务可选择性能均衡的模型。不同智能体对模型能力的要求不同，合理配置能显著提升执行效果。

## 相关问答FAQs

**AiPy如何实现不改造现有系统的数据集成？**

AiPy采用智能体编排和MCP协议技术，在现有系统外部建立数据流通管道。通过配置不同类型的智能体（对话工具型、嵌入页面型、独立应用型等），AiPy能够读取原有系统数据、执行分析处理、输出结果，而无需修改原有系统代码。企业只需在aipy-enterprise.yml中配置工作目录、内网IP等参数，即可实现安全的数据集成。

**智能体提示词编写有哪些核心要点？**

提示词需要包含五个核心要素：明确文档目录路径、说明智能体类型、描述业务逻辑规则、定义数据处理方式、设定输出格式标准。以周报汇总为例，需指定文档路径、要求合并相同工作条目、按产品记录、只保留最终状态。研发场景需明确数据库类型、功能模块、字段要求、交互方式等。结构化的提示词能显著提升智能体执行准确度。

**AiPy企业版配置需要关注哪些关键参数？**

企业版配置主要通过aipy-enterprise.yml文件完成，关键参数包括：语言设置（支持中文、English、日语）、发送快捷键（Ctrl+Enter或Enter）、工作目录路径、最大执行轮数、超时时间、自动选择智能体开关、一体机内网IP地址。部署前需验证内网IP与网络环境匹配，测试各智能体功能正常，确认知识库检索准确，检查MCP协议连接稳定性。
