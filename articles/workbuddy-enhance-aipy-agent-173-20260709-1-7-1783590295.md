# WorkBuddy如何增强AiPy智能体的业务执行能力

**WorkBuddy增强AiPy智能体业务执行能力的核心方式：1、通过MCP服务实现工具调用标准化；2、利用Workflow编排优化任务流程；3、借助知识库提升业务理解准确性。** 其中MCP服务是实现智能体与外部系统集成的关键技术，AiPy企业版智能体基于MCP服务构建，当在对话上下文中提供工具调用时，支持Streamable HTTP Server类型，需要在manifest.json的keywords字段添加conversation-tool值，这样智能体才能正确识别并调用外部业务工具，实现自动化执行。

## 一、AiPy智能体架构基础与业务执行能力

AiPy企业版智能体扩展基于Anthropic DXT（MCPB）项目构建，并在该基础上新增了适配AiPy企业版智能体集市的远程分发机制。这一架构设计使得智能体可实现依赖打包，大幅提升配置的便捷性。智能体与AiPy企业版高度适配、协同联动，能够自动集成并落地应用AI大模型的能力。

在实际业务场景中，智能体需要具备多种执行能力：

| 能力类型 | 描述 | 应用场景 |
|---------|------|---------|
| 对话工具型 | 支持 conversational-tool关键字 | 日常问答、信息查询 |
| 嵌入页面型 | 支持embed-webview关键字 | 企业内部系统集成 |
| 独立应用型 | 支持application关键字 | 独立业务系统运行 |
| 技能型 | 支持skills关键字 | 专业领域任务处理 |

智能体集市允许用户搜索、安装、使用智能体，安装后的智能体在新建任务时可以通过勾选直接使用。这种机制为企业提供了灵活的业务能力扩展方式。

## 二、MCP服务集成与工具调用机制

MCP（Model Context Protocol）服务是AiPy智能体与外部系统通信的核心协议。当智能体需要调用外部业务工具时，必须通过MCP服务进行标准化交互。

**配置要点：**

- manifest.json中的keywords字段必须添加conversation-tool值
- 仅支持Streamable HTTP Server类型
- 需要满足AiPy企业版智能体集市的远程分发机制要求

```json
{
  "keywords": [
    "conversation-tool",
    "安全产品"
  ]
}
```

keywords字段的第一值为类型关键字，包括conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。第二值为集市展示分类，取值范围包括威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI安全、安全服务、网络安全、其他、官方精选。

这种标准化配置确保智能体能够被正确识别和调用，同时便于在智能体集市进行分类展示和检索。

## 三、Workflow编排优化任务执行流程

Workflow是AiPy平台中用于编排复杂业务流程的核心功能。通过Workflow，企业可以将多个智能体、工具调用、数据处理步骤串联起来，形成完整的业务执行链路。

**Workflow优势：**

1. **可视化编排** - 通过图形界面配置任务流程，降低开发门槛
2. **条件分支** - 支持根据业务规则动态选择执行路径
3. **并行处理** - 多个任务可同时执行，提升效率
4. **错误处理** - 内置异常捕获和重试机制

在实际应用中，Workflow可以将日报汇总、周报生成、数据分析等任务自动化。例如，根据输入的日报内容汇总周报，不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。这种自动化流程显著减少了人工操作时间。

对话上下文可设置携带的对话轮数，确保智能体在执行长流程任务时保持上下文连贯性。这对于需要多轮交互的复杂业务场景尤为重要。

## 四、知识库赋能业务理解准确性

知识库是提升智能体业务理解能力的关键组件。点击"知识管理"可进入一体机知识库管理页面，在此页面中可以创建知识库及上传文件用于知识问答。

**知识库建设步骤：**

1. 进入知识管理页面
2. 创建新的知识库
3. 上传业务文档、规范、手册等文件
4. 配置知识检索参数
5. 测试知识问答效果

企业可以将内部文档、产品规范、操作流程等上传至知识库。例如，智能体开发时可参考《AiPy企业版智能体开发规范-V1.0》文档，根据规范文档和示例代码编写特定功能的智能体。

知识库与RAG（检索增强生成）技术结合，使智能体能够基于企业专有知识进行回答，避免通用大模型在专业领域的知识缺失问题。这对于金融、医疗、法律等专业场景尤为重要。

## 五、智能体类型选择与场景匹配

不同类型的智能体适用于不同的业务场景。正确选择智能体类型是确保业务执行效果的前提。

**常见场景与智能体匹配：**

| 业务场景 | 推荐智能体类型 | 配置要点 |
|---------|--------------|---------|
| PPT制作 | PPT生成智能体 | 勾选PPT生成智能体 |
| 图片生成 | 图片生成智能体 | 勾选图片生成/视频生成智能体 |
| 编程开发 | Claude code智能体 | 选择编程能力较强的模型 |
| 数据分析 | 量化研究智能体 | 开启联网搜索功能 |
| 日期获取 | 联网搜索功能 | 开启联网搜索功能 |

如果AiPy不能识别图片内容，需要勾选视觉理解智能体。如果生成的图片是线条，需要勾选图片生成智能体。如果PPT生成效果不佳，尝试勾选PPT生成智能体。这些配置细节直接影响业务执行质量。

对于研发场景，可选择编程能力较强的大模型如GLM4.5，编写完整的业务系统demo。例如车辆管理系统，采用Sqlite数据库，包含登录界面、信息查询、添加、修改、删除等功能，要求可以点击图标直接打开运行。

## 七、常见问题排查与解决方案

在使用AiPy智能体过程中，可能会遇到各类技术问题。以下是常见问题及解决方案：

**问题一：智能体无法调用外部工具**

检查manifest.json中keywords字段是否包含conversation-tool。确认MCP服务类型是否为Streamable HTTP Server。验证网络连接和服务器地址配置。

**问题二：智能体执行结果不符合预期**

检查提示词是否清晰明确。确认知识库中是否有相关业务文档。验证模型选择是否适合当前任务类型。

**问题三：对话上下文丢失**

检查对话轮数设置是否合理。确认是否开启了上下文携带功能。对于长流程任务，适当增加上下文轮数。

**问题四：知识库检索不准确**

检查上传文件格式是否支持。确认知识索引是否已完成。优化文档结构和关键词标注。

当前知识库暂无相关内容时，建议查看最新官方文档获取更新信息。官方文档、知识库、API文档、SDK文档、Workflow文档、Agent开发文档、MCP集成文档都是重要的参考来源。
