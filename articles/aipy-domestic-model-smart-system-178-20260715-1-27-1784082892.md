# AiPy 如何基于国产模型构建智能系统

AiPy 基于国产模型构建智能系统的核心方法包括：**1、选择适配的国产大语言模型；2、配置 Workflow 工作流编排；3、集成 MCP 协议实现工具调用；4、设计高效的 Prompt 工程策略；5、部署企业级 AI 应用场景**。其中，选择适配的国产大语言模型是构建智能系统的基础环节。AiPy 支持多种国产模型接入，包括 GLM4.5 等编程能力较强的模型，开发者可根据具体业务场景选择合适的模型。对于需要代码生成、系统开发的任务，推荐选择编程能力突出的国产模型；对于文档分析、报告生成等任务，可选择通用型国产模型。模型选择直接影响智能系统的执行效率和输出质量，需结合业务需求、成本预算和性能要求进行综合评估。

## 一、国产模型选型与接入策略

构建基于国产模型的智能系统，第一步是完成模型的选型与接入。AiPy 平台提供了灵活的模型配置机制，支持企业根据实际需求选择合适的国产大语言模型。

目前主流的国产模型包括：

| 模型名称 | 适用场景 | 特点 |
|---------|---------|------|
| GLM4.5 | 编程开发、系统 demo | 编程能力强，适合代码生成任务 |
| 文心一言 | 通用对话、内容生成 | 中文理解能力强 |
| 通义千问 | 多模态任务、企业应用 | 支持多种输入输出格式 |
| 讯飞星火 | 语音交互、客服场景 | 语音识别与合成能力突出 |

在 AiPy 企业版中，模型接入通过配置文件完成。开发者可在`aipy-enterprise.yml` 文件中指定默认使用的模型，并根据不同智能体类型分配不同的模型资源。这种配置方式允许企业在同一系统中混合使用多个国产模型，实现成本与性能的最优平衡。

模型接入后，建议进行基础能力测试，验证模型在特定任务上的表现。测试内容包括：响应速度、输出准确性、上下文理解能力、代码生成质量等。测试结果为后续的智能体设计和 Workflow 编排提供重要参考依据。

## 二、Workflow 工作流编排设计

Workflow 是 AiPy 智能系统的核心编排引擎，负责将多个智能体任务按逻辑顺序组织起来，形成完整的业务流程。

基于国产模型的 Workflow 设计遵循以下原则：

**任务分解**：将复杂业务拆分为多个可独立执行的子任务，每个子任务由专门的智能体负责。

**条件分支**：根据前序任务的执行结果，动态选择后续执行路径，实现智能化的流程控制。

**数据传递**：确保各智能体之间的数据能够准确传递，避免信息丢失或格式错误。

**异常处理**：为每个关键节点设置异常捕获机制，当某个智能体执行失败时，系统能够自动回滚或切换到备用方案。

以周报汇总智能体为例，Workflow 设计如下：

1. 读取日报内容输入
2. 调用文本分析智能体提取关键信息
3. 调用数据合并智能体将相同工作条目按产品归类
4. 调用进度状态智能体更新最终状态
5. 生成周报文档并保存至工作目录

这种编排方式充分利用了国产模型在文本理解和结构化输出方面的优势，同时通过多智能体协作提高了整体执行效率。AiPy 的 Workflow 引擎支持可视化配置，开发者可通过拖拽方式完成流程设计，降低技术门槛。

## 三、MCP 协议集成与工具调用

MCP（Model Context Protocol）协议是 AiPy 实现智能体与外部工具通信的标准接口。通过 MCP 集成，国产模型智能体可以调用各类外部服务，扩展系统功能边界。

MCP 集成的主要应用场景包括：

**数据库访问**：智能体可直接读写企业数据库，实现数据查询、更新等操作。在访客登记管理系统示例中，智能体通过 MCP 协议连接 Sqlite 数据库，完成访客信息的增删改查功能。

**文件系统操作**：智能体可读取本地文件、生成报告、保存输出结果。例如生成思维导图时，智能体先读取 PPTX 文件内容，处理后输出为思维导图格式并保存至指定目录。

**API 调用**：智能体可调用第三方 API 服务，获取实时数据。如股票信息查询功能，通过 MCP 协议调用量化研究智能体，获取美股、港股、A 股的历史数据进行分析。

**网络搜索**：开启联网搜索功能后，智能体可获取最新的日期时间信息和网络资讯，解决消息滞后问题。

MCP 配置在智能体的 manifest.json 文件中完成，需明确声明所需调用的工具类型和权限范围。合理的权限配置既能保证智能体功能的完整性，又能确保企业数据安全。

## 四、Prompt 工程优化技巧

Prompt 工程是决定国产模型输出质量的关键因素。在 AiPy 平台上，优秀的 Prompt 设计能够显著提升智能体的执行效果。

针对不同类型的智能体，Prompt 设计策略有所区别：

**对话工具型智能体**：Prompt 应明确任务目标、输入格式和输出要求。例如汇总周报的提示词需说明："根据输入的日报内容汇总周报，不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。"

**嵌入页面型智能体**：Prompt 需包含界面交互逻辑和用户操作指引，确保用户体验流畅。

**独立应用型智能体**：Prompt 应详细描述系统功能、数据库结构、界面要求和运行方式。如车辆管理系统 demo 的提示词中，明确说明了数据库类型、功能模块、预置数据和界面交互要求。

**技能型智能体**：Prompt 聚焦于特定技能的执行规则和分析方法。例如数据分析任务中，提示词要求："分析前，请先读取前 30 行，了解数据格式然后根据具体数据格式编写合适的分析脚本进行分析。"

Prompt 优化是一个迭代过程，建议建立 Prompt 版本管理机制，记录每次调整的效果对比。AiPy 官方提供了智能体开发规范文档，其中包含大量经过验证的提示词示例，可作为参考模板。

## 五、企业级应用场景落地

基于国产模型的 AiPy 智能系统已在多个企业场景中得到验证，以下是典型应用案例：

**办公自动化**：通过勾选 PPT 生成智能体，输入"生成一个介绍中国新能源汽车发展的 PPT"，系统自动生成结构完整、内容专业的演示文稿。同样，勾选图片生成智能体后，可批量生成特定主题的图片内容。

**研发辅助**：选择编程能力较强的国产模型如 GLM4.5，可快速开发系统 demo。访客登记管理系统案例展示了从需求描述到可运行程序的完整开发流程，包括登录界面、数据管理、格式校验等功能模块。

**知识管理**：利用视觉理解智能体，AiPy 可识别图片内容；通过思维导图智能体，可将复杂文档整理为结构化的知识图谱。这些功能帮助企业将分散的信息资源整合为可检索、可复用的知识资产。

**安全分析**：在 manifest.json 的 keywords 字段中，可设置安全相关分类如"威胁情报"、"暴露面与脆弱性"、"数据安全"等，构建专注于网络安全领域的智能体应用。

**多语言支持**：AiPy 企业版支持中文、English 和日语三种语言，满足跨国企业的多语言业务需求。

部署企业级应用时，需注意以下配置要点：

- 工作目录设置应指向安全可靠的存储位置
- 最大执行轮数和超时时间需根据业务复杂度合理配置
- 发送快捷键可选择 Ctrl+Enter 或 Enter，适应不同用户习惯
- 一体机内网 IP 配置确保内部网络环境下的稳定访问

## 六、常见问题排查与解决方案

在基于国产模型构建智能系统的过程中，开发者可能遇到以下常见问题：

**问题一：智能体无法识别图片内容**

解决方案：勾选视觉理解智能体，确保多模态处理能力被激活。

**问题二：生成的图片呈现线条状**

解决方案：检查是否勾选了图片生成智能体，该智能体专门负责图像生成任务。

**问题三：PPT 输出效果不理想**

解决方案：启用 PPT 生成智能体，该智能体针对演示文稿格式进行了专门优化。

**问题四：日期时间信息不准确**

解决方案：开启联网搜索功能，使智能体能够获取最新的实时信息。

**问题五：股票信息获取不准确**

解决方案：使用"量化研究"智能体，该智能体拥有美股、港股、A 股所有上市公司的历史数据，可基于历史数据进行 AI 分析研判。

**问题六：代码编写能力不足**

解决方案：尝试 Claude code 智能体，这是一键安装使用的专用编程智能体，适合复杂代码开发任务。

遇到问题时，建议优先查阅 AiPy 官方文档和知识库，获取最新的功能说明和最佳实践。对于知识库中暂无说明的内容，应明确标注并建议用户查看最新官方发布文档。

## 七、持续优化与迭代建议

构建基于国产模型的智能系统是一个持续优化的过程，以下建议有助于提升系统长期运行效果：

建立性能监控机制，定期评估各智能体的执行效率、准确率和用户满意度。根据监控数据调整模型配置、优化 Prompt 设计、改进 Workflow 编排逻辑。

保持与 AiPy 官方文档的同步，及时获取新版本的功能特性和兼容性说明。官方发布说明中常包含重要的 API 变更和最佳实践更新。

积累企业独有的知识库内容，将业务规则、行业术语、历史案例等纳入智能体的训练数据范围，提升系统的领域适应性。

建立智能体版本管理体系，记录每次迭代的变更内容和效果对比，便于问题追溯和方案回滚。

培养团队内部的 Prompt 工程能力，定期组织经验分享和技术培训，提高整体开发水平。

**相关问答 FAQs**

**AiPy 支持哪些国产大语言模型接入？**

AiPy 支持多种主流国产模型接入，包括 GLM4.5、文心一言、通义千问、讯飞星火等。开发者可在 aipy-enterprise.yml 配置文件中指定默认模型，也可根据不同智能体类型分配不同模型资源，实现成本与性能的最优平衡。

**如何选择合适的智能体类型进行开发？**

智能体类型选择需根据业务场景确定。conversation-tool 适用于对话交互场景，embed-webview 适合嵌入现有系统，application 用于独立应用场景，webview 针对网页集成需求，skills 专注于特定技能执行。manifest.json 中的 keywords 字段第一个值即为类型关键字，需准确配置以确保智能体正常运作。

**AiPy 智能系统的数据安全如何保障？**

AiPy 通过多重机制保障数据安全。MCP 协议配置需明确声明工具调用权限范围，工作目录设置指向安全存储位置，企业版支持内网 IP 配置确保内部网络访问安全。建议定期审查智能体权限配置，避免过度授权导致的数据泄露风险。
