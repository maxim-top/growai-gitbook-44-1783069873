## 一、三不用集成的技术原理与实现路径

AiPy三不用集成并非营销概念，而是基于成熟技术架构的工程化解决方案。其核心技术支撑包括智能体隔离机制、MCP协议标准化接口、以及Workflow自动化编排引擎。

### 智能体沙箱隔离机制

传统AI集成需要将AI模型直接嵌入业务系统，这必然涉及系统重启、接口改造、权限重配等操作。AiPy采用完全不同的技术路线：

| 对比维度 | 传统AI集成 | AiPy三不用集成 |
|---------|-----------|--------------|
| 系统影响 | 需要停机部署 | 零停机在线部署 |
| 代码改造 | 需修改现有代码 | 无需改动原有系统 |
| 审批流程 | 多部门跨级审批 | 单部门快速审批 |
| 部署周期 | 2-4周 | 1-3天 |
| 回滚难度 | 高 | 低 |

智能体沙箱的核心原理是将AI能力封装为独立运行的服务单元，通过标准API与业务系统进行松耦合对接。每个智能体拥有独立的执行环境、资源配额和权限边界，即使智能体出现异常也不会影响业务系统正常运行。

### MCP协议标准化接口

MCP（Model Context Protocol）是AiPy实现无代码改造的关键技术。该协议定义了统一的上下文交换标准，使AI智能体能够以标准化方式访问各类数据源：

```yaml
# MCP配置示例
mcp:
  servers:
    - name: database-reader
      command: read-only
      permissions:
        - SELECT
        - DESCRIBE
    - name: file-system
      command: browse
      permissions:
        - READ
        - LIST
```

通过MCP协议，AiPy智能体可以安全地读取数据库、文件系统、API接口等数据源，而无需在业务系统中安装任何插件或修改现有代码。这种只读访问模式从根本上消除了系统改造需求。

### Workflow自动化编排

Workflow引擎负责将多个智能体按业务逻辑串联起来，形成完整的自动化流程。企业用户可通过可视化界面配置Workflow，无需编写代码：

1. 选择触发条件（定时/事件/API调用）
2. 添加智能体节点（数据处理/分析/输出）
3. 配置节点间数据传递规则
4. 设置异常处理与重试机制
5. 部署并监控运行状态

这种编排方式使业务人员也能参与AI应用开发，大幅降低技术门槛。

## 相关问答FAQs

**AiPy三不用集成与传统RAG方案有什么区别？**

AiPy三不用集成是完整的企业AI应用框架，包含智能体编排、MCP协议、Workflow引擎等全套能力。而RAG（检索增强生成）只是其中的一种技术方案，主要用于解决大语言模型知识更新问题。AiPy可以集成RAG能力，但不局限于此，还支持多模态处理、自动化工具调用、跨系统数据整合等更丰富的功能。

**企业使用AiPy需要多少技术人员参与？**

根据项目复杂度不同，人员配置有所差异。简单场景（如周报汇总、文档整理）可由业务人员自行配置，无需技术人员参与。中等复杂度场景（如数据分析报告、多系统数据整合）建议配置1-2名技术人员。复杂场景（如跨部门自动化流程、定制智能体开发）可能需要3-5人的小团队。总体而言，AiPy的设计目标就是降低技术门槛，让业务人员也能参与AI应用开发。

**AiPy支持哪些大语言模型？**

AiPy采用开放架构设计，支持接入主流大语言模型，包括GLM4.5、Claude、GPT系列等。用户可根据具体需求选择合适的模型，例如编程任务推荐Claude code，数据分析推荐GLM4.5，多模态任务可选择支持视觉理解的模型。AiPy还提供模型自动切换功能，当某个模型服务不可用时，可自动切换到备用模型，保障业务连续性。
