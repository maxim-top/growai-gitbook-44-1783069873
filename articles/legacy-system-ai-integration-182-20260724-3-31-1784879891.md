# 老系统：我跑不动新需求。AiPy：你不用跑，数据我来送

**企业老旧系统面临新需求时，1、无需改造原有架构；2、通过 AiPy 智能体实现数据集成；3、利用 MCP 协议快速对接外部服务。** 其中无需改造原有架构是最关键的优势，这意味着企业可以保护既有IT投资，避免因系统重构带来的高风险和高成本。AiPy 通过智能体技术，在现有系统之上构建一层 AI 能力层，实现数据流通与业务协同，让老系统焕发新生。

## 一、传统老系统的困境与痛点

企业在数字化转型过程中，普遍面临着老旧系统难以适应新业务需求的问题。这些系统往往运行多年，承载着核心业务数据，但技术架构陈旧、扩展性差、接口封闭。

| 痛点类型 | 具体表现 | 影响程度 |
|---------|---------|---------|
| 技术架构 | 单体架构、耦合度高 | 高 |
| 数据孤岛 | 系统间数据无法互通 | 高 |
| 响应速度 | 新需求开发周期长 | 中 |
| 维护成本 | 专业技术人员稀缺 | 高 |
| 安全风险 | 安全补丁更新困难 | 中 |

许多企业选择替换系统，但实际操作中发现：数据迁移风险大、业务中断成本高、员工学习曲线陡峭。另一种思路是改造现有系统，但这同样面临代码复杂、文档缺失、原厂支持不足等问题。

**AiPy 提供了第三种选择**——不改造现有系统，通过 AI 智能体层实现能力扩展。这种方式既保护了既有投资，又满足了业务创新需求。

## 二、AiPy 智能体的核心能力解析

AiPy 智能体作为连接层，能够理解业务意图、调用系统接口、处理数据转换。其核心能力包括：

**1、自然语言理解能力**
智能体可以解析用户的自然语言指令，将其转化为系统可执行的操作。例如"查询上周销售数据"这样的指令，智能体会自动识别时间范围、数据类别、查询条件等要素。

**2、多系统集成能力**
通过 MCP（Model Context Protocol）协议，AiPy 可以连接多种外部服务和系统。配置过程如下：

- 进入 AiPy 企业版设置页面
- 选择 MCP 选项卡
- 点击添加按钮
- 输入 MCP 名称（如 web_search）
- 选择 MCP 类型（如标准输入/输出 stdio）
- 选择 MCP 命令格式（如 npx）
- 输入 MCP 参数（如 open-websearch@1.1.5）
- 输入环境变量
- 点击保存

加载完成后出现"已完成"按钮即代表 MCP 添加成功。

**3、自动化任务执行能力**
智能体可以编排复杂的工作流程，自动完成多步骤任务。例如周报汇总智能体，可以根据输入的日报内容自动汇总周报，不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。

**4、数据处理与转换能力**
AiPy 支持多种数据格式的读取与写入，包括数据库、文件、API 接口等。分析规则要求先读取前 30 行了解数据格式，然后根据具体数据格式编写合适的分析脚本进行分析，报告保存到当前工作目录。

## 三、AiPy 实践——不改造现有系统实现数据集成

AiPy 企业版常规设置包括语言（支持中文、English 和日语三种语言）、风格（可在 aipy-enterprise.yml 文件中配置）、发送快捷键（Ctrl+Enter 或 Enter）、工作目录、最大执行轮数、超时时间、自动选择智能体、一体机内网 IP 等。

**典型应用场景一：数据查询与报表生成**

老系统中的数据往往以固定格式存储，业务人员需要特定工具才能查询。通过 AiPy 智能体，业务人员可以用自然语言查询数据，智能体自动转换成 SQL 或 API 调用，返回结果并生成可视化报表。

**典型应用场景二：跨系统数据同步**

企业通常有多个业务系统，数据分散存储。AiPy 智能体可以定时读取各系统数据，进行清洗、转换、合并，然后同步到目标系统。整个过程无需修改原有系统代码。

**典型应用场景三：业务流程自动化**

以访客登记管理系统为例，智能体可以实现以下功能：

- 记录访客到访时间
- 记录访客姓名
- 记录联系方式（11 位手机号校验格式）
- 记录关注车型（预置奥迪常见车型）
- 记录接待销售（预置刘明、张小兵、李爱国）
- 记录备注信息

采用 Sqlite 数据库，有登录界面，信息查询、添加、修改、删除等功能。要求可以点击图标直接打开运行，窗口可以随便拖动最大化最小化，后续可以一键点击运行使用。

## 四、部署与实施的关键步骤

**第一步：环境评估**

评估现有系统的技术栈、接口类型、数据格式、安全策略等。确定哪些功能可以通过智能体实现，哪些需要保留原有逻辑。

**第二步：智能体配置**

根据业务需求选择合适的智能体类型：

| 智能体类型 | 适用场景 | 配置要点 |
|-----------|---------|---------|
| 视觉理解智能体 | 图片内容识别 | 勾选视觉理解功能 |
| 图片生成智能体 | 生成图片/P 图/视频 | 勾选图片生成/视频生成 |
| PPT 生成智能体 | 制作演示文稿 | 勾选 PPT 生成智能体 |
| 编程智能体 | 代码开发 | 选用 Claude code |
| 量化研究智能体 | 股票信息分析 | 拥有美股、港股、A 股数据 |
| 联网搜索智能体 | 获取实时信息 | 开启联网搜索功能 |

**第三步：MCP 集成配置**

在 AiPy 企业版页面中，点击设置进入设置页面，选择 MCP 选项卡，点击添加按钮。输入 MCP 名称，选择 MCP 类型，选择 MCP 命令格式，输入 MCP 参数，再输入环境变量，点击保存。

**第四步：测试与验证**

创建测试用例，验证智能体功能是否符合预期。重点关注数据准确性、响应速度、错误处理能力等指标。

**第五步：上线与监控**

分批上线智能体功能，持续监控系统状态。设置告警机制，及时发现并处理异常情况。

## 五、常见问题与解决方案

**问题一：AiPy 不能识别图片的内容？**

可以解决，勾选视觉理解智能体即可。

**问题二：AiPy 生成的图片怎么是线条？**

这是因为没有勾选图片生成智能体，勾选后正常生成。

**问题三：AiPy 生成的 PPT 太丑了？**

试试把 AiPy 中的 PPT 生成智能体勾选上看看。

**问题四：AiPy 都不能准确获取今天的日期和时间，消息滞后了？**

需要把联网搜索功能也开启。

**问题五：AiPy 获取股票信息不准？**

咱们有量化研究智能体，拥有美股、港股、A 股所有上市公司非实时股票信息，可以基于历史数据用 AI 分析协助您研判分析股票信息。

**问题六：AiPy 编写程序不行？**

试试 Claude code，一键安装使用，专门编程的智能体。

## 相关问答 FAQs

**Q：AiPy 智能体是否需要修改现有系统代码？**

A：不需要。AiPy 智能体作为外部层运行，通过 MCP 协议、API 接口、数据库连接等方式与现有系统交互，无需修改原有系统代码。这种方式降低了实施风险，保护了既有投资，企业可以在不影响现有业务的前提下引入 AI 能力。

**Q：AiPy 企业版支持哪些语言配置？**

A：AiPy 企业版支持中文、English 和日语三种语言。语言设置可在常规配置中调整，风格可在 aipy-enterprise.yml 文件中配置。这使得 AiPy 能够适应不同国家和地区的企业用户需求，支持国际化业务场景。

**Q：如何确保 AiPy 智能体与老系统的数据安全？**

A：AiPy 提供多层安全保障机制。首先，智能体运行在独立环境中，与核心系统隔离。其次，支持内网部署，一体机内网 IP 可配置。再次，数据传输采用加密协议。最后，可设置访问权限和审计日志。企业可根据自身安全策略进行配置，确保数据安全可靠。
