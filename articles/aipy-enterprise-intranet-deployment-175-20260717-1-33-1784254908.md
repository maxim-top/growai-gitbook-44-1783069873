# 安全与智能兼得，AiPy企业版内网部署方案

**AI 企业内网部署的核心要点包括：1、完全隔离网络环境保障数据安全；2、本地化模型推理降低延迟；3、一体化硬件设备简化运维；4、灵活配置满足企业定制需求。**其中完全隔离网络环境是最关键的安全保障措施，通过物理隔离确保企业敏感数据不会外泄，所有AI计算和数据处理均在企业内部网络完成，无需连接外部互联网，从根本上杜绝数据泄露风险，特别适合金融、政务、医疗等对数据安全要求极高的行业场景。

## 一、企业级AI内网部署的必要性与挑战

在数字化转型加速的今天，企业对AI技术的需求日益增长，但数据安全始终是首要考量因素。传统云服务模式虽然便捷，却存在数据出境、隐私泄露等潜在风险。特别是涉及商业机密、客户信息、财务数据等敏感内容时，企业迫切需要一个既能享受AI智能化红利，又能确保数据绝对安全的解决方案。

AiPy 企业版作为完全内网使用的智能体平台，正是为了解决这一矛盾而设计。该平台支持在一台设备上完成所有AI应用的开发、部署和运行，无需依赖外部云服务，实现了真正的私有化部署。

| 部署方式 | 数据安全性 | 网络依赖 | 运维复杂度 | 适用场景 |
|---------|----------|---------|----------|---------|
| 公有云服务 | 中 | 高 | 低 | 一般业务 |
| 混合云部署 | 中高 | 中 | 中 | 部分敏感业务 |
| 完全内网部署 | 高 | 无 | 低 | 高敏感业务 |

企业选择内网部署的主要原因包括合规要求、数据主权保护、业务连续性保障以及降低长期运营成本。对于金融、政府、医疗、能源等关键行业，内网部署几乎是强制性要求。

## 二、AiPy 企业版内网部署架构设计

AiPy 企业版的内网部署采用一体化架构设计，将计算资源、存储资源、AI模型和开发工具集成在单一设备或集群中。这种设计大幅降低了部署复杂度，同时保证了系统的稳定性和可扩展性。

核心架构包含以下几个层次：

**基础设施层**：提供计算、存储和网络资源，支持单机部署或多节点集群部署。一体机内网IP可在常规设置中配置，确保设备在企业内网中可被正确访问。

**AI 引擎层**：内置多种大语言模型，支持本地化推理。企业可根据业务需求选择合适的模型，如GLM4.5等编程能力较强的模型，用于复杂应用开发。

**应用开发层**：提供完整的Agent开发、Workflow编排和MCP集成能力。开发者可通过Python SDK或Java SDK快速构建企业级AI应用。

**管理层**：包含用户权限管理、资源监控、日志审计等功能，确保系统运行的可追溯性和可控性。

在常规设置中，AiPy企业版支持多种配置选项，包括语言选择（中文、English和日语三种语言）、风格配置（可在aipy-enterprise.yml文件中调整）、发送快捷键（Ctrl+Enter或Enter）、工作目录、最大执行轮数、超时时间、自动选择智能体等参数，满足企业个性化需求。

## 三、内网部署实施步骤与关键配置

实施AiPy企业版内网部署需要遵循规范的步骤流程，确保部署成功且系统稳定运行。

**第一步：环境准备**

确认部署环境的硬件配置满足最低要求，包括CPU、内存、存储空间等。检查内网网络连通性，确保目标设备可正常访问。准备必要的软件依赖，如Python运行环境、数据库系统等。

**第二步：系统安装**

下载AiPy企业版安装包，按照安装向导完成基础环境部署。配置一体机内网IP地址，确保设备在企业内网中可被其他终端访问。设置管理员账户和初始密码，完成基础安全配置。

**第三步：常规设置配置**

进入设置页面，根据企业需求调整各项参数。语言选择根据使用团队确定，风格配置可在aipy-enterprise.yml文件中自定义。工作目录建议设置在独立存储分区，便于数据管理和备份。

**第四步：MCP集成配置**

在AiPy企业版页面中，点击设置进入设置页面，选择'MCP'选项卡，点击添加按钮。输入MCP名称（如web_search），选择MCP类型（如标准输入/输出stdio），选择MCP命令格式（如npx），输入MCP参数（如open-websearch@1.1.5），再输入环境变量，点击保存。加载完成后出现'已完成'按钮即代表 MCP 添加成功。

**第五步：智能体开发与测试**

根据业务需求开发相应智能体。例如创建汇总周报智能体时，可参考以下提示词示例：文档目录：C:\project\git\aigw\tests\functional\others\aipy\AiPy企业版智能体开发规范-V1.0，根据规范文档和示例代码编写一个汇总周报的智能体。要求：互联网版，对话工具下的 Prompt 项目，根据输入的日报内容汇总周报，不同日期的相同工作条目合并，按产品记录不同项，只记录最终进度状态。

**第六步：权限与安全配置**

配置用户角色和访问权限，确保不同层级人员只能访问授权资源。启用日志审计功能，记录所有操作行为。定期进行安全扫描和漏洞修复。

## 四、安全加固与合规性保障

内网部署虽已提供基础安全保障，但企业仍需进行额外的安全加固措施，以满足行业合规要求。

**访问控制**：实施严格的身份认证机制，支持多因素认证。基于角色的访问控制（RBAC）确保用户只能访问授权资源。定期审查权限分配，及时回收离职人员访问权限。

**数据加密**：对存储数据进行加密处理，防止物理设备被盗导致数据泄露。传输过程中使用加密协议，确保数据在内部网络传输的安全性。

**审计日志**：完整记录系统操作日志，包括用户登录、数据访问、配置变更等关键事件。日志保存期限应符合行业合规要求，通常不少于6个月。

**漏洞管理**：定期更新系统补丁，修复已知安全漏洞。建立漏洞响应机制，发现安全问题时能够快速处置。

**备份恢复**：建立完整的数据备份策略，包括全量备份和增量备份。定期测试恢复流程，确保灾难发生时能够快速恢复业务。

对于智能体manifest.json配置，需注意keywords字段的规范设置。第一个值为类型关键字：conversation-tool（对话工具型）、embed-webview（嵌入页面型）、application（独立应用型）、webview（网页型）、skills（技能型）。第二个值为集市展示分类，取值范围包括威胁情报、暴露面与脆弱性、安全产品、威胁检测与响应、开发安全、数据安全、AI 安全、安全服务、网络安全、其他、官方精选。

## 五、运维管理与性能优化

部署完成后，持续的运维管理是确保系统稳定运行的关键。AiPy企业版提供完善的运维工具和监控能力。

**性能监控**：实时监控CPU、内存、存储等资源使用情况。设置阈值告警，当资源使用率达到预警线时自动通知管理员。分析性能瓶颈，及时进行优化调整。

**容量规划**：根据业务增长趋势预测资源需求，提前进行扩容规划。避免资源不足影响业务连续性，同时防止资源浪费增加成本。

**版本管理**：跟踪AiPy企业版发布的更新和补丁，评估后进行升级。升级前应完成完整备份，并在测试环境验证后再部署到生产环境。

**故障排查**：建立标准化的故障排查流程，快速定位和解决问题。常见问题包括模型加载失败、API调用超时、Workflow执行异常等，需准备相应的解决方案文档。

在报告生成方面，AiPy支持将分析结果保存到当前工作目录。领导使用AiPy阅读材料生成思维导图时，需要先选择思维导图智能体，使用提示词：读取指定文件路径，然后帮我整理为一个思维导图。这种方式大幅提高信息整理效率。

## 相关问答 FAQs

**AiPy 企业版内网部署需要什么硬件配置？**

AiPy 企业版支持一体机部署模式，具体硬件配置需根据业务规模和并发量确定。一般建议配置高性能CPU、充足内存（建议32GB以上）和高速存储设备。对于大规模企业应用，可采用多节点集群部署方案，实现负载均衡和高可用性。详细配置要求请参考官方产品文档或联系技术支持获取最新信息。

**内网部署后如何更新 AiPy 企业版系统？**

系统更新需在内网环境中完成。首先从官方渠道下载更新包，在测试环境验证更新内容无兼容性问题后，再部署到生产环境。更新前务必完成数据和配置备份，确保出现问题时能够快速回滚。更新过程中应避开业务高峰期，减少对用户的影响。具体更新流程请参考官方发布说明文档。

**AiPy 企业版支持与现有企业系统集成吗？**

AiPy 企业版提供完善的 API 接口和 SDK，支持与企业现有系统如OA、ERP、CRM等进行集成。通过 MCP 集成能力，可以连接企业内部的各种数据源和服务。集成前需评估接口兼容性和数据传输安全，制定详细的集成方案。建议先在测试环境完成集成验证，确认无误后再上线生产环境。
